# 举报账号

:::warning 负责任地استخدام
举报功能نعم一个强力操作。仅عندما确认存在违法أو违规（欺诈、诈骗、垃圾معلومات等）时才استخدام。滥用举报可能导致账号被限制أو其他不良后果。
:::

本指南الوصف如何استخدام Super Marketing 脚本内置的“举报账号”功能。该功能会自动执行固定的应用内举报流程以处理一批用户账号，但不支持在 UI أو脚本中自定义举报路径أو原因。

## 快速概览

- 脚本استخدام的固定举报流程كما يلي（不可修改）：

```text
举报 → 举报账号 → Something else → Frauds and scams
```

- 脚本会尝试进入目标用户主页，打开更多/菜单项，按固定路径选择举报并提交（若可用）。
- 由于手机端应用在不同版本与设备上 UI 可能存在差异，脚本استخدام尽力而为的选择器，因此请在小批量上先行验证。

## 通过 Super Marketing 执行的步骤

1. 打开 TikMatrix/IgMatrix 并选择要运行的设备（支持多选）。
2. 打开 `Scripts` → `Super Marketing`。
3. 将 **数据来源（Data source）** 设置为 `Usernames`，导入文本文件أو粘贴用户名（一行一个）；导入后选择“追加”أو“覆盖”。
4. 在 **用户动作（User Actions）** 中启用 **举报账号（Report account）** 开关，并确认显示的固定举报路径。
5. التكوين任务并发、延迟等المعاملات。
6. 点击 `Start Script` 开始处理列表，监控日志与设备界面以确认操作成功أو失败。

## 建议与ملاحظات

- 先在 1–5 个账号上测试，确认脚本能正确识别并执行举报流程（UI 文本أو位置变化会导致失败）。
- 保持保守的节奏与延迟，批量自动举报可能触发平台的限流أو风控。
- 如果应用 UI 发生变化（按钮文本、布局变动），脚本可能无法找到对应元素，需要更新选择器أو调整等待策略。
- 仅对确凿违规内容استخدام举报，避免骚扰أو针对无辜用户滥用该功能。
- 运行后检查日志，脚本会记录诸如“找不到菜单按钮”أو“找不到举报الخيارات”的错误，便于你调整脚本أو运行المعاملات。

## 不建议استخدام的场景

- 需要为每个案例选择自定义举报原因أو上传证据的场景：脚本不支持细粒度自定义。
- 法律/敏感事件需要人工审核أو保留证据的情况。

## 故障排查

- 报错 “Could not find Report option” / “Could not find 'Report account' option”：表示应用 UI 文本أو布局已变化。请在小批量下再次运行并收集设备截图用于调试。
- 如果脚本显示已提交举报但平台没有实际处理，平台可能对自动举报作了降权أو需要人工审查；针对مهم个案，请استخدام人工流程。
