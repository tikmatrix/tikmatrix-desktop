# Super Marketing

:::info
O script Super Marketing combina seguir/deixar de seguir, DM, interações com posts e comentários em uma única fila de tarefas, ideal para alcançar rapidamente clientes potenciais ou reengajar seguidores em uma operação.
:::

## Etapas para Executar

1. No TikMatrix/IgMatrix, selecione o(s) dispositivo(s) nos quais deseja executar o script (seleção múltipla suportada).
2. Abra `Scripts` → `Super Marketing`.
3. Escolha **Fonte de dados** (Nomes de usuário ou Links de posts), depois clique em `Importar arquivo txt` para carregar um arquivo `.txt` ou cole várias linhas diretamente; após a importação, escolha `Anexar` ou `Sobrescrever`.
4. Ative os módulos Ações de Usuário, Ações de Post e Comentários conforme necessário e configure-os de acordo com as instruções abaixo.
5. Defina o controle deslizante **Intervalo de tarefas** para definir o atraso aleatório entre alvos e clique em `Iniciar Script` para executar a tarefa.

## Configuração

### Fonte de Dados

- **Nomes de usuário / Links de posts**: determina quais recursos estão disponíveis. Seguir, deixar de seguir e DM só podem ser usados com nomes de usuário; se você só quiser interagir com posts, escolha o modo de link.
- **Método de importação**: Clique em `Importar arquivo txt` para carregar um arquivo `.txt` ou cole várias linhas diretamente; após a importação, escolha `Anexar` ou `Sobrescrever`.

### Estratégia de Uso de Dados

- **Pool compartilhado**: Todas as contas bot extraem tarefas do mesmo pool de alvos, adequado para cenários onde várias contas precisam interagir com os mesmos alvos várias vezes (ex.: push concentrado em posts populares).

- **Consumo único**: Cada conta bot processa apenas alvos diferentes, garantindo que cada alvo seja tratado por apenas uma conta uma vez, adequado para cenários que precisam evitar alcance duplicado (ex.: DM em massa ou comentários únicos únicos).

  - **Limite de consumo por tarefa**: Ao usar o modo de consumo único, você pode definir um número máximo de alvos que cada execução de tarefa consumirá. Isso evita que o primeiro lote de tarefas consuma todos os dados disponíveis, garantindo que rotações de contas subsequentes tenham alvos para processar. Defina como 0 para consumo ilimitado (comportamento padrão).

Escolha uma estratégia com base em seus objetivos e considerações de gerenciamento de riscos; ela afeta a distribuição de filas e o comportamento de concorrência.

### Ações de Usuário

- **Seguir usuários**: Executa a ação de seguir primeiro ao entrar na página de perfil.
- **Deixar de seguir usuários**: Deixa de seguir o usuário; pode executar isoladamente ou limpar seguidores após DM; não é recomendado ativar com "Seguir" simultaneamente.
- **Denunciar conta**: Denuncia a conta de usuário por comportamento fraudulento. Este recurso usa um caminho de denúncia fixo e não suporta caminhos personalizados.

  :::warning Caminho de Denúncia (Fixo)
  Denunciar → Denunciar conta → Outra coisa → Fraudes e golpes
  :::

  - Nota: Este recurso usa um caminho de denúncia predefinido. Razões ou caminhos de denúncia personalizados não são suportados.
  - Use com responsabilidade e apenas para fins legítimos. O uso indevido pode resultar em restrições de conta.

- **Enviar DM**: Seleciona aleatoriamente uma mensagem dos modelos multilinha; ative "Inserir emoji" para anexar emojis naturais automaticamente. O script detectará se o destinatário permite DMs e pulará se não estiver disponível.

  - Modelos de DM (pré-gerados): Em "Configurações de Mensagem", cole modelos de DM multilinha ou multi-parágrafo (cada linha/parágrafo é uma mensagem candidata).

  - Ordem de uso do modelo: Escolha "Aleatório" ou "Sequencial".

    - Aleatório: Seleciona aleatoriamente um modelo cada vez.

    - Sequencial: Rotaciona modelos na ordem em que foram colados/importados, adequado para estratégias de desduplicação reproduzíveis.

  - Inserir emoji: Ative para inserir automaticamente emojis em mensagens ou comentários para aumentar a naturalidade.

  - Geração em tempo real (opcional, ChatGPT ou API compatível):

    - Quando ativada, o script pode chamar seu endpoint de geração configurado (preencha API Key / URL / modelo / System Prompt) para gerar texto personalizado com base no contexto do alvo.

    - Nota: Forneça uma API Key válida e esteja ciente dos limites de taxa, custos e conformidade de privacidade (evite enviar dados sensíveis).

  - Estratégias de unicidade: Formas comuns de garantir que cada mensagem/comentário seja único:

      1. Use o modo "Sequencial" para rotacionar modelos pré-gerados (modelos usados apenas uma vez até esgotar).
      2. Use geração em tempo real (ChatGPT) para gerar texto único para cada alvo.

  - Antes de usar DM em massa, certifique-se de testar com lotes pequenos e prestar atenção aos requisitos de privacidade e conformidade.

### Ações de Post

- **Máximo de posts para processar**: Determina o número de posts que cada conta processará (1–50), necessário apenas quando a fonte de dados for "Nomes de usuário".
- **Repetir vezes**: Repete o loop de interação com posts várias vezes sem reiniciar o script.
- **Duração de visualização**: Defina o tempo de assistir mínimo e máximo (0–300 segundos). O script selecionará aleatoriamente um valor dentro do intervalo antes de realizar interações.
- **Interações com posts**: Pode ativar individualmente Curtir, Favoritar, Repostar e Compartilhar. Quando várias ações são ativadas, o sistema as distribuirá em diferentes pontos no tempo durante o período de visualização para um comportamento mais natural.

- **Comentários**: Quando ativado, você pode postar comentários em posts.

     Se "Comentários" estiver ativado, configure o seguinte em Configurações de Comentário:

  - Modelos de comentários pré-gerados: Cole modelos de comentários multilinha ou multi-parágrafo pré-gerados (cada linha ou parágrafo é um comentário candidato).

  - Ordem de uso do modelo: Escolha "Aleatório" ou "Sequencial".
    - Aleatório: Seleciona aleatoriamente dos modelos candidatos cada vez.
    - Sequencial: Usa modelos na ordem em que foram colados/importados (adequado para estratégias de desduplicação reproduzíveis).

  - Inserir emoji: Ative para inserir automaticamente emojis em comentários para aumentar a naturalidade.

  - Geração de comentários em tempo real (opcional, API ChatGPT):
    - Quando ativada, o script pode chamar APIs semelhantes ao ChatGPT para gerar texto de comentário único e relevante ao contexto com base no título/descrição do post em tempo real. Este método pode reduzir significativamente o risco de comentários duplicados, adequado para cenários que requerem um grande número de comentários únicos.
    - Nota: Usar geração em tempo real requer uma API Key válida e esteja ciente dos limites de taxa, custos e conformidade de privacidade (não envie dados sensíveis para serviços de terceiros).

  - Duas formas comuns de garantir comentários únicos:
         1. Use "Sequencial" para carregar e rotacionar modelos de comentários pré-gerados (garantindo que cada modelo seja usado apenas uma vez até esgotar).
         2. Use a API ChatGPT para gerar cada comentário em tempo real (gerado dinamicamente com base no título/conteúdo do post, naturalmente mais fácil de garantir unicidade).

  - Sugestão: Teste a estratégia de comentários com lotes pequenos primeiro, verifique a qualidade dos comentários e o tratamento da plataforma; se usar geração em tempo real, monitore o volume de chamadas da API e os custos.

  - Se usar a estratégia "Sequencial", o sistema combinará números de tarefas para rotacionar modelos em vários dispositivos, reduzindo a probabilidade de duplicação; se usar geração de IA, é recomendado aplicar regras simples de verificação de qualidade (como intervalo de comprimento, proibir URLs / palavras sensíveis).

### Mesclar tarefas (Mesclar tarefas para o mesmo nome de usuário)

- Descrição: Quando "Mesclar tarefas para o mesmo nome de usuário" está ativado, vários alvos para o mesmo nome de usuário são executados dentro de uma única unidade de tarefa (sequencialmente ou em paralelo dependendo da implementação). Mesmo se um alvo falhar, a tarefa geral não será abortada e os alvos restantes continuarão a ser executados, reduzindo a sobrecarga de troca e agendamento de tarefas.

- Comportamento quando ativado: Vários alvos direcionados à mesma conta são agrupados em uma única unidade de tarefa. As falhas afetam apenas o alvo individual e são registradas, mas não bloquearão outros alvos de continuar. Este modo é adequado para execuções de marketing em larga escala e pode reduzir significativamente a sobrecarga de início/troca de tarefas.

- Comportamento quando desativado: Cada alvo é criado como uma tarefa individual; qualquer falha é relatada imediatamente e interrompe essa sub-tarefa (sem afetar outras tarefas independentes). Este modo é adequado quando você precisa de monitoramento preciso por alvo e deseja ação imediata em falhas (ex.: intervenção humana ou reversão imediata).

- Recomendações:
  - Ative este modo para marketing de alta taxa de transferência e lotes grandes para reduzir os custos de agendamento.
  - Desative quando precisar de status de execução exato por alvo e desejar ação imediata em falhas (ex.: intervenção humana ou reversão imediata).

### Intervalo de Tarefas

- Defina valores mínimo e máximo (em minutos). O script esperará aleatoriamente dentro do intervalo entre cada alvo, mais suas próprias pausas curtas.

## Fluxo de Execução

- Se o modo de várias contas estiver ativado, TikMatrix/IgMatrix primeiro lançará TikTok/Instagram e mudará para a conta correspondente.
- No modo de nome de usuário, o sistema abre cada página de perfil usando o método de acesso selecionado, aguarda o carregamento ser concluído e então executa sequencialmente as ações seguir, DM, deixar de seguir; se o destinatário bloquear DMs, pulará automaticamente.
- As interações com posts percorrerão páginas de perfil ou abrirão links diretamente, assistirão de acordo com a quantidade e duração definidas e realizarão ações de curtir, favoritar, repostar, compartilhar, pulando anúncios, transmissões ao vivo e conteúdo já curtido.
- Ao usar links de posts, o script abrirá links um por um, executará ações configuradas e fará loop de acordo com "Repetir vezes".
- O módulo de comentários extrairá de modelos ou saída de IA. O modo sequencial combinará números de tarefas para distribuir comentários diferentes em vários dispositivos.
- Entre cada alvo, o script adicionará pequenas pausas aleatórias e então adicionará a espera aleatória do intervalo de tarefas para simular o ritmo humano.

## Melhores Práticas

- Valide texto e ritmo com lotes pequenos primeiro, depois amplie após confirmar que não há problemas.
- Combinar várias ações (como seguir + DM + curtir) pode melhorar os sinais de engajamento da conta.
- Mantenha DMs e comentários conversacionais e sinceros para evitar acionar a moderação da plataforma.
- Atualize regularmente bibliotecas de comentários ou ajuste o System Prompt da IA para tornar o conteúdo mais relevante aos temas da campanha.
- Monitore os limites de taxa da plataforma; para volumes grandes, rotacione contas ou nós de proxy.
