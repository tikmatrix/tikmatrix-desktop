# Support des applications clonées

TikMatrix et IgMatrix prennent en charge l'exécution de plusieurs applications clonées (par exemple GB TikTok, GB Instagram, Instagram Plus ou d'autres versions modifiées) sur le même appareil, chaque application ayant son propre nom de package unique. Cette fonctionnalité vous permet de gérer plusieurs comptes plus efficacement en utilisant différentes instances d'application.

:::danger Avertissement de risque
L'utilisation d'applications clonées peut violer les conditions d'utilisation de TikTok/Instagram et entraîner la suspension ou la restriction des comptes. Nous fournissons uniquement le support technique de cette fonctionnalité et **ne sommes pas responsables** de tout problème de compte résultant de l'utilisation d'applications clonées, y compris les suspensions, restrictions ou autres sanctions. Vous assumez les risques d'utilisation.
:::

## Qu'est-ce qu'un nom de package ?

Chaque application Android possède un identifiant unique appelé **nom de package**. Les applications officielles utilisent des noms de package standard :

- TikTok : `com.zhiliaoapp.musically`
- Instagram : `com.instagram.android`

Les applications clonées utilisent des noms de package différents, par exemple :

- GB TikTok : `com.zhiliaoapp.musically.gb`
- GB Instagram : `com.instagram.gb.android`
- Instagram Plus : `com.instagram.plus.android`

TikMatrix/IgMatrix peut détecter automatiquement et utiliser le nom de package correct pour chaque compte.

## Configuration des applications clonées

### Étape 1 : Installer l'application clonée

1. Installez votre application clonée préférée sur votre appareil ou téléphone cloud.
2. Assurez-vous que chaque application clonée a un nom de package unique pour éviter les conflits.
3. Connectez-vous avec des comptes différents dans chaque instance d'application.

### Étape 2 : Configurer le nom de package

Vous avez deux façons de configurer le nom de package :

#### Méthode A : Configuration au niveau du compte (recommandé)

Définissez un nom de package spécifique pour chaque compte :

1. Ouvrez **Comptes** dans la navigation de gauche.
2. Cliquez sur un compte pour l'éditer, ou cliquez sur **Nouveau** pour créer un nouveau compte.
3. Dans le champ **Nom de package**, vous pouvez :
   - **Saisir manuellement** le nom de package de l'application clonée (par exemple `com.instagram.gb.android`)
   - **Détecter automatiquement** en cliquant sur le bouton **Détecter le nom de package** (nécessite d'abord de sélectionner un appareil)
4. Cliquez sur **Enregistrer**.

:::tip Détection automatique du nom de package
Le bouton **Détecter le nom de package** peut détecter automatiquement l'application actuellement ouverte sur l'appareil sélectionné. C'est le moyen le plus simple d'obtenir le nom de package correct :

1. Sélectionnez un appareil pour le compte
2. Ouvrez l'application clonée sur cet appareil
3. Cliquez sur le bouton **Détecter le nom de package**
4. Le nom de package sera automatiquement rempli

Cette fonctionnalité fonctionne avec toutes les applications Android, y compris les applications clonées TikTok/Instagram, les applications App Cloner et les applications NomixCloner.
:::

Lorsque vous exécutez un script pour ce compte, TikMatrix/IgMatrix utilisera automatiquement le nom de package spécifié.

#### Méthode B : Valeur par défaut globale du système

Définissez un nom de package par défaut pour tous les comptes qui n'ont pas de nom de package spécifique :

1. Ouvrez **Paramètres** dans la navigation de gauche.
2. Trouvez le champ **Nom de package TikTok** ou **Nom de package Instagram**.
3. Saisissez le nom de package par défaut.
4. Cliquez sur **Enregistrer**.

:::tip Priorité
Les paramètres au niveau du compte remplacent toujours les valeurs par défaut globales du système. Si un compte a configuré son propre nom de package, celui-ci sera utilisé au lieu de la valeur par défaut.
:::

## Fonctionnement

Lorsque vous exécutez un script d'automatisation :

1. TikMatrix/IgMatrix vérifie si le compte cible a configuré un nom de package personnalisé.
2. Si oui, il utilise ce nom de package dans toutes les opérations d'automatisation UI.
3. Si non, il utilise le nom de package par défaut global du système.
4. Tous les ID de ressources et requêtes XPath sont automatiquement formatés avec le nom de package correct.

## Meilleures pratiques

### 1. Utiliser des noms d'utilisateur descriptifs

Comme vous pouvez avoir plusieurs comptes dans différentes applications clonées, utilisez des noms d'utilisateur descriptifs pour identifier à quelle application appartient chaque compte :

```text
username_gb      → GB TikTok/Instagram
username_plus    → TikTok/Instagram Plus
username         → Application officielle
```

### 2. Organiser les comptes par groupes

Créez des groupes d'appareils en fonction du type d'application clonée :

- **Groupe A** : Tous les comptes utilisant l'application GB
- **Groupe B** : Tous les comptes utilisant l'application Plus
- **Groupe C** : Tous les comptes utilisant l'application officielle

### 3. Tester avant les opérations en masse

Avant d'exécuter un script sur tous les comptes :

1. Testez d'abord avec un seul compte
2. Vérifiez que la bonne application est utilisée
3. Confirmez que toutes les opérations fonctionnent comme prévu

### 4. Maintenir la cohérence des noms de package

Une fois qu'un nom de package est défini pour un compte, ne le modifiez pas à moins de passer à une application clonée différente. Maintenir la cohérence aide à éviter la confusion et les erreurs.

## Dépannage

### Le script ouvre la mauvaise application

**Problème** : Le script ouvre la mauvaise application au lieu de l'application clonée.

**Solution** :

1. Vérifiez que le champ **Nom de package** du compte est correctement défini
2. Vérifiez que l'application clonée est installée et que le nom de package correspond
3. Utilisez le bouton **Détecter le nom de package** pour obtenir automatiquement le nom de package correct
4. Si l'application a été récemment installée, redémarrez l'appareil

### Élément UI introuvable

**Problème** : Le script signale des erreurs "élément introuvable".

**Solution** :

1. Différentes applications clonées peuvent avoir des dispositions UI légèrement différentes
2. Vérifiez que la version de l'application clonée est compatible
3. Essayez de mettre à jour vers la dernière version de TikMatrix/IgMatrix
4. Si le problème persiste, contactez le support technique

### Plusieurs comptes dans la même application

**Problème** : Vous souhaitez basculer entre plusieurs comptes dans la même instance d'application.

**Solution** :

- TikTok/Instagram prend nativement en charge la connexion multi-comptes
- Utilisez la fonctionnalité **Changer de compte** de TikMatrix/IgMatrix
- Définissez le même nom de package pour tous les comptes dans cette instance d'application
- TikMatrix/IgMatrix gérera automatiquement le basculement de compte

### Le bouton de détection du nom de package ne fonctionne pas

**Problème** : Le bouton **Détecter le nom de package** ne fonctionne pas ou affiche une erreur.

**Solution** :

1. Assurez-vous d'avoir d'abord sélectionné un appareil
2. Assurez-vous que l'application est actuellement ouverte sur l'appareil
3. Vérifiez que la connexion de l'appareil est stable
4. Essayez de rouvrir l'application et de cliquer sur détecter

## Trouver les noms de package

Si vous souhaitez vérifier manuellement les noms de package sur votre appareil :

### Utiliser les commandes ADB

```bash
# Lister tous les packages liés à TikTok
adb shell pm list packages | grep musically

# Lister tous les packages liés à Instagram
adb shell pm list packages | grep instagram
```

### Utiliser la fonctionnalité de détection automatique

Le moyen le plus simple est d'utiliser le bouton **Détecter le nom de package** intégré :

1. Ouvrez la boîte de dialogue d'édition du compte
2. Sélectionnez un appareil
3. Ouvrez l'application clonée sur l'appareil
4. Cliquez sur **Détecter le nom de package**
5. Le nom de package sera automatiquement détecté et rempli

Cette méthode fonctionne avec toutes les applications Android et est recommandée.

## Questions fréquentes

**Q : Puis-je utiliser plusieurs applications clonées sur le même appareil ?**

R : Oui, tant que chaque application clonée a un nom de package unique, vous pouvez exécuter plusieurs instances d'application sur le même appareil.

**Q : Dois-je réinstaller TikMatrix/IgMatrix ?**

R : Non, cette fonctionnalité est intégrée dans la dernière version. Mettez simplement à jour vers la dernière version et configurez les noms de package.

**Q : Mes scripts d'automatisation existants fonctionneront-ils toujours ?**

R : Oui, tous les scripts existants sont entièrement compatibles. Les comptes sans nom de package personnalisé utiliseront le nom de package par défaut.

**Q : Est-ce supporté pour TikTok et Instagram ?**

R : Oui, TikMatrix et IgMatrix prennent tous deux entièrement en charge la détection et l'automatisation des applications clonées.

**Q : Comment fonctionne la détection automatique ?**

R : Lorsque vous cliquez sur **Détecter le nom de package**, TikMatrix/IgMatrix vérifie l'application actuellement au premier plan sur l'appareil sélectionné et obtient automatiquement son nom de package.

**Q : Puis-je utiliser des applications créées avec App Cloner ou NomixCloner ?**

R : Oui, la fonctionnalité de nom de package fonctionne avec tout outil de clonage d'applications Android, y compris App Cloner et NomixCloner. Utilisez la fonctionnalité de détection automatique pour obtenir facilement le nom de package correct.
