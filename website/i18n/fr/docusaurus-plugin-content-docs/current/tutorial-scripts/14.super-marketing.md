# Super Marketing

:::info
Le script Super Marketing centralise les actions de suivi/désabonnement, messages directs, interactions avec les publications et commentaires dans une seule file d'attente de tâches, idéal pour atteindre rapidement des prospects ou réactiver d'anciens abonnés en une seule opération.
:::

## Procédure

1. Dans TikMatrix/IgMatrix, sélectionnez les appareils qui doivent exécuter le script (sélection multiple possible).
2. Ouvrez `Scripts` → `Super Marketing`.
3. Sélectionnez la **Source de données** (noms d'utilisateur ou liens de publications), puis vous pouvez cliquer sur le bouton `Importer un fichier txt` pour charger un fichier `.txt` correspondant ou coller directement plusieurs lignes de données ; après l'importation, choisissez "Ajouter" ou "Remplacer".
4. Activez les modules d'actions utilisateur, d'actions sur les publications et de commentaires selon vos besoins, et complétez la configuration en suivant les instructions ci-dessous.
5. Réglez le curseur **Intervalle entre tâches** pour définir un délai aléatoire entre les cibles, puis cliquez sur `Démarrer le script` pour exécuter la tâche.

## Description de la configuration

### Source de données

- **Noms d'utilisateur / Liens de publications** : Détermine quelles fonctionnalités sont disponibles. Le suivi, le désabonnement et les messages directs ne peuvent être utilisés qu'avec des noms d'utilisateur ; si vous souhaitez uniquement manipuler des publications, vous pouvez directement sélectionner le mode liens.
- **Méthode d'importation** : Vous pouvez cliquer sur le bouton `Importer un fichier txt` pour charger un fichier `.txt` correspondant ou coller directement plusieurs lignes de données ; après l'importation, choisissez "Ajouter" ou "Remplacer".

### Stratégie d'utilisation des données

- **Pool partagé (Shared pool)** : Tous les comptes bot tirent des tâches du même pool de cibles, adapté aux scénarios où plusieurs comptes interagissent plusieurs fois avec les mêmes cibles (par exemple, pour promouvoir collectivement une publication populaire).

- **Consommation unique (Single-consumption)** : Chaque compte bot ne traite que des cibles différentes, garantissant que chaque cible n'est traitée qu'une seule fois par un seul compte, adapté aux scénarios nécessitant d'éviter les contacts répétés (par exemple, messages directs en masse ou commentaires uniques).

  - **Consommation maximale par tâche** : En mode consommation unique, vous pouvez définir le nombre maximal de cibles consommées par exécution de tâche. Cela empêche le premier lot de tâches de consommer toutes les données, garantissant que des cibles restent disponibles lors de la rotation ultérieure des comptes. Définir à 0 signifie aucune limite (comportement par défaut).

Choisir une stratégie affectera la distribution de la file d'attente et le comportement concurrent, veuillez choisir en fonction de vos objectifs et considérations de contrôle des risques.

### Actions utilisateur

- **Suivre l'utilisateur** : Effectue en priorité l'action de suivi après être entré sur la page d'accueil.
- **Ne plus suivre l'utilisateur** : Annule le suivi, peut être exécuté seul ou pour nettoyer les abonnés après l'envoi de messages directs ; non recommandé d'activer en même temps que "Suivre".
- **Signaler le compte** : Signale un compte utilisateur pour comportement frauduleux. Cette fonctionnalité utilise un chemin de signalement fixe, ne prend pas en charge la personnalisation du chemin.

  :::warning Chemin de signalement (fixe)
  Signaler → Signaler le compte → Something else → Frauds and scams
  :::

  - Remarque : Cette fonctionnalité utilise un chemin de signalement prédéfini, ne prend pas en charge les raisons ou chemins de signalement personnalisés.
  - Veuillez utiliser cette fonctionnalité de manière responsable, uniquement à des fins légitimes. Un abus peut entraîner des restrictions de compte.
- **Envoyer un message direct** : Sélectionne et envoie aléatoirement un message parmi plusieurs lignes ; cocher "Ajouter automatiquement des emojis" ajoute automatiquement des emojis naturels. Le script vérifie si l'autre partie autorise les messages directs et ignore si l'envoi est impossible.

  - Modèles de messages directs (pré-générés) : Dans "Paramètres de message", collez plusieurs lignes ou paragraphes de contenu de message direct (chaque ligne/paragraphe est considéré comme un message candidat).

  - Ordre d'utilisation du contenu : Sélectionnez "Aléatoire (Random)" ou "Séquentiel (Sequential)".

    - Aléatoire : Sélectionne et envoie aléatoirement un modèle à chaque fois.

    - Séquentiel : Utilise en rotation selon l'ordre de collage/importation, adapté aux stratégies de déduplication reproductibles.

  - Insérer des emojis : Peut activer l'insertion automatique d'emojis dans les messages ou commentaires pour augmenter le naturel.

  - Génération en temps réel (optionnel, ChatGPT ou API compatible) :

    - Une fois activé, le script peut appeler l'interface de génération que vous avez configurée (remplir API Key / URL / Modèle / System Prompt) pour générer un contenu personnalisé basé sur le contexte de la cible.

    - Attention : Nécessite de fournir une API Key valide, et faites attention aux limites de taux, aux coûts et à la conformité en matière de confidentialité (évitez d'envoyer des données sensibles).

  - Stratégie d'unicité : Méthodes courantes pour garantir l'unicité de chaque message/commentaire :

      1. Utiliser la rotation "Séquentielle" des modèles pré-générés (les modèles ne sont utilisés qu'une seule fois jusqu'à épuisement).
      2. Utiliser la génération en temps réel (ChatGPT) pour générer un texte unique pour chaque cible.
  - Avant d'utiliser les messages directs en masse, assurez-vous de tester sur un petit lot et de respecter les exigences en matière de confidentialité et de conformité.

### Actions sur les publications

- **Nombre de publications à ignorer** : Nombre de publications à ignorer avant le traitement (0–8). Cela vous permet d'ignorer les premières publications du profil d'un utilisateur avant de commencer à interagir avec son contenu. Nécessaire uniquement lorsque la source de données est "Noms d'utilisateur".
- **Nombre de publications à traiter** : Détermine le nombre de publications que chaque compte doit manipuler (1–50), nécessaire uniquement lorsque la source de données est "Noms d'utilisateur".
- **Nombre de répétitions** : Répète le cycle d'interaction avec les publications plusieurs fois, permettant de répéter les opérations sur les publications sans redémarrer le script.
- **Durée de visionnage** : Définit la durée de visionnage minimale et maximale (0–300 secondes). Le script sélectionnera une valeur aléatoire dans cet intervalle avant d'exécuter l'interaction.
- **Interaction avec les publications** : Vous pouvez cocher séparément like, enregistrer, partager et diffuser. Lorsque plusieurs actions sont activées, le système les répartit à différents moments pendant la période de visionnage, ce qui semble plus naturel.

- **Commentaires** : Une fois activé, permet de publier des commentaires sur les publications.

     Si vous avez activé "Commentaires", configurez les éléments suivants dans les paramètres de commentaires :

  - Modèles de commentaires pré-générés : Vous pouvez coller plusieurs lignes ou paragraphes de contenu de commentaires pré-générés (chaque ligne ou paragraphe est considéré comme un commentaire candidat).

  - Ordre d'utilisation du contenu : Sélectionnez l'utilisation "Aléatoire" ou "Séquentielle".
    - Aléatoire : Sélectionne aléatoirement un contenu parmi les candidats à chaque fois.
    - Séquentiel : Utilise dans l'ordre de collage/importation (adapté aux stratégies de déduplication reproductibles).

  - Insérer des emojis : Peut activer l'insertion automatique d'emojis dans les commentaires pour augmenter le naturel.

  - Génération de commentaires en temps réel (optionnel, API ChatGPT) :
    - Une fois activé, le script peut appeler une API de type ChatGPT pour générer en temps réel des commentaires contextuellement pertinents et uniques basés sur le titre/description de la publication. Cette méthode peut réduire considérablement le risque de commentaires répétitifs, adaptée aux scénarios nécessitant un grand nombre de commentaires uniques.
    - Remarques : L'utilisation de la génération en temps réel nécessite de fournir une API Key disponible, et faites attention aux limites de taux, aux coûts et à la conformité en matière de confidentialité (n'envoyez pas de données sensibles à des services tiers).

  - Deux méthodes courantes pour garantir l'unicité des commentaires :
         1. Utiliser le chargement "Séquentiel" et la rotation des modèles de commentaires pré-générés (garantissant que chaque modèle n'est utilisé qu'une seule fois jusqu'à épuisement).
         2. Utiliser l'API ChatGPT pour générer chaque commentaire en temps réel (généré dynamiquement en fonction du titre/contenu de la publication, naturellement plus facile à garantir l'unicité).

  - Recommandation : Testez d'abord la stratégie de commentaires sur un petit lot de publications, vérifiez la qualité des commentaires et le traitement de la plateforme ; si vous utilisez la génération en temps réel, surveillez le volume d'appels API et les coûts.

  - Si vous utilisez la stratégie "Séquentielle", le système combinera le numéro de tâche pour faire tourner les modèles entre plusieurs appareils, réduisant la probabilité de répétition ; si vous utilisez la génération IA, il est recommandé d'effectuer des vérifications de qualité simples sur la sortie (par exemple, plage de longueur, interdiction d'inclure des URL / mots sensibles).

### Fusion de tâches (Fusionner les mêmes noms d'utilisateur)

- **Description** : Lorsque "Fusionner les tâches avec le même nom d'utilisateur" est activé, plusieurs cibles pour le même nom d'utilisateur seront exécutées séquentiellement ou en parallèle (selon l'implémentation du script) dans une même tâche ; même si l'une des cibles échoue, la tâche ne sera pas entièrement interrompue, les cibles restantes continueront à s'exécuter et tenteront de se terminer, réduisant ainsi les coûts de changement et de planification des tâches.

- **Comportement lorsqu'activé** : Plusieurs cibles pour le même compte seront regroupées dans une même unité de tâche, l'échec n'affecte que cette cible elle-même et est enregistré, mais ne bloque pas la poursuite de l'exécution des autres cibles. Adapté pour une utilisation lors de campagnes marketing à grande échelle, peut réduire considérablement la fréquence de démarrage/changement de tâche.

- **Comportement lorsque désactivé** : Chaque cible génère une tâche séparée, tout échec d'une cible individuelle sera immédiatement signalé et arrêtera cette sous-tâche (n'affectera pas les autres tâches indépendantes). Adapté aux scénarios nécessitant une surveillance et une alerte précises et ligne par ligne pour chaque cible.

- **Recommandations d'utilisation** :
  - Recommandé d'activer lors de campagnes marketing en masse ou lorsqu'il faut maintenir un débit élevé pour réduire les coûts de planification.
  - Lorsque vous devez confirmer ligne par ligne l'état d'exécution de chaque cible et prendre des mesures immédiatement en cas d'échec (par exemple, intervention manuelle ou rollback immédiat), veuillez désactiver cette option.

### Intervalle entre tâches

- Définissez la valeur minimale et maximale (unité : minutes), le script attendra aléatoirement entre chaque cible selon l'intervalle, et ajoutera ses propres pauses courtes.

## Flux d'exécution

- Si le mode multi-comptes est activé, TikMatrix/IgMatrix lancera d'abord TikTok/IgMatrix et basculera vers le compte correspondant.
- En mode nom d'utilisateur, le système ouvre chaque page d'accueil selon la méthode d'accès sélectionnée, attend que le chargement soit terminé, puis exécute séquentiellement les actions de suivi, message direct, désabonnement, etc. ; si l'autre partie interdit les messages directs, il ignore automatiquement.
- L'interaction avec les publications parcourt la page d'accueil ou les liens directs, regarde selon le nombre et la durée définis et exécute les actions de like, enregistrer, partager, diffuser, et ignore les publicités, lives et contenu déjà liké.
- Lors de l'utilisation de liens de publications, le script ouvre les liens un par un, exécute les actions configurées et boucle selon le "Nombre de répétitions".
- Le module de commentaires prend les mots des modèles ou de la sortie IA. Le mode séquentiel combinera le numéro de tâche pour distribuer différents commentaires entre plusieurs appareils.
- Entre chaque cible, le script ajoute une petite pause aléatoire, puis ajoute l'attente aléatoire de l'intervalle entre tâches, simulant le rythme humain.

## Meilleures pratiques

- Validez d'abord le contenu et le rythme avec un petit lot de données, puis augmentez l'échelle après confirmation.
- Combiner plusieurs actions (comme suivre + message direct + like) peut améliorer le signal d'interaction du compte.
- Les messages directs et commentaires doivent rester conversationnels et sincères, évitez de déclencher le contrôle des risques de la plateforme.
- Mettez régulièrement à jour la bibliothèque de commentaires ou ajustez le System Prompt de l'IA pour que le contenu corresponde mieux au thème de l'activité.
- Faites attention à la limitation de la plateforme, si le volume est important, vous pouvez faire tourner les comptes ou les nœuds proxy.
