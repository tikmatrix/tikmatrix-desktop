# Booster les publications

:::warning Avis de dépréciation
Cette fonctionnalité de script a été intégrée dans le [script Super Marketing](./14.super-marketing.md). Veuillez utiliser en priorité le script "Super Marketing" pour bénéficier de capacités de boost de publications plus puissantes et flexibles.
:::

La fonctionnalité de boost de publications permet d'augmenter l'engagement des publications via des opérations massives de likes, partages, favoris, retweets et visionnages. Le système simule au maximum le comportement des utilisateurs réels pour réduire le risque de détection, mais il est toujours recommandé de respecter les règles de la plateforme et de contrôler modérément la fréquence.

## Utiliser la fonctionnalité de boost de publications dans le script "Super Marketing"

1. Ouvrez le script : sélectionnez "Super Marketing".
2. Sélectionnez le type de source de données "Liste de liens de publications".
3. Collez en masse les liens de publications ou importez-les depuis un fichier texte ; après l'importation, vous pouvez choisir "Ajouter" ou "Remplacer" la liste actuelle.
4. Sélectionnez la stratégie d'utilisation des données :

    - Stratégie de pool partagé : tous les comptes bot interagissent avec les mêmes publications.

    - Stratégie de consommation unique : chaque compte bot n'interagit qu'une seule fois avec des publications différentes, garantissant que chaque publication n'est traitée qu'une seule fois par un compte.

5. Dans "Opération sur les publications — Nombre de répétitions", définissez le nombre d'interactions pour chaque publication. Si cela inclut des commentaires, il est recommandé de définir le nombre de commentaires à 1 pour éviter les commentaires en double.
6. Dans "Opération sur les publications", sélectionnez les types d'interactions nécessaires : like, partage, retweet, favori, commentaire, visionnage de vidéo, etc.

     Si vous sélectionnez "Commentaire", veuillez effectuer les configurations suivantes dans les paramètres de commentaire :

     - Modèles de commentaires prégénérés : vous pouvez coller plusieurs lignes ou paragraphes de commentaires prégénérés (chaque ligne ou paragraphe est considéré comme un commentaire candidat).

     - Ordre d'utilisation des messages : sélectionnez "Aléatoire" ou "Séquentiel".
         - Aléatoire : tire aléatoirement un message parmi les candidats à chaque fois.
         - Séquentiel : utilise les messages dans l'ordre collé/importé (convient aux stratégies de déduplication reproductibles).

     - Insérer des emojis : vous pouvez activer l'insertion automatique d'emojis dans les commentaires pour améliorer le naturel.

     - Génération de commentaires en temps réel (optionnel, API ChatGPT) :
         - Une fois activé, le script peut appeler une API de type ChatGPT pour générer des commentaires contextuels et uniques en temps réel basés sur le titre/la description de la publication. Cette méthode peut réduire considérablement le risque de commentaires en double et convient aux scénarios nécessitant un grand nombre de commentaires uniques.
         - Remarques : l'utilisation de la génération en temps réel nécessite de fournir une clé API valide, et il faut faire attention aux limites de taux, aux coûts et à la conformité en matière de confidentialité (ne pas envoyer de données sensibles à des services tiers).

     - Deux méthodes courantes pour garantir des commentaires uniques :
         1. Utiliser "Séquentiel" pour charger et alterner les modèles de commentaires prégénérés (garantit que chaque modèle n'est utilisé qu'une seule fois jusqu'à épuisement).
         2. Utiliser l'API ChatGPT pour générer chaque commentaire en temps réel (génération dynamique basée sur le titre/contenu de la publication, naturellement plus facile à garantir l'unicité).

     - Recommandation : testez d'abord la stratégie de commentaires sur un petit lot de publications, vérifiez la qualité des commentaires et le traitement par la plateforme ; si vous utilisez la génération en temps réel, surveillez le volume d'appels API et les coûts.
7. Configurez les autres paramètres du script (comme le nombre de concurrences, les délais, le pool de comptes, etc.), confirmez que tout est correct, puis lancez le script.

## Conseils d'utilisation et précautions

- Il est recommandé de tester d'abord sur un petit nombre de publications pour confirmer que la stratégie et les paramètres ne déclenchent pas les contrôles de la plateforme.
- Le contenu des commentaires doit être diversifié et naturel, évitez de copier-coller le même texte.
- Configurez raisonnablement les délais et la concurrence pour simuler un comportement humain et réduire les risques.
- Pour une exécution à long terme, vérifiez régulièrement la santé des comptes et les journaux de tâches, et ajustez la stratégie en fonction de la situation.
