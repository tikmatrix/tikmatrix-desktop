# Messages privés en masse

:::warning Avis de dépréciation
Cette fonctionnalité a été intégrée dans le [script Super Marketing](./14.super-marketing.md). Veuillez utiliser en priorité le script "Super Marketing" pour bénéficier de capacités de messages privés en masse et de gestion des messages plus puissantes et flexibles.
:::

La fonctionnalité de messages privés en masse permet d'établir un contact avec les utilisateurs cibles en envoyant des messages privés (DM) en masse, adaptée aux scénarios de promotion, de service client et de réactivation d'utilisateurs. Le système prend en charge deux sources de messages : les modèles prégénérés et la génération en temps réel, et simule au maximum l'envoi manuel pour réduire le risque de détection.

## Utiliser la fonctionnalité de messages privés en masse dans le script "Super Marketing"

1. Ouvrez le script : sélectionnez "Super Marketing".
2. Sélectionnez le type de source de données "Liste de comptes utilisateurs (liste cible)".
3. Collez en masse les noms d'utilisateur cibles ou importez-les depuis un fichier texte (un par ligne) ; après l'importation, vous pouvez choisir "Ajouter" ou "Remplacer".
4. Sélectionnez la stratégie d'utilisation des données :

    - Stratégie de pool partagé : tous les comptes bot envoient des messages privés à la même liste cible.

    - Stratégie de consommation unique : chaque compte bot ne traite que des cibles différentes, garantissant que chaque cible ne reçoit un message que d'un seul compte une seule fois.

5. Définissez le nombre d'envois de messages pour chaque cible (généralement 1 fois, pour éviter le harcèlement répété).
6. Dans "Paramètres de message", configurez le contenu et la stratégie d'envoi des messages privés :

    - Modèles de messages prégénérés : vous pouvez coller plusieurs lignes ou paragraphes de messages privés prégénérés (chaque ligne ou paragraphe est considéré comme un message candidat).

    - Ordre d'utilisation des messages : sélectionnez "Aléatoire" ou "Séquentiel".

        - Aléatoire : tire aléatoirement un modèle pour l'envoi à chaque fois.

        - Séquentiel : alterne l'utilisation des modèles dans l'ordre collé/importé (convient aux stratégies de déduplication reproductibles).

    - Insérer des emojis : vous pouvez activer l'insertion automatique d'emojis dans les messages pour améliorer le naturel.

    - Génération de messages en temps réel (optionnel, API ChatGPT) :

        - Une fois activé, le script peut appeler ChatGPT ou une API similaire pour générer des messages personnalisés en temps réel basés sur le nom d'utilisateur cible ou le contexte disponible, adapté aux scénarios nécessitant un grand nombre de messages uniques.

        - Remarque : l'activation de la génération en temps réel nécessite de fournir une clé API valide, et il faut faire attention aux limites de taux, aux coûts et à la conformité en matière de confidentialité (éviter d'envoyer des données sensibles).

    - Deux méthodes courantes pour garantir des messages uniques :

        1. Utiliser "Séquentiel" pour alterner les modèles prégénérés (chaque modèle n'est utilisé qu'une seule fois jusqu'à épuisement).

        2. Utiliser ChatGPT pour générer chaque message en temps réel (génération dynamique basée sur les informations cibles, unicité plus élevée).

7. Configurez la méthode d'ouverture de l'utilisateur (si vous devez consulter le profil utilisateur) : sélectionnez "Rechercher l'utilisateur" ou "Ouvrir directement le profil (Direct Open Profile)".
8. Configurez l'intervalle entre les tâches (min/max, unité : minutes), la concurrence, le pool de comptes et d'autres paramètres, confirmez que tout est correct, puis lancez le script.

:::tip À propos de l'intervalle entre les tâches
Il est recommandé de définir un intervalle min/max et d'activer l'intervalle aléatoire pour éviter que tous les appareils n'exécutent simultanément, créant un pic anormal, réduisant ainsi le risque de détection ou de limitation par la plateforme.
:::

## Conseils d'utilisation et précautions

- Le contenu des messages privés doit être conforme et diversifié, évitez d'envoyer un grand nombre de contenus identiques en peu de temps.
- Si vous utilisez la génération en temps réel, surveillez strictement le volume d'appels API et les coûts, et assurez-vous de ne pas divulguer d'informations sensibles à l'extérieur.
- Les tâches peuvent échouer ; en cas d'échec, veuillez réessayer ou consulter les journaux, le système ignorera automatiquement les cibles déjà complétées.
