# Zwiększ zaangażowanie postów

:::warning Powiadomienie o wycofaniu
Ta funkcjonalność została zintegrowana z [Super Marketing](./14.super-marketing.md). Proszę używać skryptu Super Marketing dla bardziej zaawansowanych i elastycznych funkcji zwiększania zaangażowania postów.
:::

Funkcja Boost Posts zwiększa zaangażowanie poprzez masowe polubienie, udostępnianie, dodawanie do ulubionych, repostowanie i oglądanie postów. System próbuje symulować rzeczywiste zachowanie użytkownika, aby zmniejszyć ryzyko wykrycia, ale nadal powinieneś przestrzegać zasad platformy i umiarkowanego użytkowania.

## Używanie funkcji Boost Posts w Super Marketing

1. Otwórz skrypt: wybierz "Super Marketing".
2. Wybierz typ źródła danych "Post URL list".
3. Wklej adresy URL postów zbiorczo lub zaimportuj z pliku tekstowego; po zaimportowaniu wybierz "Append" lub "Overwrite" istniejącej listy.
4. Wybierz strategię użycia danych:

    - Wspólna pula: wszystkie konta botów wchodzą w interakcję z tymi samymi postami.

    - Pojedyncze zużycie: każde konto bota wchodzi w interakcję z różnymi postami, więc każdy post jest obsługiwany tylko przez jedno konto.

5. Ustaw liczbę powtórzeń na post w "Post Actions — Repeats". Jeśli komentarze są włączone, ustaw powtórzenia komentarzy na 1, aby uniknąć duplikatów komentarzy.
6. W "Post Actions" wybierz typy interakcji, które chcesz: Like, Share, Repost, Favorite, Comment, Watch video, itp.

     Jeśli wybierzesz "Comment", skonfiguruj opcje komentarzy w następujący sposób:

     - Wstępnie wygenerowane szablony komentarzy: wklej wieloliniowe lub wieloakapitowe szablony komentarzy (każda linia/akapit jest traktowana jako kandydat na komentarz).

     - Kolejność użycia szablonu: wybierz **Random** lub **Sequential**.
         - Random: losowo wybierz jeden szablon za każdym razem.
         - Sequential: użyj szablonów w kolejności wklejonej/zaimportowanej (przydatne do deterministycznej deduplikacji).

     - Wstawianie emoji: opcjonalnie włącz automatyczne wstawianie emoji do komentarzy, aby zwiększyć naturalność.

     - Generowanie komentarzy w czasie rzeczywistym (opcjonalne — ChatGPT API):
         - Gdy włączone, skrypt może wywołać API podobne do ChatGPT, aby wygenerować świadome kontekstu, unikalne komentarze na podstawie tytułu/opisu posta. To znacznie zmniejsza powtarzające się komentarze i jest odpowiednie, gdy wymagane są liczne unikalne komentarze.
         - Uwagi: użycie generowania w czasie rzeczywistym wymaga ważnego klucza API i uwagi na limity stawek, koszty i prywatność (unikaj wysyłania wrażliwych danych do usług zewnętrznych).

     - Dwa powszechne sposoby zapewnienia unikalnych komentarzy:
         1. Ładuj i rotuj wstępnie wygenerowane szablony w trybie **Sequential** (każdy szablon jest używany raz do wyczerpania).
         2. Użyj API ChatGPT, aby wygenerować unikalny komentarz na post (dynamiczne generowanie na podstawie treści posta).

     - Rekomendacja: przetestuj swoją strategię komentarzy na małej partii, przejrzyj jakość komentarzy i obsługę platformy; jeśli używasz generowania w czasie rzeczywistym, monitoruj użycie API i koszty.

7. Skonfiguruj inne parametry skryptu (współbieżność, opóźnienia, pula kont), zweryfikuj ustawienia, a następnie uruchom skrypt.

## Rekomendacje i uwagi

- Najpierw przetestuj na małej liczbie postów, aby upewnić się, że ustawienia nie uruchamiają obrony platformy.
- Zachowaj różnorodny i naturalny tekst komentarza; unikaj kopiowania tego samego tekstu.
- Ustaw rozsądne opóźnienia i współbieżność, aby naśladować ludzkie zachowanie i zmniejszyć ryzyko.
- W przypadku długich uruchomień monitoruj kondycję konta i dzienniki zadań oraz w razie potrzeby dostosuj strategie.
