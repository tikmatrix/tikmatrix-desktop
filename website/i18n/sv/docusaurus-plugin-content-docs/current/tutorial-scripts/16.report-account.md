# Rapportera Konto

:::warning Ansvarsfullt Bruk
Att rapportera konton är en kraftfull åtgärd. Använd rapporteringsfunktionen endast för legitima överträdelser (bedrägeri, bluffar, spam, etc.). Missbruk kan resultera i kontobegränsningar eller andra negativa effekter.
:::

Rapportera Konto-guiden förklarar hur man använder den inbyggda rapporteringsfunktionen som exponeras av Super Marketing-skriptet. Funktionen automatiserar det fasta in-app rapporteringsflödet för att hjälpa till att bearbeta en lista med användarkonton, men den stöder inte anpassning av rapporteringsvägen eller rapporteringsorsaker utöver det fasta valet som beskrivs nedan.

## Snabb översikt

- Rapporteringsflödet som används av skripten är fast och kan inte ändras i UI:t eller skripten:

```txt
Rapportera → Rapportera konto → Något annat → Bedrägerier och bluffar
```

- Skriptet försöker navigera användarens profil, öppna rapporteringsmenyn, välja den fasta vägen ovan och skicka rapporten när det är möjligt.
- Eftersom mobilt UI skiljer sig mellan appversioner och enheter använder skriptet best-effort UI-väljare; testa alltid på en liten batch först.

## Steg för att köra (via Super Marketing)

1. Öppna TikMatrix/IgMatrix och välj enheten/enheterna att köra på (flerval stöds).
2. Öppna `Skript` → `Super Marketing`.
3. Ställ in **Datakälla** till `Användarnamn` och importera en textfil eller klistra in användarnamn (ett per rad). Efter import välj `Lägg till` eller `Skriv över`.
4. I **Användaråtgärder**, aktivera **Rapportera konto** (den nya växeln) och verifiera den visade fasta rapporteringsvägen.
5. Konfigurera uppgiftssamtidighet, fördröjningar och andra uppgiftsinställningar efter behov.
6. Klicka på `Starta Skript` för att börja bearbeta listan. Övervaka loggar och enhetsskärmar för framgång/misslyckande-feedback.

## Tips och rekommendationer

- Testa med 1–5 konton först för att bekräfta att rapport-UI:t upptäcks och sekvensen fungerar på din målappversion och enheter.
- Håll konservativ takt och fördröjningar; tung automatiserad rapportering kan utlösa plattformsförsvar eller hastighetsbegränsningar.
- Denna funktion använder en fast in-app-väg. Om app-UI:t ändras (text eller elementpositioner), kan automatiseringen misslyckas tills väljare uppdateras.
- Använd inte rapportering för att trakassera eller rikta in sig på oskyldiga användare. Rapportera endast tydliga överträdelser som är konsekventa med plattformens villkor.
- Granska loggar efter en körning; skriptet kommer att logga misslyckanden såsom saknade menyknappar eller saknade rapporteringsalternativ så att du kan justera tidpunkt eller väljare.

## När man inte ska använda

- Om du behöver välja anpassade rapporteringsorsaker eller skicka detaljerad bevisning per fall — skriptet stöder inte den nivån av anpassning.
- För juridiska eller känsliga ärenden där manuell granskning krävs.

## Felsökning

- "Kunde inte hitta Rapportera-alternativ" / "Kunde inte hitta 'Rapportera konto'-alternativ": UI-texten eller layouten har troligen ändrats. Kör igen med en liten testuppsättning och fånga enhetsskärmdumpar för felsökning.
- Om rapporter skickas in men inte har någon effekt kan plattformen ignorera automatiserade rapporter eller tillämpa modereringsregler. Överväg manuell granskning för kritiska fall.
