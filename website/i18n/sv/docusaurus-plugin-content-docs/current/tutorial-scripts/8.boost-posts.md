# Boosta Inlägg

:::warning Utgångsmeddelande
Denna funktionalitet har integrerats i [Super Marketing](./14.super-marketing.md). Använd Super Marketing-skriptet för mer kraftfulla och flexibla inläggsboostningsfunktioner.
:::

Boosta Inlägg-funktionen ökar engagemang genom att massgilla, dela, favoritmarkera, omposta och titta på inlägg. Systemet försöker simulera verkligt användarbeteende för att minska detektionsrisk, men du bör fortfarande följa plattformsregler och måttlig användning.

## Använda Boosta Inlägg-funktionen i Super Marketing

1. Öppna skriptet: välj "Super Marketing".
2. Välj datakälltyp "Inläggs-URL-lista".
3. Klistra in inläggs-URL:er i bulk eller importera från en textfil; efter import välj "Lägg till" eller "Skriv över" den befintliga listan.
4. Välj en dataanvändningsstrategi:

    - Delad pool: alla bot-konton interagerar med samma inlägg.

    - Engångsförbrukning: varje bot-konto interagerar med olika inlägg så att varje inlägg hanteras av endast ett konto.

5. Ställ in upprepningsantal per inlägg i "Inläggsåtgärder — Upprepningar". Om kommentarer är aktiverade, ställ in kommentarsupprepningar till 1 för att undvika duplicerade kommentarer.
6. I "Inläggsåtgärder" välj de interaktionstyper du vill ha: Gilla, Dela, Omposta, Favoritmarkera, Kommentera, Titta på video, etc.

     Om du väljer "Kommentera", konfigurera kommentarsalternativen enligt följande:

     - Förgenererade kommentarsmallar: klistra in flerradiga eller flerstyckes kommentarsmallar (varje rad/stycke behandlas som en kandidatkommentar).

     - Mallanvändningsordning: välj **Slumpmässig** eller **Sekventiell**.
         - Slumpmässig: välj slumpmässigt en mall varje gång.
         - Sekventiell: använd mallar i inklistrad/importerad ordning (användbart för deterministisk deduplicering).

     - Emoji-infogning: aktivera valfritt automatisk infogning av emoji i kommentarer för att öka naturligheten.

     - Realtidskommentargenerering (valfritt — ChatGPT API):
         - När aktiverat kan skriptet anropa ett ChatGPT-liknande API för att generera kontextmedvetna, unika kommentarer baserade på inläggstiteln/beskrivningen. Detta minskar avsevärt upprepade kommentarer och är lämpligt när många unika kommentarer krävs.
         - Noteringar: att använda realtidsgenerering kräver en giltig API-nyckel och uppmärksamhet på hastighetsbegränsningar, kostnader och integritet (undvik att skicka känslig data till tredjepartstjänster).

     - Två vanliga sätt att säkerställa unika kommentarer:
         1. Ladda och rotera förgenererade mallar i **Sekventiellt** läge (varje mall används en gång tills den är förbrukad).
         2. Använd ChatGPT API för att generera en unik kommentar per inlägg (dynamisk generering baserad på inläggsinnehåll).

     - Rekommendation: testa din kommentarsstrategi på en liten batch, granska kommentarskvalitet och plattformshantering; om du använder realtidsgenerering, övervaka API-användning och kostnad.

7. Konfigurera andra skriptparametrar (samtidighet, fördröjningar, kontopool), verifiera inställningar och starta sedan skriptet.

## Rekommendationer och noteringar

- Testa på ett litet antal inlägg först för att säkerställa att inställningarna inte utlöser plattformsförsvar.
- Håll kommentarstext varierad och naturlig; undvik att kopiera samma text.
- Ställ in rimliga fördröjningar och samtidighet för att efterlikna mänskligt beteende och minska risk.
- För långa körningar, övervaka kontohälsa och uppgiftsloggar och justera strategier vid behov.
