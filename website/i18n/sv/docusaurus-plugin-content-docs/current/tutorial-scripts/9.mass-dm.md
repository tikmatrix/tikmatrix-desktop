# Mass-DM

:::warning
Denna funktion har slagits samman i [Super Marketing-skriptet](./14.super-marketing.md). Föredra Super Marketing för mer kraftfulla och flexibla massmeddelande- och meddelandehanteringsfunktioner.
:::

Mass-DM används för att bygga kontakter genom att skicka direktmeddelanden (DM:er) i bulk—användbart för marknadsföring, kunduppföljning och återengagemang. Systemet stöder både förgenererade mallar och realtidsgenerering, och det syftar till att efterlikna mänsklig tidpunkt för att minska detektionsrisk.

## Använda Mass-DM inuti Super Marketing

1. Öppna Super Marketing-skriptet.
2. Välj datakälltyp: "Användarkontonslista (mållista)".
3. Klistra in målanvändarnamn i bulk eller importera en `.txt`-fil (ett användarnamn per rad). Efter import kan du välja att "Lägg till" eller "Ersätt" den aktuella listan.
4. Välj en dataanvändningsstrategi:

   - Delad pool: alla bot-konton kommer att meddela från samma målpool.

   - Engångsförbrukning: varje bot-konto bearbetar olika mål så att varje mål får ett meddelande från endast ett konto.

5. Ställ in hur många meddelanden som ska skickas per mål (vanligtvis 1 för att undvika upprepade kontakter).
6. Konfigurera meddelandeinställningar:

   - Förgenererade mallar: klistra in flera rader eller stycken av förberedda DM-kopior. Varje rad/stycke behandlas som ett kandidatmeddelande.

   - Meddelandeordning: välj "Slumpmässig" eller "Sekventiell".

   - Slumpmässig: varje sändning väljer en slumpmässig mall.

   - Sekventiell: rotera mallar i inklistrad/importordning (användbart när du behöver en reproducerbar dedupliceringsstrategi).

   - Infoga emojis: aktivera automatisk infogning av emojis för att öka naturligheten.

   - Realtidsmeddelandegenerering (valfritt, ChatGPT API):

   - När aktiverat kan skriptet anropa ChatGPT eller ett liknande API för att generera personliga meddelanden baserade på målanvändarnamnet eller tillgängligt sammanhang—lämpligt när du behöver många unika meddelanden.

   - Notering: att aktivera realtidsgenerering kräver en giltig API-nyckel och du måste överväga hastighetsbegränsningar, kostnader och integritet (undvik att skicka känslig data till tredjepartens API:er).

   - Två vanliga sätt att garantera unika meddelanden:

   1. Använd sekventiell rotation av förgenererade mallar (varje mall används en gång tills den är förbrukad).

   2. Använd ChatGPT realtidsgenerering för att skapa unika meddelanden per mål.

7. Konfigurera hur du öppnar användarprofiler (om uppgiften behöver visa profiler): välj "Sök" eller "Öppna Profil Direkt".
8. Konfigurera uppgiftsintervall (min/max i minuter), samtidighet och kontopoolinställningar och starta sedan skriptet.

:::tip Om uppgiftsintervall
Ställ in ett minimum- och maximumintervall och aktivera slumpmässiga fördröjningar. Förskjutna tidpunkter minskar risken för synkroniserade aktivitetstoppar och sänker detektionsrisk.
:::

## Rekommendationer och noteringar

- Håll DM-innehållet kompatibelt och varierat; undvik att skicka många identiska meddelanden på kort tid.
- Om du använder realtidsgenerering, övervaka API-användning och kostnader noggrant och undvik att skicka känslig information.
- Uppgifter kan misslyckas; kontrollera loggar och försök igen misslyckade rader. Bearbetade mål kommer att hoppas över vid efterföljande körningar.
