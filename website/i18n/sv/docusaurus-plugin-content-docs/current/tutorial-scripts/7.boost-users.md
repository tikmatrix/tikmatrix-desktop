# Boosta Användare

:::warning Utgångsmeddelande
Denna funktionalitet har integrerats i [Super Marketing](./14.super-marketing.md). Använd Super Marketing-skriptet för mer kraftfulla och flexibla användarboostningsfunktioner.
:::

Boosta Användare-funktionen ökar exponering och engagemang genom att massfölja eller avfölja målkonton. Systemet försöker simulera verkligt användarbeteende för att minska detektionsrisk, men du bör fortfarande följa plattformsregler och måttlig användning.

## Använda Boosta Användare-funktionen i Super Marketing

1. Öppna skriptet: välj "Super Marketing".
2. Välj datakälltyp "Användarkontonslista".
3. Klistra in användarnamn i bulk eller importera från en textfil (ett användarnamn per rad); efter import välj "Lägg till" eller "Skriv över".
4. Välj en dataanvändningsstrategi:

    - Delad pool: alla bot-konton interagerar med samma mål.

    - Engångsförbrukning: varje bot-konto hanterar olika mål så att varje mål bearbetas av endast ett konto.

5. Ställ in upprepningsantal per mål i "Användaråtgärder — Upprepningar" (vanligtvis inställd på 1).
6. Välj åtgärdstyp: Följ eller Avfölj. Du kan ytterligare konfigurera alternativ i **Användaråtgärder**:

        - Profilnavigeringsmetod: välj **Sök Användare** eller **Öppna Profil Direkt**.

        - Skicka Direktmeddelande (valfritt): om aktiverat, klistra in förgenererade flerradiga DM-mallar (en mall per rad eller stycke). Konfigurera mallanvändningsordning:

                - **Slumpmässig**: välj slumpmässigt en mall varje gång.
                - **Sekventiell**: använd mallar i inklistrad/importerad ordning (kan fortsätta över uppgiftskörningar).

            Du kan också välja att automatiskt infoga emoji i mallar för att öka naturligheten.

7. Konfigurera andra skriptparametrar (samtidighet, fördröjningar, kontopool), verifiera inställningar och starta sedan skriptet.

## Rekommendationer och noteringar

- Testa på ett litet antal mål först för att säkerställa att inställningarna inte utlöser plattformsförsvar.
- Ställ in rimliga fördröjningar och samtidighet för att efterlikna mänskligt beteende och minska risk.
- För långa körningar, övervaka kontohälsa och uppgiftsloggar och justera strategier vid behov.
- Boosta Användare-operationer är inte garanterade att lyckas 100%; försök igen misslyckade uppgifter eller undersök målkontostatus vid behov.
