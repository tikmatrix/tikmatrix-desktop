# Stöd för klonade appar

TikMatrix och IgMatrix stöder körning av flera klonade appar (såsom GB TikTok, GB Instagram, Instagram Plus eller andra modifierade versioner) på samma enhet, var och en med sitt eget paketnamn. Denna funktion gör det möjligt för dig att hantera flera konton mer effektivt genom att använda olika appinstanser.

:::danger Riskvarning
Användning av klonade appar kan bryta mot TikTok/Instagrams användarvillkor och kan resultera i kontosuspension eller blockering. Vi tillhandahåller endast teknisk support för denna funktion och är **INTE ansvariga** för några kontorelaterade problem, inklusive blockeringar, suspensioner eller restriktioner orsakade av användning av klonade appar. Använd på egen risk.
:::

## Vad är ett paketnamn?

Varje Android-app har en unik identifierare som kallas **paketnamn**. De officiella apparna använder standardpaketnamn:

- TikTok: `com.zhiliaoapp.musically`
- Instagram: `com.instagram.android`

Klonade appar använder olika paketnamn som:

- GB TikTok: `com.zhiliaoapp.musically.gb`
- GB Instagram: `com.instagram.gb.android`
- Instagram Plus: `com.instagram.plus.android`

TikMatrix/IgMatrix kan automatiskt upptäcka och använda rätt paketnamn för varje konto.

## Konfigurera klonade appar

### Steg 1: Installera klonade appar

1. Installera din föredragna klonade app(ar) på din enhet eller CloudPhone.
2. Se till att varje klonad app har ett unikt paketnamn för att undvika konflikter.
3. Logga in på olika konton i varje appinstans.

### Steg 2: Konfigurera paketnamn

Du har två alternativ för att konfigurera paketnamn:

#### Alternativ A: Kontonivåkonfiguration (rekommenderas)

Ställ in ett specifikt paketnamn för varje konto:

1. Öppna **Konton** i vänster navigation.
2. Klicka på ett konto för att redigera det, eller klicka på **Lägg till** för att skapa ett nytt.
3. I fältet **Paketnamn** kan du:
   - **Manuellt ange** den klonade appens paketnamn (t.ex. `com.instagram.gb.android`)
   - **Auto-upptäck** genom att klicka på knappen **Upptäck paket** (kräver enhetsval först)
4. Klicka på **Spara**.

:::tip Auto-upptäck paketnamn
Knappen **Upptäck paket** upptäcker automatiskt den för närvarande öppnade appen på den valda enheten. Detta är det enklaste sättet att få rätt paketnamn:

1. Välj en enhet för kontot
2. Öppna den klonade appen på den enheten
3. Klicka på knappen **Upptäck paket**
4. Paketnamnet fylls automatiskt i

Denna funktion fungerar med alla Android-appar, inklusive TikTok/Instagram-kloner, App Cloner-appar och NomixCloner-appar.
:::

När du kör skript för detta konto kommer TikMatrix/IgMatrix automatiskt att använda det angivna paketnamnet.

#### Alternativ B: Systemomfattande standard

Ställ in ett standardpaketnamn för alla konton som inte har ett specifikt:

1. Öppna **Inställningar** i vänster navigation.
2. Hitta fältet **TikTok-paketnamn** eller **Instagram-paketnamn**.
3. Ange standardpaketnamnet.
4. Klicka på **Spara**.

:::tip Prioritet
Inställningar på kontonivå åsidosätter alltid den systemomfattande standarden. Om ett konto har sitt eget konfigurerade paketnamn kommer det att användas istället för standarden.
:::

## Hur det fungerar

När du kör ett automatiseringsskript:

1. TikMatrix/IgMatrix kontrollerar om målkontot har ett anpassat paketnamn konfigurerat.
2. Om ja, använder det det paketnamnet för alla UI-automatiseringsoperationer.
3. Om nej, återgår det till det systemomfattande standardpaketnamnet.
4. Alla resurs-ID och XPath-frågor formateras automatiskt med rätt paketnamn.

## Bästa praxis

### 1. Använd beskrivande användarnamn

Eftersom du kan ha flera konton i olika klonade appar, använd beskrivande användarnamn för att identifiera vilken app varje konto tillhör:

```text
username_gb      → GB TikTok/Instagram
username_plus    → TikTok/Instagram Plus
username         → Officiell app
```

### 2. Organisera konton efter grupper

Skapa enhetsgrupper baserat på den klonade apptypen:

- **Grupp A**: Alla konton som använder GB-appar
- **Grupp B**: Alla konton som använder Plus-appar
- **Grupp C**: Alla konton som använder officiella appar

### 3. Testa före massoperationer

Innan du kör skript på alla konton:

1. Testa med ett enda konto först
2. Verifiera att rätt app används
3. Bekräfta att alla operationer fungerar som förväntat

### 4. Håll paketnamn konsekventa

När du väl har ställt in ett paketnamn för ett konto, ändra det inte om du inte byter till en annan klonad app. Konsekvens hjälper till att undvika förvirring och fel.

## Felsökning

### Skriptet öppnar fel app

**Problem**: Skriptet öppnar fel app istället för den klonade appen.

**Lösning**:

1. Kontrollera att kontots **Paketnamn**-fält är korrekt inställt
2. Verifiera att den klonade appen är installerad och att paketnamnet stämmer
3. Använd knappen **Upptäck paket** för att automatiskt få rätt paketnamn
4. Starta om enheten om appen nyligen installerades

### Kan inte hitta UI-element

**Problem**: Skriptet rapporterar "element hittades inte"-fel.

**Lösning**:

1. Olika klonade appar kan ha lite olika UI-layouter
2. Kontrollera om den klonade appversionen är kompatibel
3. Försök uppdatera till den senaste versionen av TikMatrix/IgMatrix
4. Kontakta supporten om problemet kvarstår

### Flera konton i samma app

**Problem**: Vill växla mellan flera konton i samma appinstans.

**Lösning**:

- TikTok/Instagram stöder flera inloggade konton naturligt
- Använd funktionen **Byt konto** i TikMatrix/IgMatrix
- Ställ in samma paketnamn för alla konton i den appinstansen
- TikMatrix/IgMatrix hanterar kontobyte automatiskt

### Upptäck paket-knappen fungerar inte

**Problem**: Knappen **Upptäck paket** fungerar inte eller visar ett fel.

**Lösning**:

1. Se till att du har valt en enhet först
2. Säkerställ att appen för närvarande är öppen på enheten
3. Kontrollera att enhetsanslutningen är stabil
4. Försök öppna appen igen och klicka på upptäck

## Hitta paketnamn

Om du vill manuellt kontrollera paketnamn på din enhet:

### Använd ADB-kommando

```bash
# Lista alla TikTok-relaterade paket
adb shell pm list packages | grep musically

# Lista alla Instagram-relaterade paket
adb shell pm list packages | grep instagram
```

### Använd auto-upptäckfunktionen

Det enklaste sättet är att använda den inbyggda knappen **Upptäck paket**:

1. Öppna kontoedigeringsdialogen
2. Välj enheten
3. Öppna den klonade appen på enheten
4. Klicka på **Upptäck paket**
5. Paketnamnet upptäcks och fylls automatiskt i

Denna metod fungerar för alla Android-appar och är den rekommenderade metoden.

## Vanliga frågor

**F: Kan jag använda flera klonade appar på samma enhet?**

S: Ja, så länge varje klonad app har ett unikt paketnamn kan du köra flera appinstanser på samma enhet.

**F: Behöver jag installera om TikMatrix/IgMatrix?**

S: Nej, denna funktion är inbyggd i den senaste versionen. Uppdatera bara till den senaste versionen och konfigurera paketnamn.

**F: Kommer mina befintliga automatiseringsskript fortfarande att fungera?**

S: Ja, alla befintliga skript är fullt kompatibla. Konton utan ett anpassat paketnamn kommer att använda standardpaketnamnet.

**F: Fungerar detta för både TikTok och Instagram?**

S: Ja, både TikMatrix och IgMatrix stöder fullt ut klonappupptäckt och automatisering.

**F: Hur fungerar auto-upptäckfunktionen?**

S: När du klickar på **Upptäck paket** kontrollerar TikMatrix/IgMatrix vilken app som för närvarande är i förgrunden på den valda enheten och hämtar dess paketnamn automatiskt.

**F: Kan jag använda appar från App Cloner eller NomixCloner?**

S: Ja, paketnamn-funktionen fungerar med alla Android-appkloningsverktyg, inklusive App Cloner och NomixCloner. Använd auto-upptäckfunktionen för att enkelt få rätt paketnamn.
