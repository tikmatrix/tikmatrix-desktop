# Super Marketing

:::info
Super MarketingScript把Segui/Non Seguire Più、Messaggio Diretto、帖子Interazioni和Commento集中到同一条Attività队列里，适合在一轮Operazioni中快速触达潜客或唤醒老Follower。
:::

## Operazioni步骤

1. 在 TikMatrix/IgMatrix 中选择需要EseguiScript的Dispositivi（可多选）。
2. Apri `脚本` → `超级营销`。
3. 选择 **数据来源**（用户名或帖子链接），然后可以点击 `导入txt文件` 按钮加载对应的 `.txt` 文件或者直接Incolla多行数据；Importa后Opzionale择“追加”或“覆盖”。
4. 根据需求开Abilita户动作、帖子动作和Commento模块，并参照下方IstruzioniCompletatoConfigurazione。
5. Impostazioni **Attività间隔** 滑块以定义目标之间的Casuale延迟，最后点击 `开始脚本`EseguiAttività。

## ConfigurazioneIstruzioni

### 数据来源

- **用户名 / 帖子链接**：决定哪些FunzionalitàDisponibile。Segui、Non Seguire Più、Messaggio Diretto只能配合用户名Utilizzo；如果只想Operazioni帖子，可以直接选择链接模式。
- **Importa方式**： 可以点击 `导入txt文件` 按钮加载对应的 `.txt` 文件或者直接Incolla多行数据；Importa后Opzionale择“追加”或“覆盖”。

### 数据Utilizzo策略

- **共享池（Shared pool）**：Tutti bot Account从同一目标池中取Attività,适合让多个Account对同一目标产生多volteInterazioni的场景（例如对Popolari帖子集中推动）。

- **单volte消耗（Single-consumption）**：Ogni个 bot Account只处理不同的目标,确保Ogni个目标仅被一个Account处理一volte,适合需要避免重复触达的场景（例如Messaggi Diretti in Massa或一轮唯一Commento）。

  - **Ogni个Attività最大消耗量**：在单volte消耗模式下,可以ImpostazioniOgnivolteAttivitàEsegui最多消耗的目标数量。这样可以防止第一批Attività把Tutti数据都消耗完,确保后续Account轮换时仍有目标Disponibile。Impostazioni为 0 表示不限制（Predefinito行为）。

选择策略会影响队列的分配与并发行为,请根据目标与风控考虑选择。

### 用户动作

- **Segui用户**：在进入Home后优先EseguiSeguiOperazioni。
- **Non Seguire Più用户**：Non Seguire Più，可单独Esegui或在Messaggio Diretto后清理Follower；不Consigliato与“Segui”同时开启。
- **Segnala Account**：Segnala用户Account存在欺诈行为。此FunzionalitàUtilizzoFisso的Segnala路径，不SupportoPersonalizzato路径。

  :::warning Segnala路径（Fisso）
  Segnala → Segnala Account → Something else → Frauds and scams
  :::

  - Istruzioni：此FunzionalitàUtilizzo预定义的Segnala路径，不SupportoPersonalizzatoSegnala原因或路径。
  - 请负责任地Utilizzo此Funzionalità，仅用于合法目的。滥用可能导致Account受限。
- **发送Messaggio Diretto**：在多行消息中Casuale抽取一条发送；勾选“Automatico加表情”可Automatico附加自然的 emoji。Script会检测对方Vuoi允许Messaggio Diretto，无法发送时会Salta。

  - Messaggio Diretto模板（预生成）：在“消息Impostazioni”中Incolla多行或多段Messaggio Diretto文案（Ogni行/Ogni段视为一条候选消息）。

  - 文案Utilizzo顺序：选择“Casuale（Random）”或“按顺序（Sequential）”。

    - Casuale：OgnivolteCasuale抽取一条模板发送。

    - 按顺序：按Incolla/Importa顺序轮换Utilizzo，适合需要可复现的去重策略。

  - 插入表情（emoji）：可开启Automatico在消息或Commento中插入表情以提高自然度。

  - 实时生成（Opzionale，ChatGPT 或兼容 API）：

    - Abilita后，Script可调用你Configurazione的生成接口（填写 API Key / URL / 模型 / System Prompt）基于目标上下文生成个性化文案。

    - Nota：需提供有效 API Key，并Nota速率限制、费用与隐私合规（避免发送敏感数据）。

  - 唯一性策略：保证Ogni条消息/Commento唯一的常见方式：

      1. Utilizzo“按顺序”轮换预生成模板（模板仅Utilizzo一volte直到耗尽）。
      2. Utilizzo实时生成（ChatGPT）为Ogni个目标生成唯一文本。
  - 在UtilizzoMessaggi Diretti in Massa前，请务必测试小In Massa并Nota隐私与合规要求。

### 帖子动作

- **跳过帖子数量**: 在开始处理前要跳过的帖子数量（0–8）。这允许您在开始与用户内容互动之前跳过其主页上的前几个帖子。仅当数据源为"用户名"时才需要。
- **处理帖子数量**：决定Ogni个Account要Operazioni的帖子数量（1–50）,数据源为"用户名"时才需要。
- **重复volte数**：重复Esegui帖子Interazioni循环多volte，无需重新AvviaScript即可对帖子重复EseguiOperazioni。
- **Visualizzazioni时长**：Impostazioni最短和最长Visualizzazioni时间（0–300 secondi）。Script会在区间内抽取Casuale值再EseguiInterazioni。
- **帖子Interazioni**：可分别勾选Mi Piace、Preferiti、Inoltra和Condividi。Abilita多个动作时，系统会把它们分布在Visualizzazioni时段的不同时间点，显得更自然。

- **Commento**：Abilita后可对帖子发表Commento。

     如果Abilita了“Commento”，请在CommentoImpostazioni中进行以下Configurazione：

  - 预生成Commento模板：可Incolla多行或多段预生成Commento文案（Ogni行或Ogni段视为一条候选Commento）。

  - 文案Utilizzo顺序：选择“Casuale”或“按顺序”Utilizzo。
    - Casuale：Ognivolte从候选文案中Casuale抽取一条。
    - 按顺序：按Incolla/Importa的顺序依volteUtilizzo（适合需要可复现的去重策略）。

  - 插入表情（emoji）：可开启Automatico在Commento中插入表情以提高自然度。

  - 实时生成Commento（Opzionale，ChatGPT API）：
    - Abilita后，Script可调用 ChatGPT 类 API，根据帖子Titolo/Descrizione实时生成上下文相关且唯一的Commento文案。此方式可显著降低重复Commento的风险，适合需要大量唯一Commento的场景。
    - Nota事项：Utilizzo实时生成需提供Disponibile的 API Key，并Nota速率限制、费用与隐私合规（不要发送敏感数据到第三方服务）。

  - 保证唯一Commento的两种常见方式：
         1. Utilizzo“按顺序”加载并轮换预生成Commento模版（保证Ogni条模版只被Utilizzo一volte直到耗尽）。
         2. Utilizzo ChatGPT API 实时生成Ogni条Commento（基于帖文Titolo/ContenutoDinamico生成，giorni然更易保证唯一性）。

  - Consigliato：先用小In Massa帖子测试Commento策略，检查Commento质量与平台处理情况；若Utilizzo实时生成，请监控 API 调用量与成本。

  - 若Utilizzo“顺序”策略，系统会结合Attività编号在多Dispositivi间轮换模板，降低重复概率；若Utilizzo AI 生成，Consigliato对输出做简单质量检查规则（比如长度范围、禁止包含 URL / 敏感词）。

### Attività合并（合并相同用户名）

- **Istruzioni**：当开启“合并相同用户名的Attività”时，针对同一用户名的多个目标会在一个Attività内顺序或并行（视Script实现）Esegui；即使其中某个目标Fallito，Attività不会整体中断，剩余目标会RiprendiEsegui并尝试Completato，从而减少Attività切换与调度开销。

- **开启时的行为**：多个针对相同Account的目标会被归入同一Attività单元，Fallito仅影响该目标本身并被记录，但不会阻塞其它目标的RiprendiEsegui。适合大规模Marketing跑量时Utilizzo，可以显著降低AttivitàAvvia/切换频率。

- **Chiudi时的行为**：Ogni个目标会单独产生一个Attività，任何单个目标的Fallito都会立即上报并Ferma该子Attività（不会影响其它独立Attività）。适合需要对单个目标进行精确、逐条监控与告警的场景。

- **UtilizzoConsigliato**：
  - 在大In MassaMarketing或需要维持较高吞吐量时Consigliato开启以降低调度成本。
  - 当你需要逐条ConfermaOgni个目标的Esegui状态并在Fallito时立刻采取行动（例如人工介入或立即回滚）时，请Chiudi此选项。

### Attività间隔

- Impostazioni最小值和最大值（单位：minuti），Script会在Ogni个目标之间按区间CasualeIn Attesa，并叠加自身的短Pausa顿。

## Esegui流程

- 如果指定了开启了多Account模式，TikMatrix/IgMatrix 会先Avvia TikTok/IgMatrix 并切换到对应Account。
- 在用户名模式下，系统按所选访问方式ApriOgni个Home，In Attesa加载Completato后依volteEseguiSegui、Messaggio Diretto、Non Seguire Più等动作；如果对方禁止Messaggio Diretto会AutomaticoSalta。
- 帖子Interazioni会滑动VisualizzazioniHome或直接链接，按照Impostazioni的数量与时长Visualizzazioni并EseguiMi Piace、Preferiti、Inoltra、Condividi，且会SaltaPubblicità、Live和已Mi PiaceContenuto。
- Utilizzo帖子链接时，Script会逐条Apri链接，EseguiConfigurazione好的动作，并根据“重复volte数”循环。
- Commento模块会从模板或 AI 输出中取词。顺序模式会结合Attività编号在多Dispositivi之间分配不同的Commento。
- 在Ogni个目标之间，Script会加入少量Casuale停顿，再叠加Attività间隔的CasualeIn Attesa，模拟真人节奏。

## 最佳实践

- 先用小In Massa数据验证文案与节奏，Conferma无误后再放大规模。
- 组合多种动作（如Segui + Messaggio Diretto + Mi Piace）能PotenziaAccountInterazioni信号。
- Messaggio Diretto与Commento保持口语化且真诚，避免触发平台风控。
- 定期AggiornaCommento库或调整 AI 的 System Prompt，使Contenuto更贴合活动主题。
- 留意平台限流，若量级较大可轮换Account或Proxy节点。
