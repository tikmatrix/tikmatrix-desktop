## 🧭 1. 디바이스 그룹화란 무엇인가(왜 확장이 가능한가)

**디바이스 그룹화**를 사용하면 실제 Android 휴대폰을 용도/위험도/팀별로 서로 다른 **Group**으로 분류할 수 있습니다.  
각 휴대폰은 **최대 8개의 TikTok 계정**을 바인딩할 수 있으며, 각 Group은 독립적으로 다른 스크립트를 실행할 수 있습니다.

- **시나리오**별 분류: 워밍업, 게시, 팔로워 증가/언팔로우, 라이브 지원  
- **위험도**별 분류: 테스트 계정 vs 핵심 수익화 계정  
- **팀**별 분류: 누가 어떤 디바이스를 담당하고, 누가 어떤 작업을 모니터링하는지

> **핵심 개념:** 디바이스 정리 → 자동화 예측 가능 → 더 안전한 확장.

---

## 🧩 2. 작동 원리(개념 모델)

- **디바이스**: USB/Wi-Fi로 연결된 실제 Android 휴대폰  
- **계정 용량**: 디바이스당 **≤ 8개** 계정  
- **Group**: 작업/위험도/지역별로 디바이스를 집계 (예: `WarmUp-A`, `Posting-EU`)  
- **스크립트**: Group별로 실행되며, 매개변수와 스케줄이 서로 영향을 주지 않음

| 계층 | 예시 | 역할 |
|---|---|---|
| 디바이스 | Pixel_12_03 | 하드웨어 ID 및 프록시 바인딩 |
| 계정 | 디바이스당 6–8개 | 생산 단위 |
| Group | `WarmUp-A` | 작업/위험 격리 |
| 스크립트 | 워밍업/게시/팔로우 | 그룹별 자동화 |

---

## ⚙️ 3. 빠른 시작(단계)

1. **디바이스 연결**, TikMatrix에서 온라인 확인  
2. **디바이스에 계정 바인딩** (≤ 8개/대)  
3. **Group 생성** (예: `WarmUp-A`, `Posting-Main`, `Follow-Geo-US`)  
4. **디바이스를 Group에 할당**  
5. **Group에 스크립트 선택**: 워밍업, 게시, 팔로우/언팔로우, DM 등  
6. **매개변수 구성**: 지연, 무작위성, 디바이스별 독립 프록시  
7. **스케줄 설정**: 시간차 시작, 순환 실행

> 권장사항: 먼저 소규모로 지표를 검증한 후, 점진적으로 그룹 내 디바이스 수를 확대하세요.

---

## 🗓️ 4. 확장 가능한 스케줄링 패러다임

- **시간차 시작**: 그룹 간 5–15분 간격  
- **롤링 웨이브**: 워밍업 → 게시 → 스트리밍/상호작용  
- **야간 중작업**: 저트래픽 시간에 게시/정리  
- **지역별 버킷**: 지역 + 프록시 풀별로 Group 분할

| 모드 | 적용 시나리오 | 예시 |
|---|---|---|
| 시간차 | 급증 및 탐지 감소 | 6분마다 10대 시작 |
| 롤링 | 다단계 퍼널 | 워밍업 2h → 게시 1h → 스트리밍 30m |
| 지역별 | IP/콘텐츠 관련성 | `Post-EU`, `Warm-NA`, `Boost-SEA` |

---

## 🧠 5. 모범 사례 및 위험 관리

- **사람처럼 무작위**: 지연/제스처/입력 속도 모두 변동 필요  
- **디바이스별 프록시**: IP 격리; 공유 VPN/로테이션 대형 풀 피하기  
- **동시 실행 한도**: 그룹 내 합리적인 동시 실행 유지  
- **상태 모니터링**: 비정상 CAPTCHA/오류율/연결 끊김 시 즉시 알림  
- **위험 격리**: 테스트 그룹과 메인 그룹 **엄격히 분리**

> **경험 법칙:** 안정적인 디바이스 + 깨끗한 프록시 + 시간차 스케줄 = 최소 위험 관리.

---

## 👥 6. 팀 협업이 더 이상 혼란스럽지 않음

- **책임별 Group 명명**: `WarmUp-Alice`, `Post-Bob`  
- **매개변수 템플릿 공유**: 작업 유형별로 JSON 하나 고정  
- **통합 변경 창**: 약속된 시간에만 스크립트/버전 업그레이드

## 📋 7. 예시 청사진(20대 디바이스 / 120–160 계정)

|---|---:|---:|---|---|
| WarmUp-A | 8 | 6–8 | 워밍업 스크립트 | 09:00–12:00(시간차) |

## ✅ 8. 체크리스트

| 카테고리 | 권장사항 |
|---|---|
| 그룹화 | 작업/위험도/지역/팀별 분할 |
| 계정 | ≤ 8/디바이스; 순환 사용 |
| 프록시 | 디바이스별 주거용 프록시; 평판 모니터링 |
| 스케줄 | 시간차, 롤링 웨이브, 야간 중작업 |
| 보안 | 사람처럼 무작위; 상태 알림; 점진적 진행 |

- 🧠 **사람처럼 자동화**: 무작위 클릭/스와이프/입력  
- 🎛️ **디바이스 수준 격리**: 프록시, 타이밍, 매개변수 모두 독립적  
- 🕒 **안정적인 스케줄링**: 장기간 안정적 실행 지원
