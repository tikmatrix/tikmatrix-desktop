## 我们に谁创造价值

- **中小团队/MCN/品牌方/灰度试验团队**：需要在 5～100 デバイス规模上で稳定実行“每日重复但需要拟人化”の运营动作。
- **增长とコンテンツ运营**：需要高できます控の“一括但非机械”の動作编排，兼顾安全策略と效率。

---

## 核心价值主张（に什么选 TikMatrix/IgMatrix）

1. **できます编排の一括自動化**：以“タスク → 脚本 → 数据源”の模型构建できます复用流水线，覆盖预热、发布、互动、収集の全链路动作。
2. **拟人化と风控策略**：引擎具备ランダム化时序、节奏制御、人机手势シミュレート、异常中断恢复など机制，尽量贴近実際の人間動作特征。
3. **规模化と稳定性**：サポート真机/云机混合，USB/TCP ADB 接続，保障 5→20→50→100 台の线性扩展と稳定スケジューリング。
4. **数据できます观测**：タスク日志、デバイス投屏、アカウント统计と结果数据エクスポート。

## 機能地图（步骤 4 の能力边界）

### 1) タスク编排とスケジューリング

- 多アカウント/多デバイスし发策略，ランダム化実行顺序
- 失败再試行、断点续跑、资源占用管理（素材/アカウント/代理）

### 2) 脚本中心

- **スーパーマーケティング脚本**：済み融合 Boost 用户/投稿、群发ダイレクトメッセージ、一括コメントなど能力
- アカウント预热脚本：日常閲覧、停留、轻互动
- コンテンツ发布脚本：動画/文案/标签/话题管理，定时发布
- 数据収集脚本：抓取ユーザー名信息，构建下一轮ターゲット清单

### 3) 人机と风控

- 触控/滑动/停顿/观看时长のランダム化
- 异常检测と速率限制，避免一次性高频動作

> **边界声明**：TikMatrix/IgMatrix 不提供デバイス、アカウント、代理本身；我们聚焦于 **运营动作の自動化**。

---

## 上で线建议（从 0 到 规模化）

1. **認証期（1–5 台）**：串联デバイス→アカウント→代理→单一脚本の最小闭环
2. **试点期（10–20 台）**：引入スーパーマーケティング脚本 + 数据収集闭环；观察风控阈值
3. **扩展期（20–50 台）**：分组限速、ランダム化策略、多数据源轮换
4. **规模期（50-100 台）**：分批スケジューリング、错峰実行
