# アカウント管理

アカウント管理は、すべてのTikTokまたはInstagramのIDを、それらが実行されるデバイス、必要な認証情報、自動化が依存するデータと同期させます。整理されたアカウントリストにより、各携帯電話に手動で触れることなく、デバイスの割り当て、アカウントの健全性の監視、キャンペーンの準備が簡単になります。

## 手動でアカウントを作成

1. 左ナビゲーションで**アカウント**を開きます。
2. **追加**をクリックして、編集ダイアログを開きます。
3. 基本フィールドを入力します：
   - **メール**: アプリで使用されるアドレス。TikMatrix/IgMatrixは最初にメール + パスワードでログインを試みます。メールが空白の場合、自動的にユーザー名 + パスワードに切り替わります。
   - **パスワード**: アプリで使用される認証情報。IgMatrixは2FAシークレット付きの拡張パスワードも受け入れます（下記参照）。
   - **ユーザー名** — TikTok/Instagramで表示されるハンドル。これは唯一の必須フィールドです。
   - **ログインステータス**を切り替えて、アプリセッションがデバイスで既に認証されているかどうかをマークします（1 = ログイン済み、0 = 未ログイン）。
   - **ステータス**を切り替えて、アカウントをアクティブに保つ（0 = 有効）または無効にする（1 = 無効）。
4. デバイスバッジを選択して、アカウントを携帯電話またはクラウドフォンにバインドします。デバイスが選択されていない場合、アカウントは未割り当てのままです。
5. **保存**をクリックします。リストは自動的に更新されます。

## ログイン

- **メール**と**パスワード**の両方が提供されている場合、TikMatrix/IgMatrixは割り当てられたデバイスでそのペアでログインします。
- **メール**フィールドが空の場合、代わりに**ユーザー名** + **パスワード**の組み合わせを試します。
- ログインスクリプトがどこで実行されるかを知るために、デバイスインデックスが選択されていることを確認してください。
- ログイン自動化は既に認証されたセッションを検出し、自動的にスキップします。

## 2FAサポート（IgMatrixのみ）

IgMatrixは、自動ログイン中に二要素認証を処理できます。

### 2FAパスワード形式

2FAを有効にするには、次の形式でパスワードを保存します：

```text
your_password:your_2fa_secret
```

**例:**

```text
mypassword123:JBSWY3DPEHPK3PXP
```

## アカウントのインポートとエクスポート

- **エクスポート**は、すべてのアカウントをTikMatrix/IgMatrixのダウンロードフォルダの`accounts.xlsx`に書き込みます。これを生きたバックアップとして、またはバルク編集のテンプレートとして使用します。
- **インポート**では、そのExcelシートからアカウントを一括作成または更新できます。ヘッダー形式が不明な場合は、まず**エクスポート**をクリックし、ダウンロードしたファイルを編集してから、再度インポートします。
- インポート中、`id`を含む行は既存のレコードを更新します。`id`のない行は新しいアカウントを作成します。

> IgMatrix 2FAログインの場合、`username`、`pwd`（`password:2fa_secret`形式を使用）、および`device_index`列を入力します。

### Excel列リファレンス

| 列 | 必須 | 説明 |
| --- | --- | --- |
| `id` | いいえ | 内部識別子。既存のアカウントを更新する場合にのみ含めます。 |
| `email` | いいえ | アプリで使用されるログインメール。連絡先と回復に役立ちます。 |
| `pwd` | いいえ | クイックリファレンスまたは自動ログイン用の保存されたパスワード。 |
| `username` | はい | TikTok/Instagramハンドル。ワークスペース内で一意である必要があります。 |
| `device` | いいえ | アカウントにバインドされたデバイスシリアルまたはクラウドフォン識別子。 |
| `device_index` | いいえ | UIに表示されるわかりやすいデバイスラベル。主に参照用。 |
| `logined` | いいえ | セッションフラグ（`1` = ログイン済み、`0` = 未ログイン）。 |
| `status` | いいえ | 可用性フラグ（`0` = 有効、`1` = 無効）。無効なアカウントはスクリプト実行に参加しません。 |
| `nickname` | いいえ | TikTokデータ同期から取得した最新のニックネーム。 |
| `country` | いいえ | TikTokプロファイルから推測された国。 |
| `followers` | いいえ | TikTokから同期されたフォロワー数。 |
| `following` | いいえ | このプロファイルがフォローしているアカウント数。 |
| `hearts` | いいえ | プロファイルの総いいね（ハート）。 |
| `videos` | いいえ | 公開された動画数。 |
| `friends` | いいえ | 利用可能な場合の相互フレンド数。 |

### TikMatrix vs. IgMatrixフィールド

- **TikMatrix**は、**TikTokデータの同期**を実行するか、データクエリボタンを使用した後、統計フィールド（`nickname`、`country`、`followers`、`following`、`hearts`、`videos`、`friends`）を入力します。
- **IgMatrix**は基本列（`email`、`pwd`、`username`、`device`、`logined`、`status`）のみを使用し、分析列を無視します。

## 追加の動作

- **無効**とマークされたアカウントは、すべての自動化スクリプト、スケジュールされたタスク、バルクアクションから除外されます。新しいジョブを起動する前に再度有効にしてください。
- **アカウントマッチ**スクリプトは、既にログインしているアプリセッションのデバイスをスキャンし、アカウントリストに対応するエントリを自動的に作成し、検出されたデバイスにリンクします。これは、手動で準備された携帯電話をオンボードする最速の方法です。
