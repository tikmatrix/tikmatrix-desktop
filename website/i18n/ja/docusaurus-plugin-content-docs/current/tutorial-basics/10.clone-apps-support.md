# クローンアプリサポート

TikMatrix 和 IgMatrix サポート在同じデバイス上で运行複数クローンアプリ（例など GB TikTok、GB Instagram、Instagram Plus または其他修改版本），每个アプリ都有自己独立のパッケージ名。此機能让您能够通じて使用異なるのアプリ实例更高效地管理複数アカウント。

:::danger 风险提示
使用クローンアプリできます能违反 TikTok/Instagram の服务条款，できます能导致アカウント被封禁または限制。我们仅提供此機能の技术サポート，**不负责**因使用クローンアプリ导致の任何アカウント问题，包括封禁、限制または其他处罚。请自行承担使用风险。
:::

## 什么是パッケージ名？

每个 Android アプリ都有一个唯一の标识符，称に **パッケージ名**。官方アプリ使用标准パッケージ名：

- TikTok: `com.zhiliaoapp.musically`
- Instagram: `com.instagram.android`

クローンアプリ使用異なるのパッケージ名，例など：

- GB TikTok: `com.zhiliaoapp.musically.gb`
- GB Instagram: `com.instagram.gb.android`
- Instagram Plus: `com.instagram.plus.android`

TikMatrix/IgMatrix できます以自動检测しに每个アカウント使用正确のパッケージ名。

## 設定クローンアプリ

### 第一步：安装クローンアプリ

1. 在您のデバイスまたは云携帯電話上で安装首选のクローンアプリ。
2. 确保每个クローンアプリ都有唯一のパッケージ名以避免冲突。
3. 在每个アプリ实例中ログイン異なるのアカウント。

### 第二步：配置パッケージ名

您有两种配置パッケージ名の方式：

#### 方式 A：アカウント级别配置（推荐）

に每个アカウント設定特定のパッケージ名：

1. 在左侧导航開く **アカウント**。
2. 点击某个アカウント进行编辑，または点击 **新增** 创建新アカウント。
3. 在 **パッケージ名** 字段中，您できます以：
   - **手動输入** クローンアプリのパッケージ名（例など `com.instagram.gb.android`）
   - **自動检测** 点击 **检测パッケージ名** 按钮（需要先选择デバイス）
4. 点击 **保存**。

:::tip 自動检测パッケージ名
**检测パッケージ名** 按钮できます以自動检测所选デバイス上で当前開くのアプリ。这是获取正确パッケージ名最简单の方式：

1. にアカウント选择一个デバイス
2. 在该デバイス上で開くクローンアプリ
3. 点击 **检测パッケージ名** 按钮
4. パッケージ名将自動填入

此機能适用于任何 Android アプリ，包括 TikTok/Instagram クローンアプリ、App Cloner アプリ和 NomixCloner アプリ。
:::

当您に此アカウント运行脚本时，TikMatrix/IgMatrix 将自動使用指定されたのパッケージ名。

#### 方式 B：系统全局默认值

に所有没有特定パッケージ名のアカウント設定默认パッケージ名：

1. 在左侧导航開く **設定**。
2. 找到 **TikTok パッケージ名** または **Instagram パッケージ名** 字段。
3. 输入默认パッケージ名。
4. 点击 **保存**。

:::tip 优先级
アカウント级别の設定始终会覆盖系统全局默认值。など果アカウント配置自己のパッケージ名，将使用该パッケージ名而不是默认值。
:::

## 工作原理

当您运行任何自動化脚本时：

1. TikMatrix/IgMatrix 检查ターゲットアカウント是否配置カスタムパッケージ名。
2. など果有，则在所有 UI 自動化操作中使用该パッケージ名。
3. など果没有，则回退到系统全局默认パッケージ名。
4. 所有资源 ID 和 XPath 查询都会自動使用正确のパッケージ名进行形式化。

## 最適な实践

### 1. 使用説明性ユーザー名

由于您できます能在異なるのクローンアプリ中有複数アカウント，请使用説明性ユーザー名来识别每个アカウント属于哪个アプリ：

```text
username_gb      → GB TikTok/Instagram
username_plus    → TikTok/Instagram Plus
username         → 官方应用
```

### 2. 按组织アカウント分组

に基づいてクローンアプリ类型创建デバイス组：

- **A 组**：所有使用 GB アプリのアカウント
- **B 组**：所有使用 Plus アプリのアカウント
- **C 组**：所有使用官方アプリのアカウント

### 3. 一括操作前先测试

在对所有アカウント运行脚本之前：

1. 先用单个アカウント进行测试
2. 認証使用の是正确のアプリ
3. 确认所有操作都按预期工作

### 4. 保持パッケージ名一致

一旦にアカウント設定パッケージ名，除非切换到異なるのクローンアプリ，否则不要更改它。保持一致性有助于避免混淆和错误。

## 故障排除

### 脚本開く错误のアプリ

**问题**：脚本開く错误のアプリ而不是クローンアプリ。

**解决方案**：

1. 检查アカウントの **パッケージ名** 字段是否正确設定
2. 認証クローンアプリ済み安装且パッケージ名匹配
3. 使用 **检测パッケージ名** 按钮自動获取正确のパッケージ名
4. など果アプリ是最近安装の，请重启デバイス

### 找不到 UI 元素

**问题**：脚本报告"找不到元素"错误。

**解决方案**：

1. 異なるのクローンアプリできます能有略微異なるの UI 布局
2. 检查クローンアプリ版本是否兼容
3. 尝试更新到最新版本の TikMatrix/IgMatrix
4. など果问题持续存在，请联系技术サポート

### 同じアプリ中の複数アカウント

**问题**：想要在同じアプリ实例中切换複数アカウント。

**解决方案**：

- TikTok/Instagram 原生サポート多アカウントログイン
- 使用 TikMatrix/IgMatrix の **切换アカウント** 機能
- に该アプリ实例中の所有アカウント設定相同のパッケージ名
- TikMatrix/IgMatrix 将自動处理アカウント切换

### 检测パッケージ名按钮不工作

**问题**：**检测パッケージ名** 按钮不工作または显示错误。

**解决方案**：

1. 确保済み先选择デバイス
2. 确保アプリ当前在デバイス上で開く
3. 检查デバイス接続是否稳定
4. 尝试重新開くアプリし点击检测

## 查找パッケージ名

など果您想手動检查デバイス上でのパッケージ名：

### 使用 ADB 命令

```bash
# 列出所有 TikTok 相关的包
adb shell pm list packages | grep musically

# 列出所有 Instagram 相关的包
adb shell pm list packages | grep instagram
```

### 使用自動检测機能

最简单の方法是使用内置の **检测パッケージ名** 按钮：

1. 開くアカウント编辑对话框
2. 选择デバイス
3. 在デバイス上で開くクローンアプリ
4. 点击 **检测パッケージ名**
5. パッケージ名将自動检测し填入

此方法适用于任何 Android アプリ，是推荐の方式。

## 常见问题

**问：我できます以在同じデバイス上で使用複数クローンアプリ吗？**

答：できます以，只要每个クローンアプリ都有唯一のパッケージ名，您就できます以在同じデバイス上で运行複数アプリ实例。

**问：我需要重新安装 TikMatrix/IgMatrix 吗？**

答：不需要，此機能済み内置在最新版本中。只需更新到最新版本し配置パッケージ名即できます。

**问：我现有の自動化脚本还能工作吗？**

答：できます以，所有现有脚本都完全兼容。没有カスタムパッケージ名のアカウント将使用默认パッケージ名。

**问：TikTok 和 Instagram 都サポート吗？**

答：是の，TikMatrix 和 IgMatrix 都完全サポートクローンアプリ检测和自動化。

**问：自動检测機能是など何工作の？**

答：当您点击 **检测パッケージ名** 时，TikMatrix/IgMatrix 会检查所选デバイス上で当前前台のアプリ，し自動获取其パッケージ名。

**问：我できます以使用 App Cloner または NomixCloner 创建のアプリ吗？**

答：できます以，パッケージ名機能适用于任何 Android アプリ克隆工具，包括 App Cloner 和 NomixCloner。使用自動检测機能できます以轻松获取正确のパッケージ名。
