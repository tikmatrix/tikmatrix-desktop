# 簡単なプロキシローテーションガイド

プロキシローテーションは、TikMatrixが電話が使用するIPアドレスを自動的に変更するのに役立ちます。ログイン、アカウント切り替え、または機密性の高いタスクを実行する前にクリーンなネットワークが必要な場合に使用してください。

## 必要なもの

- リンクをクリックするか、簡単なURLを呼び出すことで「IPをローテーション」または「IPを変更」をサポートするプロキシサービス。
- プロキシローテーションリンク（*ローテーションURL*、*リフレッシュURL*、または*IP変更URL*とも呼ばれる）。
- オプション：プロキシプロバイダーが必要とするパスワードまたはトークン。

## デバイスのローテーションリンクを保存する

1. TikMatrixデスクトップアプリで**デバイス管理**を開き、テーブルビューに切り替えます。
2. コントロールしたいデバイスを見つけて、*プロキシローテーションURL*列の**設定**をクリックします。
3. フォームに入力します：
   - **プロキシローテーションURL**：プロキシプロバイダーからのリンクを貼り付けます。TikMatrixは、新しいIPが必要になるたびにこのリンクを開きます。
   - **リクエストメソッド**：プロバイダーが明示的に`POST`を使用するように指示しない限り、`GET`のままにしておいてください。
   - **ヘッダー**と**ボディ**：プロバイダーが使用する短い例を提供しない限り、空のままにしておいてください。必要な場合は、表示されているとおりにボックスに例をコピーしてください。
   - **タイムアウト**：TikMatrixがプロキシサービスの応答を待つ時間。デフォルトの10秒はほとんどのサービスで機能します。
4. **保存**をクリックします。TikMatrixはすべてを自動的に保存します。ファイルを編集する必要はありません。

保存後、テーブルにはデバイスの横に最後のローテーション結果が表示されます。緑色のラベルは最後のローテーションが成功したことを意味し、灰色のラベルはまだ実行されていないことを意味し、赤色のラベルはプロキシサービスがエラーを返したことを意味します。

## リンクが機能することを確認する

1. 同じテーブル行に留まり、**プロキシローテーションをテスト**をクリックします。
2. TikMatrixはすぐにリンクを呼び出します。
   - 機能する場合、成功メッセージが表示され、「最終ローテーション」時刻が更新されます。
   - 失敗した場合は、メッセージを記録し、リンクとパスワードを再確認してから、もう一度試してください。

ヒント：多くのプロキシプロバイダーは、ローテーションの頻度を制限しています。テストが失敗した場合は、数秒待ってから再試行してください。

## スクリプト中にプロキシローテーションを使用する

1. スクリプトを実行するとき、最初に**プロキシローテーション**をチェックします。TikMatrixは、TikTokを開く前に各デバイスのIPを一つずつ変更します。
2. 各デバイスの右側にあるログボタンをクリックして、各ローテーションに関する詳細情報を表示できます。

## よくある質問

- **すべてのスクリプトでローテーションする必要がありますか？** いいえ。TikTokに新しいIPを見せたいときに使用してください。たとえば、多くのアカウント間で切り替える前などです。
- **設定はどこに保存されますか？** TikMatrixがストレージを処理します。デバイスごとに1回だけフォームに入力する必要があります。
- **プロバイダーリンクが複雑に見えます。それでも動作しますか？** はい。リンクがプロバイダーから提供され、ブラウザで動作する限り、TikMatrixも開くことができます。
- **それでも失敗した場合、誰が助けてくれますか？** プロバイダーの指示とエラーメッセージをTikMatrixサポートに送信して、一緒に確認できます。
