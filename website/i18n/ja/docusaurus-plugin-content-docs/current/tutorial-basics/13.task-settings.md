# タスク設定

## タスクスケジューリングルール

- スケジュールされた開始時刻の10秒以内にタスクが実行されます。
- 各デバイスは一度に1つのタスクのみを実行します。追加のタスクはデバイスごとに自動的にキューに入れられます。
- 最大同時タスク数はライセンス層によって制限されます。無料ユーザーは自動化タスクを実行せず、「待機中」状態のままになります。
- 各リトライの前に、システムは5秒から60秒のランダム化された遅延を適用します。

このダイアログでは、タスクのリトライ方法とデバイスで使用される実行順序を制御できます。以下は、オプションとその使用方法です。

## 自動リトライ

機能：

- 失敗したタスクを、成功するかリトライ制限に達するまで自動的にリトライします。

使用方法：

- 失敗が一時的なもの（ネットワークのヒッチ、一時的なアプリの問題）である場合に有効にします。
- グローバルに有効にする前に、小さなタスクセットでテストします。

## 最大リトライ回数

機能：

- 失敗したタスクの自動リトライ試行の最大回数を設定します。

使用方法：

- 長いリトライループを避けるために控えめな値（2〜3）を選択するか、一時的な障害が頻繁な場合は増やします。
- UIは適切な境界を強制します（例：1〜10）。

## ランダム順序

機能：

- デバイスごとにランダム化されたシーケンスでタスクを実行し、同期された動作を減らします。

使用方法：

- 大規模なデバイスプールの場合、または予測しにくいタスクパターンが必要な場合に有効にします。

## 変更の適用

- 変更は即座に適用されます。ダイアログには成功またはエラー通知が表示されます。

## 見つける場所

- デバイスまたはタスクページを開き、タスク設定（歯車）アイコンをクリックします。

## クイックヒント

- `最大リトライ回数` = 2〜3から開始し、観察された失敗に基づいて調整します。
- 大規模な実行をより自然に見せるために`ランダム順序`を使用します。
- 広範な使用の前に、小さなサンプルで`自動リトライ`を確認します。
