# スーパーマーケティング

:::info
スーパーマーケティングスクリプトは、フォロー/アンフォロー、DM、投稿インタラクション、コメントを単一のタスクキューに組み合わせ、潜在的な顧客に迅速にリーチしたり、フォロワーを1つの操作で再エンゲージするのに最適です。
:::

## 実行手順

1. TikMatrix/IgMatrixで、スクリプトを実行するデバイスを選択します（複数選択サポート）。
2. `スクリプト` → `スーパーマーケティング`を開きます。
3. **データソース**（ユーザー名または投稿リンク）を選択し、`txtファイルをインポート`をクリックして`.txt`ファイルをロードするか、複数行を直接ペーストします。インポート後、`追加`または`上書き`を選択します。
4. 必要に応じてユーザーアクション、投稿アクション、コメントモジュールを有効にし、以下の指示に従って構成します。
5. **タスク間隔**スライダーを設定してターゲット間のランダム遅延を定義し、`スクリプトを開始`をクリックしてタスクを実行します。

## 設定

### データソース

- **ユーザー名 / 投稿リンク**: 利用可能な機能を決定します。フォロー、アンフォロー、DMはユーザー名でのみ使用できます。投稿のみとやり取りしたい場合は、リンクモードを選択します。
- **インポート方法**: `txtファイルをインポート`をクリックして`.txt`ファイルをロードするか、複数行を直接ペーストします。インポート後、`追加`または`上書き`を選択します。

### データ使用戦略

- **共有プール**: すべてのボットアカウントが同じターゲットプールからタスクを引き出します。複数のアカウントが同じターゲットと複数回やり取りする必要があるシナリオに適しています（例：人気投稿への集中プッシュ）。

- **単一消費**: 各ボットアカウントは異なるターゲットのみを処理し、各ターゲットが1つのアカウントによって1回だけ処理されることを保証します。重複リーチを避ける必要があるシナリオに適しています（例：大量DMまたは単一のユニークコメント）。

  - **タスクごとの消費制限**: 単一消費モードを使用する場合、各タスク実行が消費するターゲットの最大数を設定できます。これにより、最初のバッチのタスクがすべての利用可能なデータを消費するのを防ぎ、後続のアカウントローテーションが処理するターゲットを確保します。無制限の消費には0を設定します（デフォルトの動作）。

目標とリスク管理の考慮事項に基づいて戦略を選択してください。これはキュー配分と並行動作に影響します。

### ユーザーアクション

- **ユーザーをフォロー**: プロフィールページに入るときに最初にフォローアクションを実行します。
- **ユーザーをアンフォロー**: ユーザーをアンフォローします。単独で実行するか、DM後にフォロワーをクリーンアップできます。「フォロー」と同時に有効にすることは推奨されません。
- **アカウントを報告**: 詐欺行為でユーザーアカウントを報告します。この機能は固定の報告パスを使用し、カスタムパスはサポートしていません。

  :::warning 報告パス（固定）
  報告 → アカウントを報告 → その他 → 詐欺と詐欺
  :::

  - 注意: この機能は事前定義された報告パスを使用します。カスタム報告理由やパスはサポートされていません。
  - 責任を持って使用し、正当な目的のためにのみ使用してください。誤用するとアカウント制限が発生する可能性があります。

- **DMを送信**: 複数行テンプレートからランダムにメッセージを選択します。「絵文字を挿入」を有効にすると、自動的に自然な絵文字が追加されます。スクリプトは受信者がDMを許可しているかどうかを検出し、利用できない場合はスキップします。

  - DMテンプレート（事前生成）: 「メッセージ設定」で、複数行または複数段落のDMテンプレートをペーストします（各行/段落が候補メッセージです）。

  - テンプレート使用順序: 「ランダム」または「順次」を選択します。

    - ランダム: 毎回ランダムにテンプレートを選択します。

    - 順次: ペースト/インポートされた順序でテンプレートをローテーションします。再現可能な重複排除戦略に適しています。

  - 絵文字を挿入: 有効にすると、メッセージやコメントに絵文字を自動的に挿入して自然さを増します。

  - リアルタイム生成（オプション、ChatGPTまたは互換API）:

    - 有効にすると、スクリプトは構成された生成エンドポイント（APIキー / URL / モデル / システムプロンプトを入力）を呼び出して、ターゲットコンテキストに基づいてパーソナライズされたテキストを生成できます。

    - 注意: 有効なAPIキーを提供し、レート制限、コスト、プライバシーコンプライアンスに注意してください（機密データを送信しないでください）。

  - 一意性戦略: 各メッセージ/コメントが一意であることを保証する一般的な方法:

      1. 「順次」モードを使用して事前生成されたテンプレートをローテーションします（テンプレートは使い果たされるまで一度だけ使用されます）。
      2. リアルタイム生成（ChatGPT）を使用して、各ターゲットに対して一意のテキストを生成します。

  - 大量DMを使用する前に、必ず小さいバッチでテストし、プライバシーとコンプライアンス要件に注意してください。

### 投稿アクション

- **スキップする投稿数**: 処理を開始する前にスキップする投稿数（0～8）。これにより、コンテンツとのやり取りを開始する前に、ユーザーのプロフィールの最初の数投稿をスキップできます。データソースが「ユーザー名」の場合にのみ必要です。
- **処理する最大投稿数**: 各アカウントが処理する投稿数を決定します（1～50）、データソースが「ユーザー名」の場合にのみ必要です。
- **繰り返し回数**: スクリプトを再起動せずに投稿インタラクションループを複数回繰り返します。
- **視聴時間**: 最小および最大視聴時間を設定します（0～300秒）。スクリプトはインタラクションを実行する前に範囲内でランダムに値を選択します。
- **投稿インタラクション**: いいね、お気に入り、リポスト、シェアを個別に有効にできます。複数のアクションが有効になっている場合、システムはより自然な動作のために視聴期間中の異なる時点でそれらを配分します。

- **コメント**: 有効にすると、投稿にコメントを投稿できます。

     「コメント」が有効になっている場合、コメント設定で以下を構成します:

  - 事前生成されたコメントテンプレート: 複数行または複数段落の事前生成されたコメントテンプレートをペーストします（各行または段落が候補コメントです）。

  - テンプレート使用順序: 「ランダム」または「順次」を選択します。
    - ランダム: 毎回候補テンプレートからランダムに選択します。
    - 順次: ペースト/インポートされた順序でテンプレートを使用します（再現可能な重複排除戦略に適しています）。

  - 絵文字を挿入: 有効にすると、コメントに絵文字を自動的に挿入して自然さを増します。

  - リアルタイムコメント生成（オプション、ChatGPT API）:
    - 有効にすると、スクリプトはChatGPTのようなAPIを呼び出して、投稿のタイトル/説明に基づいてコンテキストに関連した一意のコメントテキストをリアルタイムで生成できます。この方法は重複コメントのリスクを大幅に減らし、大量の一意のコメントが必要なシナリオに適しています。
    - 注意: リアルタイム生成を使用するには有効なAPIキーが必要であり、レート制限、コスト、プライバシーコンプライアンスに注意してください（機密データをサードパーティサービスに送信しないでください）。

  - 一意のコメントを保証する2つの一般的な方法:
         1. 「順次」を使用して事前生成されたコメントテンプレートをロードしてローテーションします（各テンプレートが使い果たされるまで一度だけ使用されることを保証します）。
         2. ChatGPT APIを使用して各コメントをリアルタイムで生成します（投稿のタイトル/コンテンツに基づいて動的に生成され、自然に一意性を保証しやすくなります）。

  - 提案: まず小さいバッチでコメント戦略をテストし、コメントの品質とプラットフォームの処理を確認します。リアルタイム生成を使用する場合は、API呼び出しのボリュームとコストを監視してください。

  - 「順次」戦略を使用する場合、システムはタスク番号を組み合わせて複数のデバイス間でテンプレートをローテーションし、重複の可能性を減らします。AI生成を使用する場合は、簡単な品質チェックルール（長さ範囲、URL /機密語の禁止など）を適用することをお勧めします。

### タスクをマージ（同じユーザー名のタスクをマージ）

- 説明: 「同じユーザー名のタスクをマージ」が有効になっている場合、同じユーザー名の複数のターゲットは単一のタスクユニット内で実行されます（実装に応じて順次または並列）。1つのターゲットが失敗しても、全体のタスクは中止されず、残りのターゲットは実行を続け、タスクの切り替えとスケジューリングのオーバーヘッドを削減します。

- 有効時の動作: 同じアカウントを対象とする複数のターゲットが単一のタスクユニットにグループ化されます。失敗は個々のターゲットにのみ影響し、記録されますが、他のターゲットの続行をブロックしません。このモードは大規模なマーケティング実行に適しており、タスクの開始/切り替えのオーバーヘッドを大幅に削減できます。

- 無効時の動作: 各ターゲットは個別のタスクとして作成されます。失敗はすぐに報告され、そのサブタスクを停止します（他の独立したタスクには影響しません）。このモードは、ターゲットごとの正確な監視とアラートが必要な場合に適しています。

- 推奨事項:
  - 高スループット、大バッチマーケティングの場合は、スケジューリングコストを削減するためにこのモードを有効にします。
  - ターゲットごとの正確な実行ステータスが必要で、失敗時に即座にアクション（例：人間の介入や即座のロールバック）を取りたい場合は無効にします。

### タスク間隔

- 最小値と最大値を設定します（分単位）。スクリプトは各ターゲット間で範囲内でランダムに待機し、さらに独自の短い一時停止を追加します。

## 実行フロー

- マルチアカウントモードが有効になっている場合、TikMatrix/IgMatrixは最初にTikTok/Instagramを起動し、対応するアカウントに切り替えます。
- ユーザー名モードでは、システムは選択されたアクセス方法を使用して各プロフィールページを開き、ロードが完了するのを待ってから、フォロー、DM、アンフォローアクションを順次実行します。受信者がDMをブロックしている場合、自動的にスキップします。
- 投稿インタラクションは、プロフィールページをスクロールするか、リンクを直接開き、設定された数量と期間に従って視聴し、いいね、お気に入り、リポスト、シェアアクションを実行し、広告、ライブストリーム、既にいいねしたコンテンツをスキップします。
- 投稿リンクを使用する場合、スクリプトはリンクを1つずつ開き、構成されたアクションを実行し、「繰り返し回数」に従ってループします。
- コメントモジュールはテンプレートまたはAI出力から引き出します。順次モードでは、タスク番号を組み合わせて複数のデバイス間で異なるコメントを配分します。
- 各ターゲット間で、スクリプトは小さなランダムな一時停止を追加し、次にタスク間隔からのランダム待機を追加して人間のリズムをシミュレートします。

## ベストプラクティス

- まず小さいバッチでテキストとペースを検証し、問題がないことを確認してからスケールアップします。
- 複数のアクション（フォロー + DM + いいねなど）を組み合わせると、アカウントのエンゲージメントシグナルを向上させることができます。
- DMとコメントは会話的で誠実に保ち、プラットフォームのモデレーションをトリガーしないようにします。
- コメントライブラリを定期的に更新するか、AIシステムプロンプトを調整して、コンテンツをキャンペーンテーマにより関連させます。
- プラットフォームのレート制限を監視します。大量の場合は、アカウントまたはプロキシノードをローテーションします。
