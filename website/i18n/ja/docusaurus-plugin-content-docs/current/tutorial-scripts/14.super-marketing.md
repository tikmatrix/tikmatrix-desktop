# スーパーマーケティング

:::info
スーパーマーケティング脚本把フォロー/取关、ダイレクトメッセージ、投稿互动和コメント集中到同じ条タスク队列里，适合在一轮操作中快速触达潜客または唤醒老フォロワー。
:::

## 操作步骤

1. 在 TikMatrix/IgMatrix 中选择需要実行脚本のデバイス（できます多选）。
2. 開く `脚本` → `スーパーマーケティング`。
3. 选择 **数据来源**（ユーザー名または投稿リンク），然后できます以点击 `インポートtxt文件` 按钮加载对应の `.txt` 文件または者直接粘贴多行数据；インポート后できます选择“追加”または“覆盖”。
4. に基づいて需求开启用户动作、投稿动作和コメント模块，し参照下方说明完成配置。
5. 設定 **タスク間隔** 滑块以定义ターゲット之间のランダム延迟，最后点击 `开始脚本`実行タスク。

## 配置说明

### 数据来源

- **ユーザー名 / 投稿リンク**：决定哪些機能できます用。フォロー、取关、ダイレクトメッセージ只能配合ユーザー名使用；など果只想操作投稿，できます以直接选择リンク模式。
- **インポート方式**： できます以点击 `インポートtxt文件` 按钮加载对应の `.txt` 文件または者直接粘贴多行数据；インポート后できます选择“追加”または“覆盖”。

### 数据使用策略

- **共享池（Shared pool）**：所有 bot アカウント从同じターゲット池中取タスク,适合让複数アカウント对同じターゲット产生多次互动の场景（例など对热门投稿集中推动）。

- **单次消耗（Single-consumption）**：每个 bot アカウント只处理異なるのターゲット,确保每个ターゲット仅被一个アカウント处理一次,适合需要避免重复触达の场景（例など一括ダイレクトメッセージまたは一轮唯一コメント）。

  - **每个タスク最大消耗量**：在单次消耗模式下,できます以設定每次タスク运行最多消耗のターゲット数量。这样できます以防止第一批タスク把所有数据都消耗完,确保后续アカウント轮换时仍有ターゲットできます用。設定に 0 表示不限制（默认動作）。

选择策略会影响队列の分配とし发動作,请に基づいてターゲットと风控考虑选择。

### 用户动作

- **フォロー用户**：在进入ホームページ后优先実行フォロー操作。
- **取关用户**：取消フォロー，できます单独运行または在ダイレクトメッセージ后清理フォロワー；不建议と“フォロー”同時に开启。
- **举报アカウント**：举报用户アカウント存在欺诈動作。此機能使用固定の举报パス，不サポートカスタムパス。

  :::warning 举报パス（固定）
  举报 → 举报アカウント → Something else → Frauds and scams
  :::

  - 说明：此機能使用预定义の举报パス，不サポートカスタム举报原因またはパス。
  - 请负责任地使用此機能，仅用于合法目の。滥用できます能导致アカウント受限。
- **发送ダイレクトメッセージ**：在多行消息中ランダム抽取一条发送；勾选“自動加表情”できます自動附加自然の emoji。脚本会检测对方是否允许ダイレクトメッセージ，无法发送时会跳过。

  - ダイレクトメッセージ模板（预生成）：在“消息設定”中粘贴多行または多段ダイレクトメッセージ文案（每行/每段视に一条候选消息）。

  - 文案使用顺序：选择“ランダム（Random）”または“按顺序（Sequential）”。

    - ランダム：每次ランダム抽取一条模板发送。

    - 按顺序：按粘贴/インポート顺序轮换使用，适合需要できます复现の去重策略。

  - 插入表情（emoji）：できます开启自動在消息またはコメント中插入表情以提高自然度。

  - 实时生成（できます选，ChatGPT または兼容 API）：

    - 启用后，脚本できます调用你配置の生成接口（填写 API Key / URL / 模型 / System Prompt）基于ターゲット上で下文生成パーソナライズ文案。

    - 注意：需提供有效 API Key，し注意速率限制、费用と隐私合规（避免发送敏感数据）。

  - 唯一性策略：保证每条消息/コメント唯一の常见方式：

      1. 使用“按顺序”轮换预生成模板（模板仅使用一次直到耗尽）。
      2. 使用实时生成（ChatGPT）に每个ターゲット生成唯一文本。
  - 在使用一括ダイレクトメッセージ前，请务必测试小一括し注意隐私と合规必要。

### 投稿动作

- **处理投稿数量**：决定每个アカウント要操作の投稿数量（1–50）,数据源に"ユーザー名"时才需要。
- **重复回数**：重复実行投稿互动循环多次，无需重新启动脚本即できます对投稿重复実行操作。
- **观看时长**：設定最短和最长观看时间（0–300 秒）。脚本会在区间内抽取ランダム值再実行互动。
- **投稿互动**：できます分别勾选いいね、お気に入り、转发和シェア。启用複数动作时，系统会把它们分布在观看时段の異なる時に间点，显得更自然。

- **コメント**：启用后できます对投稿发表コメント。

     など果启用“コメント”，请在コメント設定中进行以下配置：

  - 预生成コメント模板：できます粘贴多行または多段预生成コメント文案（每行または每段视に一条候选コメント）。

  - 文案使用顺序：选择“ランダム”または“按顺序”使用。
    - ランダム：每次从候选文案中ランダム抽取一条。
    - 按顺序：按粘贴/インポートの顺序依次使用（适合需要できます复现の去重策略）。

  - 插入表情（emoji）：できます开启自動在コメント中插入表情以提高自然度。

  - 实时生成コメント（できます选，ChatGPT API）：
    - 启用后，脚本できます调用 ChatGPT 类 API，に基づいて投稿タイトル/説明实时生成上で下文相关且唯一のコメント文案。此方式できます显著降低重复コメントの风险，适合需要大量唯一コメントの场景。
    - 注意事项：使用实时生成需提供できます用の API Key，し注意速率限制、费用と隐私合规（不要发送敏感数据到第三方服务）。

  - 保证唯一コメントの两种常见方式：
         1. 使用“按顺序”加载し轮换预生成コメント模版（保证每条模版只被使用一次直到耗尽）。
         2. 使用 ChatGPT API 实时生成每条コメント（基于帖文タイトル/コンテンツ动态生成，天然更易保证唯一性）。

  - 建议：先用小一括投稿测试コメント策略，检查コメント质量と平台处理情况；若使用实时生成，请监控 API 调用量と成本。

  - 若使用“顺序”策略，系统会结合タスク编号在多デバイス间轮换模板，降低重复概率；若使用 AI 生成，建议对输出做简单质量检查ルール（比など长度范围、禁止含む URL / 敏感词）。

### タスク合し（合し相同ユーザー名）

- **说明**：当开启“合し相同ユーザー名のタスク”时，针对同じユーザー名の複数ターゲット会在一个タスク内顺序または並行（视脚本实现）実行；即使其中某个ターゲット失败，タスク不会整体中断，剩余ターゲット会继续実行し尝试完成，从而减少タスク切换とスケジューリング开销。

- **开启时の動作**：複数针对相同アカウントのターゲット会被归入同じタスク单元，失败仅影响该ターゲット本身し被记录，但不会阻塞其它ターゲットの继续実行。适合大规模营销跑量时使用，できます以显著降低タスク启动/切换频率。

- **关闭时の動作**：每个ターゲット会单独产生一个タスク，任何单个ターゲットの失败都会立即上で报し停止该子タスク（不会影响其它独立タスク）。适合需要对单个ターゲット进行精确、逐条监控と告警の场景。

- **使用建议**：
  - 在大一括营销または需要维持较高吞吐量时建议开启以降低スケジューリング成本。
  - 当你需要逐条确认每个ターゲットの実行状态し在失败时立刻采取行动（例など手動介入または立即回滚）时，请关闭此选项。

### タスク間隔

- 設定最小值和最大值（单位：分），脚本会在每个ターゲット之间按区间ランダムなど待，し叠加自身の短暂停顿。

## 実行流程

- など果指定された开启多アカウント模式，TikMatrix/IgMatrix 会先启动 TikTok/IgMatrix し切换到对应アカウント。
- 在ユーザー名模式下，系统按所选訪問方式開く每个ホームページ，など待加载完成后依次実行フォロー、ダイレクトメッセージ、取关など动作；など果对方禁止ダイレクトメッセージ会自動跳过。
- 投稿互动会滑动閲覧ホームページまたは直接リンク，按照设定の数量と时长观看し実行いいね、お気に入り、转发、シェア，且会跳过广告、ライブ配信和済みいいねコンテンツ。
- 使用投稿リンク时，脚本会逐条開くリンク，実行配置好の动作，しに基づいて“重复回数”循环。
- コメント模块会从模板または AI 输出中取词。顺序模式会结合タスク编号在多デバイス之间分配異なるのコメント。
- 在每个ターゲット之间，脚本会加入少量ランダム停顿，再叠加タスク間隔のランダムなど待，シミュレート実際の人間节奏。

## 最適な实践

- 先用小一括数据認証文案と节奏，确认无误后再放大规模。
- 组合多种动作（などフォロー + ダイレクトメッセージ + いいね）能提升アカウント互动信号。
- ダイレクトメッセージとコメント保持口语化且真诚，避免触发平台风控。
- 定期更新コメント库または调整 AI の System Prompt，使コンテンツ更贴合活动主题。
- 留意平台限流，若量级较大できます轮换アカウントまたは代理节点。
