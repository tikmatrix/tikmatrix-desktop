# 发布投稿

发布投稿脚本用于在TikTok上で发布動画または画像。

## 步骤

1. **准备素材**：
   - 选项1：点击`素材库`按钮上で传動画/画像到素材库。
   - 选项2：使用本地文件夹作に素材来源，在脚本配置中設定文件夹パス。
2. 选择要运行脚本のデバイス。
3. 点击`脚本` > `发布投稿`。
4. 配置タスク設定：
    - **素材来源**：选择素材库または本地文件夹。
    - **本地文件夹パス**：など果使用本地文件夹，指定された含む動画/画像のパス。
    - **发布方式**：选择发布方式：系统シェアまたは添加按钮(`+`)または通じて検索使用声音。
    - **声音名称**：当使用"通じて検索使用声音"发布方式时，你できます以选择：
      - 输入声音名称进行検索（例など："original sound"，"trending music"）
      - 直接输入TikTok声音リンク（例など：`https://www.tiktok.com/t/ZP8hvBwCb/`）
    - **コンテンツ类型**：选择動画または画像。
    - **添加声音开关**：启用/禁用添加声音。
    - **上で传など待时间**：設定上で传后の延迟时间（5-60秒），确保TikTok服务器完全处理コンテンツ后再继续操作。
    - **添加产品リンク开关**：启用/禁用添加产品リンク。
    - **Caption 形式**：选择两种 caption 形式之一：
      - **多行模式**：每行是一条独立 caption，系统按顺序循环使用。
      - **Spintax 模式**：使用 `{选项1|选项2}` 语法生成ランダム文本。
    - **タイトル**：に基づいて所选形式输入 caption。サポート提及用户（など @TikMatrixPhoneFarm）和标签（など #tag1 #tag2 #tag3）。详细形式说明见下文。
    - **素材标签**：按标签过滤素材。など果未設定，使用所有素材。
5. 点击`开始脚本`开始実行。

## Caption 形式说明

系统サポート两种 caption 形式，让您灵活制御投稿タイトル。

### 多行模式（默认）

**工作原理：**
- 每一行代表一条独立 caption
- 系统按顺序使用 caption し循环
- 在一行内使用 `###` 添加换行符

**示例：**

```
第一条caption###支持换行
第二条caption###也支持换行
第三条caption不换行
```

**平台差异：**
- **TikTok**：使用 `##` 分隔タイトル和説明（例など：`タイトル##长説明`）
- **Instagram**：不サポート `##`，会显示警告

### Spintax 模式

**工作原理：**
- 只有一个 caption 模板
- 使用 `{选项1|选项2|选项3}` 语法实现ランダム变化
- 使用**原生换行符**（按回车键）实现多行 caption
- 不要使用 `###` - 它不会被解析

**示例：**

```
{今天|明天|后天}的天气真{好|不错|棒}
欢迎关注我的账号##想了解更多请点击链接
```

**平台差异：**
- **TikTok**：`##` できます用于分隔タイトル和説明
- **Instagram**：不サポート `##`

**注意：** 在 Spintax 模式下，など果检测到 `###`，系统会提示使用原生换行符。

## 素材管理

### 音楽使用模式

当**发布方式**設定に"通じて検索使用声音"时，脚本サポート两种異なるの音楽添加模式：

#### 1. 声音検索模式

输入声音名称在TikTok内検索音楽：

- 示例：`original sound`，`trending music`，`viral beat`
- 脚本会検索TikTokの音楽库し选择第一个匹配结果
- 这种模式适用于使用热门または流行声音

#### 2. 声音リンク模式

输入直接のTikTok声音リンク使用特定音楽：

- サポートのリンク形式：
  - 短リンク：`https://www.tiktok.com/t/ZP8hvBwCb/`
  - 音楽页面：`https://www.tiktok.com/music/original-sound-7473897776619834646`
  - VMリンク：`https://vm.tiktok.com/ZMhvBwCbD/`
- 脚本自動检测以`https://`开头のリンクし直接開く
- 这种模式适用于使用来自特定動画の特定声音

### 素材库

- **素材标签**：に素材添加标签，在发布时进行过滤。など果未指定された标签，使用所有素材。
- **素材タイトル**：你できます以に每个素材設定カスタムタイトル。など果設定，它优先于发布設定。
- **素材ユーザー名**：你できます以に每个素材設定カスタムユーザー名。匹配ユーザー名の素材优先使用。
- **素材ソート**：に素材設定ソート号（对多画像投稿有用；数字越小越靠前）。

### 本地文件夹来源

- **文件夹结构**：

  ```text
  /素材路径/
  ├── caption.txt (可选 - 根目录兜底)
  ├── sound_names.txt (可选 - 根目录兜底)  
  ├── sound_urls.txt (可选 - 根目录兜底)
  ├── @用户名1/
  │   ├── video1.mp4
  │   ├── video2.mp4
  │   ├── image1.jpg
  │   ├── image2.jpg
  │   ├── caption.txt (可选)
  │   ├── sound_names.txt (可选)
  │   └── sound_urls.txt (可选)
  ├── @用户名2/
  │   ├── content1.mp4
  │   ├── content2.jpg
  │   ├── caption.txt (可选)
  │   ├── sound_names.txt (可选)
  │   └── sound_urls.txt (可选)
  └── ...
  ```

- **文件夹パス**：指定された含むユーザー名子文件夹の根パス。
- **ユーザー名子文件夹**：每个ユーザー名必须有自己の子文件夹（例など，`@ユーザー名1`，`@ユーザー名2`）。
- **根文件夹兜底**：など果ユーザー名文件夹不存在またはユーザー名に空，将从根文件夹读取素材和設定。
- **サポート形式**：
  - 動画：MP4
  - 画像：JPG, PNG
- **タイトル文件**：在每个ユーザー名文件夹中できます选の`caption.txt`文件，用于カスタムタイトル。など果找不到，将回退到根文件夹の`caption.txt`。
  - **形式说明**：Caption.txt 仅サポート Spintax 模板形式（と UI の Spintax 模式用法相同）：
    - 配置一个 Spintax 模板，使用 `{选项1|选项2}` 语法实现ランダム变化
    - 使用原生换行符（回车键）实现多行
    - サポート `##` タイトル/説明分隔（仅 TikTok，Instagram 不サポート）
  - **注意**：Caption.txt 不サポート Multiline 顺序选择模式
- **音楽文件**：
  - **sound_names.txt**：含む用于検索の音楽名称，每行一个。按顺序读取，到末尾后循环读取。
  - **sound_urls.txt**：含むTikTok音楽リンク，每行一个。按顺序读取，到末尾后循环读取。
  - など果ユーザー名文件夹不存在，从根文件夹の音楽文件读取。
- **文件选择**：
  - 对于動画：每次发布ランダム选择一个動画
  - 对于画像：に基づいて画像数量設定选择多张画像
- **使用跟踪**：文件会被自動标记に済み使用，避免重复。

### Caption.txt 文件形式说明

当使用**本地文件夹来源**时，`caption.txt` 文件用于配置 Spintax 模板。此文件できます以覆盖 UI 中のタイトル配置。

**重要说明**：Caption.txt 仅サポート Spintax 模板形式，不サポート Multiline 顺序选择模式。

#### Spintax 模板形式

配置一个 Spintax 模板，使用 `{选项1|选项2}` 语法生成ランダム文本变化。使用原生换行符（按回车键）实现多行：

```text
{早上好|你好|嗨}！看看这个{令人惊叹的|不可思议的|很棒的}视频！
{关注获取更多|点赞分享|标记你的朋友}
```

できます能の输出：

- "早上で好！看看这个令人惊叹の動画！"
- "你好！看看这个不できます思议の動画！"
- "嗨！看看这个很棒の動画！"

**复杂 Spintax 变化：**

```text
{🔥|✨|🎉} {不要错过|查看|观看}这个{令人惊叹的|不可思议的|很棒的}{视频|内容|帖子}！
{关注获取更多|点赞分享|标记你的朋友} {#热门|#病毒式|#fyp} #tiktok
```

**Spintax と TikTok タイトル/説明分隔：**

```text
{激动人心的|令人惊叹的|不可思议的}{新闻|更新|发现}##这是您需要了解的关于这个{精彩|美妙|非凡}{发展|更新|故事}的内容！
```

#### 完整 Caption.txt 示例

一个完整の Spintax 模板示例：

```text
{早晨动力|每日灵感|开始你的一天}！💪
{喜欢|享受|迷恋}这个{美丽的|令人惊叹的|华丽的}景色！🌅
{不敢相信|对此很兴奋|对此很激动}{这一刻|这次经历|这次冒险}！✨
```

**带 TikTok タイトル/説明分隔の示例：**

```text
{精彩|有趣|好玩}的视频内容##这是一段{令人惊叹|不可思议|很棒}的{视频|内容|作品}，欢迎{点赞|关注|转发}！
```

  **Sound_names.txt 文件示例：**

  ```text
  原声
  热门音乐
  病毒式节拍
  流行歌曲
  舞蹈音乐
  ```

  **Sound_urls.txt 文件示例：**

  ```text
  https://www.tiktok.com/t/ZP8hvBwCb/
  https://www.tiktok.com/music/original-sound-7473897776619834646
  https://vm.tiktok.com/ZMhvBwCbD/
  https://www.tiktok.com/t/ZP8hvBwCb/
  ```

## 常见问题

1. に什么タスク会失败？
   - 自動化脚本是に基づいてTikTok/Instagramアプリの動作进行シミュレート操作の，某些情况下できます能会失败，例など网络问题、アプリ响应异常、意外の弹窗、デバイス性能问题、デバイス互換性问题など。
   - など果多次再試行仍然失败，提交工单获取帮助。
2. 怎么样实现无人值守运行？
   - 创建デバイス分组, し在分组のPost設定中使用定时スケジューリング機能，ソフトウェア会每天在指定された时间自動创建し运行发布タスク。
3. 怎么实现幻灯片投稿のソート？
   - 在素材库中に每个画像素材設定ソート号，数字越小越靠前。发布时脚本会に基づいてソート号选择画像。
   - 你也できます以一括インポート素材时使用符合下面规范の文件名来設定ソート号：
     - `sort_1_画像説明.png`
     - `sort_2_画像説明.png`
     - `sort_3_画像説明.png`
4. 怎么に每个アカウント使用異なるの素材,タイトル和音楽？
   - 使用本地文件夹时，に每个ユーザー名创建子文件夹，在子文件夹中创建`caption.txt`，`sound_names.txt`または`sound_urls.txt`文件来定义カスタムタイトル和音楽。脚本会优先使用匹配ユーザー名の文件。
   - 使用素材库时，に每个素材設定ユーザー名字段,タイトル字段,脚本会优先使用匹配ユーザー名の素材和タイトル。
5. サポート哪些素材形式？
   - 動画：MP4
   - 画像：JPG,PNG

6. 画像和動画素材できます以重复使用吗？

  不できます以。に保护您のアカウント，系统会在素材发布后自動将其标记に済み使用，避免被重复选用。这样できます以减少重复发布の风险，降低アカウント被风控または封禁のできます能性。
