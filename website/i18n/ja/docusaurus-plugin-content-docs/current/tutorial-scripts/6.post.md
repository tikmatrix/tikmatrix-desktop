# 投稿を公開

**投稿を公開**スクリプトは、TikTokで動画や画像を公開するために使用されます。

## 手順

1. **素材の準備**:
   - オプション1：`メディアライブラリ`ボタンをクリックして、メディアライブラリに動画/画像をアップロードします。
   - オプション2：ローカルフォルダを素材ソースとして使用し、スクリプト設定でフォルダパスを設定します。
2. スクリプトを実行するデバイスを選択します。
3. `スクリプト` > `投稿を公開`をクリックします。
4. タスク設定を構成します:
    - **素材ソース**: メディアライブラリまたはローカルフォルダを選択します。
    - **ローカルフォルダパス**: ローカルフォルダを使用する場合、動画/画像を含むパスを指定します。
    - **投稿方法**: システム共有、追加ボタン（`+`）、または「検索でサウンドを使用」から投稿方法を選択します。
    - **サウンド名**: 「検索でサウンドを使用」を使用する場合、以下が可能です:
      - サウンド名を入力して検索（例：「オリジナルサウンド」、「トレンドミュージック」）
      - TikTokサウンドリンクを直接入力（例：`https://www.tiktok.com/t/ZP8hvBwCb/`）
    - **コンテンツタイプ**: 動画または画像を選択します。
    - **サウンド追加トグル**: サウンドの追加を有効化/無効化します。
    - **アップロード待機時間**: アップロード後の遅延を設定（5～60秒）し、TikTokのサーバーがコンテンツを完全に処理してから続行するようにします。
    - **商品リンク追加トグル**: 商品リンクの追加を有効化/無効化します。
    - **キャプション形式**: 2つのキャプション形式から選択します:
      - **複数行**: 各行が個別のキャプションです。システムは順番に回転させます。
      - **Spintax**: `{オプション1|オプション2}`構文をランダムテキスト生成に使用します。
    - **キャプション**: 選択した形式に基づいてキャプションを入力します。メンション（例：@TikMatrixPhoneFarm）とハッシュタグ（例：#tag1 #tag2 #tag3）をサポートします。詳細な形式の説明は以下を参照してください。
    - **素材タグ**: タグで素材をフィルタリングします。設定されていない場合、すべての素材が使用されます。
5. **スクリプトを開始**をクリックして実行を開始します。

## キャプション形式

システムは2つのキャプション形式をサポートし、投稿キャプションを柔軟に制御できます。

### 複数行モード（デフォルト）

**仕組み:**
- 各行が1つの独立したキャプションを表します
- システムは順番にキャプションを使用し、最初に戻ってループします
- 行内で`###`を使用してキャプション内に改行を追加します

**例:**

```
最初のキャプション###改行あり
2番目のキャプション###改行もあり
3番目のキャプション改行なし
```

**プラットフォーム固有の機能:**
- **TikTok**: `##`を使用してタイトルと説明を分離します（例：`タイトル##長い説明`）
- **Instagram**: `##`はサポートされておらず、警告が表示されます

### Spintaxモード

**仕組み:**
- キャプションテンプレートは1つだけ
- ランダムなバリエーションには`{オプション1|オプション2|オプション3}`構文を使用します
- 複数行キャプションには**ネイティブ改行**（Enterキーを押す）を使用します
- `###`は使用しないでください - 解釈されません

**例:**

```
{今日|明日|昨日}天気は{良い|素晴らしい|最高}です
私のアカウントをフォローしてください##詳細情報はリンクをクリック
```

**プラットフォーム固有の機能:**
- **TikTok**: `##`はタイトルと説明を分離するために機能します
- **Instagram**: `##`はサポートされていません

**注意:** Spintaxモードでは、キャプションに`###`が表示される場合、システムは代わりにネイティブ改行の使用を提案します。

## 素材管理

### 音楽使用モード

**投稿方法**が*検索でサウンドを使用*に設定されている場合、スクリプトは音楽を追加するための2つのモードをサポートします:

#### 1. サウンド検索モード

TikTok内で検索するサウンド名を入力します:

- 例: `オリジナルサウンド`、`トレンドミュージック`、`バイラルビート`
- スクリプトはTikTokのサウンドライブラリを検索し、最初に一致する結果を選択します
- トレンド/人気のあるサウンドに最適

#### 2. サウンドリンクモード

特定のサウンドを使用するには、直接TikTokサウンドURLを入力します:

- サポートされているリンク形式:
  - ショートリンク: `https://www.tiktok.com/t/ZP8hvBwCb/`
  - ミュージックページ: `https://www.tiktok.com/music/original-sound-7473897776619834646`
  - VMリンク: `https://vm.tiktok.com/ZMhvBwCbD/`
- スクリプトは`https://`で始まるリンクを自動的に検出し、直接開きます
- 特定の動画から特定のサウンドを使用するのに最適

### メディアライブラリ

- **素材タグ**: 公開時にフィルタリングするために素材にタグを追加します。指定されていない場合、すべての素材が使用されます。
- **素材キャプション**: 各素材にカスタムキャプションを割り当てることができます。設定されている場合、公開設定を上書きします。
- **素材ユーザー名**: 各素材にユーザー名を割り当てることができます。一致するユーザー名を持つ素材が優先されます。
- **素材の並べ替え**: 素材に並べ替え番号を割り当てます（複数画像投稿に便利）。小さい番号が早く表示されます。

### ローカルフォルダソース

- **フォルダ構造**:

  ```text
  /material-root/
  ├── caption.txt (オプション - ルートフォールバック)
  ├── sound_names.txt (オプション - ルートフォールバック)
  ├── sound_urls.txt (オプション - ルートフォールバック)
  ├── @username1/
  │   ├── video1.mp4
  │   ├── video2.mp4
  │   ├── image1.jpg
  │   ├── image2.jpg
  │   ├── caption.txt (オプション)
  │   ├── sound_names.txt (オプション)
  │   └── sound_urls.txt (オプション)
  ├── @username2/
  │   ├── content1.mp4
  │   ├── content2.jpg
  │   ├── caption.txt (オプション)
  │   ├── sound_names.txt (オプション)
  │   └── sound_urls.txt (オプション)
  └── ...

  ````

- **フォルダパス**: ユーザー名サブフォルダを含むルートパスを指定します。
- **ユーザー名サブフォルダ**: 各ユーザー名には独自のサブフォルダが必要です（例：`@username1`、`@username2`）。
- **ルートフォルダフォールバック**: ユーザー名フォルダが存在しない、またはユーザー名が空の場合、素材と設定はルートフォルダから読み取られます。
- **サポートされている形式**:

  - 動画: MP4
  - 画像: JPG、PNG
- **キャプションファイル**: カスタムキャプション用のユーザー名フォルダごとのオプションの`caption.txt`。見つからない場合はルート`caption.txt`にフォールバックします。
  - **形式の説明**: Caption.txtはSpintaxテンプレート形式のみをサポートします（UIのSpintaxモードと同じ使用方法）:
    - ランダムなバリエーションには`{オプション1|オプション2}`構文を使用して1つのSpintaxテンプレートを構成します
    - 複数行コンテンツにはネイティブ改行（Enterキー）を使用します
    - `##`タイトル/説明分割をサポートします（TikTokのみ、Instagramは除く）
  - **注意**: Caption.txtは複数行の順次選択モードをサポートしていません
- **音楽ファイル**:

  - **sound_names.txt**: 検索用のサウンド名、1行に1つ。順番に読み取り、最後に達するとループします。
  - **sound_urls.txt**: TikTokサウンドURL、1行に1つ。順番に読み取り、最後に達するとループします。
  - ユーザー名フォルダが存在しない場合、ルート音楽ファイルから読み取ります。
- **ファイル選択**:

  - 動画: 毎回ランダムに1つ選択
  - 画像: 数に応じて複数選択
- **使用追跡**: ファイルは繰り返しを防ぐために自動的に使用済みとしてマークされます。

### Caption.txtファイル形式

**ローカルフォルダソース**を使用する場合、`caption.txt`ファイルはSpintaxテンプレートを構成するために使用されます。このファイルはUIからのキャプション設定を上書きできます。

**重要な注意**: Caption.txtはSpintaxテンプレート形式のみをサポートし、複数行の順次選択モードはサポートしません。

#### Spintaxテンプレート形式

ランダムなテキストバリエーションには`{オプション1|オプション2}`構文を使用して1つのSpintaxテンプレートを構成します。複数行コンテンツにはネイティブ改行（Enterを押す）を使用します:

```text
{おはようございます|こんにちは|やあ}! この{素晴らしい|信じられない|最高の}動画をチェックしてください!
{もっとフォローしてください|いいねとシェア|友達をタグ付け}
```

可能な出力:

- "おはようございます! この素晴らしい動画をチェックしてください!"
- "こんにちは! この信じられない動画をチェックしてください!"
- "やあ! この最高の動画をチェックしてください!"

**バリエーション付きの複雑なSpintax:**

```text
{🔥|✨|🎉} {見逃さないで|チェックアウト|見る} この{素晴らしい|信じられない|最高の} {動画|コンテンツ|投稿}!
{もっとフォローしてください|いいねとシェア|友達をタグ付け} {#trending|#viral|#fyp} #tiktok
```

**TikTokタイトル/説明分割付きのSpintax:**

```text
{エキサイティング|素晴らしい|信じられない} {ニュース|更新|発見}##この{素晴らしい|素晴らしい|注目すべき} {開発|更新|ストーリー}について知っておくべきことはこちらです!
```

#### 完全なCaption.txtの例

完全なSpintaxテンプレートの例:

```text
{朝のモチベーション|毎日のインスピレーション|1日を始めよう}! 💪
この{美しい|素晴らしい|驚くべき}景色を{愛して|楽しんで|夢中}! 🌅
この{瞬間|経験|冒険}を{信じられない|とても興奮|感激}! ✨
```

**TikTokタイトル/説明分割付きの例:**

```text
{素晴らしい|信じられない|最高の} {動画|コンテンツ|投稿}##この{素晴らしい|素晴らしい|注目すべき} {瞬間|経験|ストーリー}について知っておくべき{すべて|すべて|何}はこちらです!
```


  **Sound_names.txtの例:**

  ```text
  オリジナルサウンド
  トレンドミュージック
  バイラルビート
  人気曲
  ダンスミュージック
  ```

  **Sound_urls.txtの例:**

  ```text
  https://www.tiktok.com/t/ZP8hvBwCb/
  https://www.tiktok.com/music/original-sound-7473897776619834646
  https://vm.tiktok.com/ZMhvBwCbD/
  https://www.tiktok.com/t/ZP8hvBwCb/
  ```

## FAQ

1. **タスクが失敗するのはなぜですか？**

   - 自動化スクリプトはTikTok/Instagramの動作に基づいてユーザーアクションをシミュレートします。失敗はネットワークの問題、アプリの不具合、予期しないポップアップ、デバイスのパフォーマンスの問題、互換性の問題などにより発生する可能性があります。
   - 再試行後も失敗が続く場合は、サポートチケットを提出してください。

2. **無人でタスクを実行するにはどうすればよいですか？**

   - デバイスグループを作成し、グループの投稿設定でスケジュールを設定します。ソフトウェアは毎日指定された時間に自動的に公開タスクを作成して実行します。

3. **スライドショー投稿の順序を制御するにはどうすればよいですか？**

   - メディアライブラリの各画像素材に並べ替え番号を設定します。小さい番号が早く表示されます。
   - このパターンに従うファイル名を使用して素材を一括インポートすることもできます:

     - `sort_1_description.png`
     - `sort_2_description.png`
     - `sort_3_description.png`

4. **各アカウントに異なる素材、キャプション、音楽を使用するにはどうすればよいですか？**

   - ローカルフォルダの場合: 各ユーザー名のサブフォルダを作成し、カスタムキャプションと音楽を定義するために`caption.txt`、`sound_names.txt`、または`sound_urls.txt`ファイルを追加します。スクリプトは一致するユーザー名フォルダを優先します。
   - メディアライブラリの場合: 各素材にユーザー名とキャプションフィールドを設定すると、スクリプトは一致する素材を優先します。

5. **どの素材形式がサポートされていますか？**

   - 動画: MP4
   - 画像: JPG、PNG

6. 画像と動画の素材は再利用できますか？

  いいえ。アカウントを保護するために、システムは投稿後に素材を自動的に使用済みとしてマークし、再度選択されないようにします。これにより、重複投稿を防ぎ、アカウントのフラグやBANのリスクを軽減します。
