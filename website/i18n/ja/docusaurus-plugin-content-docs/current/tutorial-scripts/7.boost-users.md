# ユーザーブースト

:::warning 廃止のお知らせ
この機能は[スーパーマーケティング](./14.super-marketing.md)に統合されました。より強力で柔軟なユーザーブースト機能については、スーパーマーケティングスクリプトを使用してください。
:::

ユーザーブースト機能は、ターゲットアカウントを一括でフォローまたはフォロー解除することで、露出とエンゲージメントを増やします。システムは実際のユーザー動作をシミュレートして検出リスクを減らそうとしますが、プラットフォームルールに従い、適度に使用する必要があります。

## スーパーマーケティングでユーザーブースト機能を使用

1. スクリプトを開く：「スーパーマーケティング」を選択します。
2. データソースタイプ「ユーザーアカウントリスト」を選択します。
3. ユーザー名を一括で貼り付けるか、テキストファイルからインポートします（1行に1つのユーザー名）。インポート後、「追加」または「上書き」を選択します。
4. データ使用戦略を選択します：

    - 共有プール：すべてのボットアカウントが同じターゲットとインタラクトします。

    - 単一消費：各ボットアカウントが異なるターゲットを処理するため、各ターゲットは1つのアカウントのみで処理されます。

5. 「ユーザーアクション — 繰り返し」でターゲットごとの繰り返し回数を設定します（通常は1に設定）。
6. アクションタイプを選択：フォローまたはフォロー解除。**ユーザーアクション**でさらにオプションを構成できます：

        - プロファイルナビゲーション方法：**ユーザーを検索**または**プロファイルを直接開く**を選択します。

        - ダイレクトメッセージを送信（オプション）：有効にした場合、事前生成された複数行のDMテンプレートを貼り付けます（1行または段落に1つのテンプレート）。テンプレート使用順序を構成します：

                - **ランダム**：毎回ランダムに1つのテンプレートを選択します。
                - **順次**：貼り付け/インポートされた順序でテンプレートを使用します（タスク実行間で継続できます）。

            自然さを高めるために、テンプレートに絵文字を自動的に挿入することもできます。

7. 他のスクリプトパラメータ（並行性、遅延、アカウントプール）を構成し、設定を確認してから、スクリプトを開始します。

## 推奨事項と注意事項

- まず少数のターゲットでテストして、設定がプラットフォームの防御をトリガーしないことを確認します。
- 人間の動作を模倣し、リスクを減らすために、合理的な遅延と並行性を設定します。
- 長時間実行する場合は、アカウントの健全性とタスクログを監視し、必要に応じて戦略を調整します。
- ユーザーブースト操作は100％成功を保証しません。必要に応じて失敗したタスクを再試行するか、ターゲットアカウントのステータスを調査してください。
