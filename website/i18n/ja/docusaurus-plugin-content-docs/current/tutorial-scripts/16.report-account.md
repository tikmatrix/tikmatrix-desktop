# アカウントを報告

:::warning 責任ある使用
アカウントの報告は強力なアクションです。報告機能は、正当な違反（詐欺、スパムなど）に対してのみ使用してください。誤用すると、アカウント制限やその他の悪影響を引き起こす可能性があります。
:::

アカウント報告ガイドでは、スーパーマーケティングスクリプトによって公開された組み込み報告機能の使用方法を説明します。この機能は、固定されたアプリ内報告フローを自動化してユーザーアカウントのリストを処理するのに役立ちますが、以下で説明する固定選択以外の報告パスまたは報告理由のカスタマイズはサポートしていません。

## 概要

- スクリプトで使用される報告フローは固定されており、UIやスクリプトで変更できません：

```txt
報告 → アカウントを報告 → その他 → 詐欺とスキャム
```

- スクリプトは、ユーザーのプロファイルに移動し、報告メニューを開き、上記の固定パスを選択し、可能な場合は報告を送信しようとします。
- モバイルUIはアプリのバージョンとデバイスによって異なるため、スクリプトはベストエフォートのUIセレクターを使用します。常に最初に小規模なバッチでテストしてください。

## 実行手順（スーパーマーケティング経由）

1. TikMatrix/IgMatrixを開き、実行するデバイスを選択します（複数選択サポート）。
2. `スクリプト` → `スーパーマーケティング`を開きます。
3. **データソース**を`ユーザー名`に設定し、テキストファイルをインポートするかユーザー名を貼り付けます（1行に1つ）。インポート後、`追加`または`上書き`を選択します。
4. **ユーザーアクション**で、**アカウントを報告**（新しいトグル）を有効にし、表示された固定報告パスを確認します。
5. 必要に応じて、タスクの並行性、遅延、その他のタスク設定を構成します。
6. `スクリプトを開始`をクリックしてリストの処理を開始します。成功/失敗のフィードバックについては、ログとデバイス画面を監視してください。

## ヒントと推奨事項

- まず1〜5個のアカウントでテストして、報告UIが検出され、シーケンスがターゲットアプリのバージョンとデバイスで機能することを確認します。
- 保守的なペーシングと遅延を保ちます。大量の自動報告は、プラットフォームの防御またはレート制限をトリガーする可能性があります。
- この機能は固定されたアプリ内パスを使用します。アプリUIが変更された場合（テキストまたは要素の位置）、セレクターが更新されるまで自動化が失敗する可能性があります。
- 無実のユーザーを嫌がらせまたはターゲットにするために報告を使用しないでください。プラットフォームの規約に一致する明確な違反のみを報告してください。
- 実行後にログを確認してください。スクリプトは、メニューボタンや報告オプションの欠落などの失敗をログに記録するため、タイミングやセレクターを調整できます。

## 使用しない場合

- カスタム報告理由を選択したり、ケースごとに詳細な証拠を提出したりする必要がある場合 - スクリプトはそのレベルのカスタマイズをサポートしていません。
- 手動レビューが必要な法的または機密事項の場合。

## トラブルシューティング

- 「報告オプションが見つかりませんでした」/「'アカウントを報告'オプションが見つかりませんでした」：UIテキストまたはレイアウトが変更された可能性があります。小規模なテストセットで再実行し、デバッグのためにデバイスのスクリーンショットをキャプチャしてください。
- 報告が送信されているが効果がない場合、プラットフォームは自動報告を無視するか、モデレーションルールを適用している可能性があります。重要なケースでは手動レビューを検討してください。
