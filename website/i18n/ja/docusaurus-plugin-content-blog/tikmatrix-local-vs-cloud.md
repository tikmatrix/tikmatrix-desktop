## 🧭 1. 「ローカルデプロイ」の意味（そして違い）

ほとんどの「クラウドコントローラー」は、スマートフォンの画面と認証情報をサードパーティのサーバー経由でパイプします。
**TikMatrixはコンピュータ上で直接実行**され、USB/Wi-Fi経由でAndroidデバイスと通信します — 中間にコマンド/制御サーバーはありません。

* リモートセッションリレーなし
* ベンダー側の認証情報保存なし
* 強制マルチテナントインフラなし

> **原則:** あなたのハードウェア、あなたのネットワーク、あなたのデータ — **設計上ローカルに保持**。

---

## 🔒 2. データ所有権とデフォルトのプライバシー

ローカルは機密データをあなたの境界内に保ちます。

| 資産 | クラウド制御 | TikMatrixローカル |
| ------------------- | -------------------------------- | ------------------------------- |
| アカウント認証情報 | サーバー側でプロキシ/保存されることが多い | **ローカルのみに保存** |
| デバイスログ/画面 | サードパーティリレーを経由する可能性 | **LAN内に留まる** |
| コンテンツ資産 | リモートディスク/CDNにアップロード | **あなたのマシンから提供** |
| 規制上のエクスポージャー | マルチリージョンデータフットプリント | **シングルテナント、制御可能** |

> **ゼロトラスト姿勢:** インターネットは敵対的であると仮定し、マシンから出るものを最小限に抑えます。

---

## ⚡ 3. リアルタイム信頼性（レイテンシ、ジッタ、「クラウドグレムリン」）

リモートオーケストレーションはラウンドトリップと輻輳を導入します。ローカルはそれらを除去します。

* タップ、スワイプ、ビデオの再生/一時停止の**低レイテンシ**
* ベンダーのアップタイムやリレー帯域幅への**依存なし**
* スロットルされたクラウドネットワークからの「ファントム」障害が**少ない**

**結果:** より高いタスク完了率、より安定した長期実行セッション、ランダムな切断が少ない。

---

## 🧱 4. セキュリティモデル: より少ない攻撃面

すべてのクラウドホップは攻撃面を追加します（API、認証トークン、ソケット、ストレージバケット）。
ローカルファーストはこの爆発半径を減らします。

* セッションにアクセスできるベンダースーパー管理者なし
* 列挙する共有マルチテナントキューなし
* 他人のS3バケットに存在する「有用な」デバッグスナップショットなし

> **多層防御:** あなたが所有するハードウェア上にコントロールプレーン + データプレーンを保持します。

---

## 🧰 5. パワーユーザーの柔軟性（プロキシ、ルーティング、ツール）

ローカルは環境の完全な制御を提供します:

* デバイスを**デバイスごとのレジデンシャルプロキシ**にバインド
* カスタムDNS、スプリットトンネルVPN、または国固有のルートを使用
* 独自の**CIスクリプト、スケジューラー、またはSIEM**と統合
* マルチスクリーンストリーミング用のGPU/コーデック設定を微調整

クラウドプラットフォームは標準化する必要があります; ローカルセットアップは**専門化**できます。

---

## 💸 6. 予測可能なコストと線形スケーリング

クラウドの「シート」価格は成功を罰します; 帯域幅とリレー分が積み重なります。

| 成長段階 | クラウドコストカーブ | ローカルコストカーブ |
| ------------- | ----------------------------- | --------------------------------------- |
| 1〜10台 | 魅力的な「スターター」プラン | 1台のデスクトップで処理可能 |
| 20〜60台 | コストが跳ね上がる（帯域幅/リレー） | USBハブ/2台目のPCを追加 |
| 100台以上 | プレミアムエンタープライズティア | コモディティPC上で**水平スケール** |

**ローカルはハードウェアのようにスケール**し、SaaS請求のようにはスケールしません。

---

## 📏 7. 安定性 > ショートカット（運用規律）

私たちは短期的な爆発ではなく、**長期的な資産構築**のために最適化しています。

* **決定論的実行:** 同じマシン、同じネットワーク、同じ結果
* **再現可能な環境:** PC設定をスナップショットして複製
* **制御された変更ウィンドウ:** アップグレード時期を決定

> ショートカット（完全リモート制御）は初期段階では簡単に感じますが、規模とコンプライアンスの下では痛手となります。

---

## 🧪 8. ベンチマークスナップショット（代表的なラボセットアップ）

> 1台のワークステーション（i7/32GB）、電源付きハブ経由で20台の物理Android、LANプロキシ。

| 指標 | クラウドライクリレー | TikMatrixローカル |
| ------------------------------ | ---------------- | --------------- |
| ジェスチャーラウンドトリップ | 180〜350ms | **30〜60ms** |
| 2時間セッション切断率 | 8〜12% | **<2%** |
| バルク投稿成功（20台） | 86〜90% | **96〜99%** |

*指標のみ; 実世界ではプロキシ品質、USB電源、デバイス状態により変動します。*

---

## 🧩 9. クラウドがまだOKな場合（エッジケース）

* **監査/可観測性のみ:** 読み取り専用ダッシュボード（コントロールプレーンなし）
* **バースト計算:** 認証情報に触れないレンダリングまたはAIタスク
* **サイト間のチームコラボレーション:** あなたのハードウェア上に**セルフホスト**ゲートウェイを使用

制御または認証情報が関与する場合は、**ローカルに保持**します。

## ✅ 10. リスク制御チェックリスト（ローカルファースト）

| カテゴリ | 推奨事項 |
| ---------- | ---------------------------------------------------------- |
| データ | 認証情報/ログをローカルに保存; 保管時に暗号化; 定期的なバックアップ |
| ネットワーク | デバイスごとのレジデンシャルプロキシ; 共有VPNを避ける |
| デバイス | 物理Android; 電源付きハブ; 正常なケーブル |
| 運用 | 時間差スケジュール; 人間らしいランダム性; ヘルスアラート |
| 更新 | バージョンを固定; 変更ウィンドウ; ロールバック計画 |
| コンプライアンス | ログをオンプレミスに保持; データフローを文書化 |

## ⚡ マーケターがTikMatrixを選ぶ理由（設計上ローカルファースト）

* 🧠 **人間らしい自動化:** 検出を減らすためのランダム化されたタップ、スワイプ、タイピング
* 🎛️ **デバイスごとの分離:** デバイスレベルでのプロキシ、タイミング、タスクの分散
* 🕒 **信頼性の高いスケジューリング:** リレーボトルネックなしの長時間実行ジョブ
* 🔐 **デフォルトでプライベート:** ベンダーリレーなし、強制データアップロードなし
* 🧩 **オープン統合:** スクリプト、プロキシ、監視スタックに接続

ローカルデプロイは、制御が属する場所 — **あなたと共に** — に保ち、安定性、プライバシー、スケールを提供します。
