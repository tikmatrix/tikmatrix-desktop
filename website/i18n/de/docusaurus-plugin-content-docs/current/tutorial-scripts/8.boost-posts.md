# Posts fördern

:::warning Abschaltwarnung
Die Funktionalität dieses Skripts ist in [Super Marketing](./14.super-marketing.md) integriert. Bitte verwenden Sie Super Marketing für leistungsfähigere und flexiblere Möglichkeiten zur Post-Förderung.
:::

Mit der Post-Förderungsfunktion können Sie das Engagement durch Massen-Likes, Reposts, Favoriten und Aufrufe erhöhen. Das System versucht, das Verhalten echter Benutzer zu simulieren, um das Erkennungsrisiko zu verringern; befolgen Sie jedoch die Plattformrichtlinien und kontrollieren Sie die Vorgangshäufigkeit.

## Verwendung in Super Marketing

1. Öffnen Sie das Skript: Wählen Sie „Super Marketing".
2. Wählen Sie den Datenquellentyp „Post-Link-Liste".
3. Fügen Sie Post-Links massenhaft ein oder importieren Sie aus einer Textdatei; nach dem Import wählen Sie „Hinzufügen" oder „Überschreiben".
4. Wählen Sie eine Datennutzungsstrategie:

    - Gemeinsamer Pool: Alle Bots interagieren mit denselben Posts.

    - Einmalige Nutzung: Jeder Bot interagiert mit unterschiedlichen Posts, sodass jeder Post nur von einem Konto verarbeitet wird.

5. Legen Sie im Bereich „Post-Aktionen — Wiederholungen" die Anzahl der Interaktionen pro Post fest. Wenn Kommentare aktiviert sind, empfehlen wir, 1 einzustellen, um Duplikate zu vermeiden.
6. Wählen Sie im Bereich „Post-Aktionen" Interaktionstypen aus: Like, Teilen, Repost, Favorit, Kommentar, Video-Aufrufe usw.

     Wenn „Kommentar" ausgewählt ist, konfigurieren Sie Kommentarparameter wie folgt:

     - Vorgefertigte Kommentarvorlagen: Fügen Sie mehrzeilige oder mehrabsätzige Kommentarvorlagen ein (jede Zeile/Absatz wird als separater Kandidat gezählt).

     - Reihenfolge der Template-Nutzung: Wählen Sie **Zufall** oder **Nacheinander**.
         - Zufall: Jedes Mal wird ein Template zufällig ausgewählt.
         - Nacheinander: Templates werden in der Einfüge-/Importreihenfolge verwendet (nützlich für deterministische Deduplizierung).

     - Emoji-Einfügung: Sie können optional das automatische Einfügen von Emojis in Kommentare aktivieren, um mehr Natürlichkeit zu erreichen.

     - Echtzeit-Kommentargenerierung (optional – ChatGPT API):
         - Wenn aktiviert, kann das Skript ChatGPT-ähnliche APIs aufrufen, um kontextabhängige, eindeutige Kommentare basierend auf Post-Titel/Beschreibung zu generieren. Dies reduziert Kommentar-Wiederholungen erheblich und ist geeignet, wenn viele eindeutige Kommentare benötigt werden.
         - Hinweise: Für Echtzeitgenerierung ist ein gültiger API-Schlüssel erforderlich; beachten Sie Anforderungslimitierungen, Kosten und Datenschutzprobleme (senden Sie keine sensiblen Daten an Drittanbieter).

     - Zwei häufige Möglichkeiten, um Kommentar-Eindeutigkeit sicherzustellen:
         1. Laden Sie vorgefertigte Vorlagen und verwenden Sie sie nacheinander im Modus „Nacheinander" (jede Vorlage wird einmal verwendet, bis sie erschöpft ist).
         2. Verwenden Sie ChatGPT API für dynamische Echtzeit-Generierung eines eindeutigen Kommentars für jeden Post (basierend auf Inhalt).

     - Empfehlung: Testen Sie Ihre Kommentarstrategie zunächst auf einer kleinen Auswahl, bewerten Sie die Qualität und das Plattformverhalten; bei Verwendung von Echtzeitgenerierung überwachen Sie die API-Nutzung und Kosten.

7. Konfigurieren Sie die restlichen Parameter (Parallelität, Verzögerungen, Konten-Pool), überprüfen Sie die Einstellungen und starten Sie das Skript.

## Empfehlungen und Hinweise

- Testen Sie zuerst auf einer kleinen Anzahl von Posts, um sicherzustellen, dass die Einstellungen keine Schutzmaßnahmen der Plattform auslösen.
- Kommentare sollten vielfältig und natürlich sein; vermeiden Sie das Einfügen identischer Texte.
- Legen Sie angemessene Verzögerungen und Parallelität fest, um das Verhalten eines Menschen zu simulieren.
- Überwachen Sie bei längeren Läufen den Status der Konten und Aufgabenprotokolle und passen Sie die Strategie bei Bedarf an.
