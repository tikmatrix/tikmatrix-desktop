# Benutzer fördern

:::warning Abschaltwarnung
Die Funktionalität dieses Skripts ist in [Super Marketing](./14.super-marketing.md) integriert. Bitte verwenden Sie Super Marketing für leistungsfähigere und flexiblere Möglichkeiten zur Benutzerförderung.
:::

Mit der Benutzerförderungsfunktion können Sie die Reichweite und das Engagement durch Massen-Abonnieren oder Abmelden von Zielkonten erhöhen. Das System versucht, das Verhalten echter Benutzer zu simulieren, um das Erkennungsrisiko zu verringern; befolgen Sie jedoch die Plattformrichtlinien und kontrollieren Sie die Vorgangshäufigkeit.

## Verwendung in Super Marketing

1. Öffnen Sie das Skript: Wählen Sie „Super Marketing".
2. Wählen Sie den Datenquellentyp „Benutzernamen-Liste".
3. Fügen Sie Benutzernamen massenhaft ein oder importieren Sie aus einer Textdatei (ein Benutzername pro Zeile); nach dem Import wählen Sie „Hinzufügen" oder „Überschreiben".
4. Wählen Sie eine Datennutzungsstrategie:

   - Gemeinsamer Pool: Alle Bots interagieren mit denselben Zielen.

   - Einmalige Nutzung: Jeder Bot verarbeitet unterschiedliche Ziele, sodass jedes Ziel nur von einem Konto verarbeitet wird.

5. Legen Sie im Bereich „Benutzeraktionen — Wiederholungen" die Anzahl der Wiederholungen für jedes Ziel fest (normalerweise 1).
6. Wählen Sie den Operationstyp: Abonnieren oder Abmelden. Zusätzlich können Sie im Bereich „Benutzeraktionen" konfigurieren:

    - Methode zum Öffnen des Profils: Wählen Sie **Benutzersuche** oder **Direktes Profilöffnen (Direct Open Profile)**.

    - Persönliche Nachricht senden (optional): Wenn aktiviert, fügen Sie vorgefertigte mehrzeilige Nachrichtenvorlagen ein (jede Zeile oder jeder Absatz = ein Template). Konfigurieren Sie die Reihenfolge der Template-Nutzung:

          - **Zufall**: Jedes Mal wird ein zufälliges Template ausgewählt.
          - **Nacheinander**: Templates werden in der Einfüge-/Importreihenfolge verwendet (können zwischen Aufgabenläufen fortgesetzt werden).

        Sie können auch das automatische Einfügen von Emojis in Templates aktivieren, um mehr Natürlichkeit zu erreichen.

7. Konfigurieren Sie die restlichen Parameter (Parallelität, Verzögerungen, Konten-Pool), überprüfen Sie die Einstellungen und starten Sie das Skript.

## Empfehlungen und Hinweise

- Testen Sie zuerst auf einer kleinen Anzahl von Zielen, um sicherzustellen, dass die Einstellungen keine Schutzmaßnahmen der Plattform auslösen.
- Legen Sie angemessene Verzögerungen und Parallelität fest, um das Verhalten eines Menschen zu simulieren.
- Überwachen Sie bei längeren Läufen den Status der Konten und Aufgabenprotokolle und passen Sie die Strategie bei Bedarf an.
- Benutzerförderungsvorgänge garantieren keinen 100% Erfolg; wiederholen Sie die Aufgabe bei Fehlern oder überprüfen Sie den Status des Zielkontos.
