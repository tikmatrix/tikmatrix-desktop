# Konto melden

:::warning Verantwortungsvolle Nutzung
Die Melde-Funktion ist ein leistungsstarkes Tool. Nutzen Sie es nur bei bestätigten Verstößen (Betrug, Betrügereien, Spam usw.). Missbrauch von Meldungen kann zu Kontobeschränkungen oder anderen unerwünschten Folgen führen.
:::

Diese Anleitung beschreibt die integrierte Funktion „Konto melden" im Super Marketing Skript. Die Funktion automatisiert einen festgelegten internen Meldepfad zur Verarbeitung einer Liste von Konten, unterstützt aber nicht die Änderung des Meldepfads oder die Auswahl benutzerdefinierter Gründe.

## Überblick

- Festgelegter Meldepfad, der vom Skript verwendet wird (kann nicht geändert werden):

```text
Melden → Konto melden → Anderes → Betrügerei und Betrug
```

- Das Skript versucht, die Profilseite zu öffnen, das Optionsmenü zu öffnen, den festgelegten Pfad zu durchlaufen und die Meldung einzureichen, wenn möglich.
- Die Schnittstelle der mobilen App kann zwischen Versionen und Geräten variieren, daher nutzt die Automatisierung die besten verfügbaren Selektoren – testen Sie unbedingt auf einer kleinen Auswahl.

## Ausführung über Super Marketing

1. Öffnen Sie TikMatrix/IgMatrix und wählen Sie Gerät(e) zum Ausführen aus (Mehrfachauswahl wird unterstützt).
2. Öffnen Sie `Skripte` → `Super Marketing`.
3. Stellen Sie **Datenquelle** auf `Benutzernamen` ein und importieren Sie eine Datei oder fügen Sie Benutzernamen ein (einen pro Zeile). Nach dem Import wählen Sie `Hinzufügen` oder `Überschreiben`.
4. Aktivieren Sie im Bereich **Benutzeraktionen** den Schalter **Konto melden** und überprüfen Sie den angezeigten festgelegten Meldepfad.
5. Konfigurieren Sie Aufgaben-Parallelität, Verzögerungen und andere Parameter.
6. Klicken Sie auf `Skript starten`, beobachten Sie Protokolle und Gerätebildschirm, um die erfolgreiche Ausführung zu bestätigen.

## Empfehlungen und Hinweise

- Testen Sie zunächst auf 1–5 Konten, um sicherzustellen, dass die UI korrekt erkannt wird und die Sequenz für Ihre App-Version und Geräte funktioniert.
- Verwenden Sie vorsichtige Intervalle und Verzögerungen: Massen-automatische Meldungen können Beschränkungen auslösen oder Plattform-Schutzsysteme auslösen.
- Wenn sich die App-UI geändert hat (Schaltflächen-Text oder Element-Positionen), kann das Skript erforderliche Elemente nicht finden – Selector-Updates oder Erwartungsänderungen sind erforderlich.
- Reichen Sie Meldungen nur bei offensichtlichen Verstößen ein, nutzen Sie die Funktion nicht zum Belästigen oder Harassen.
- Überprüfen Sie nach Ausführung die Protokolle: Das Skript protokolliert Fehler wie "keine Menü-Schaltfläche" oder "keine 'Melden'-Option", was beim Debuggen hilft.

## Wann nicht verwenden

- Wenn Sie einen benutzerdefinierten Meldungsgrund wählen oder Beweise beifügen müssen – das Skript bietet solche Funktionen nicht.
- In rechtlich bedeutsamen oder sensiblen Fällen, die manuelle Überprüfung erfordern.

## Fehlerbehebung

- Fehler "Could not find Report option" / "Could not find 'Report account' option": Die App-Schnittstelle hat sich wahrscheinlich geändert. Starten Sie einen Test mit kleiner Auswahl neu und sammeln Sie Screenshots zum Debuggen.
- Wenn Meldungen eingereicht werden, aber die Plattform keine Maßnahmen ergreift, ignoriert die Plattform möglicherweise automatische Meldungen oder erfordert manuelle Moderation – erwägen Sie in solchen Fällen manuelle Überprüfung.
