# การตั้งค่างาน

## กฎการกำหนดเวลางาน

- งานที่กำหนดไว้จะถูกดำเนินการภายใน 10 วินาทีหลังจากเวลาเริ่มต้น
- อุปกรณ์แต่ละเครื่องจะรันงานเพียงหนึ่งงานในแต่ละครั้ง งานเพิ่มเติมจะถูกจัดคิวอัตโนมัติสำหรับแต่ละอุปกรณ์
- งานที่ทำงานพร้อมกันสูงสุดจะถูกจำกัดโดยระดับไลเซนส์ ผู้ใช้ฟรีจะไม่สามารถรันงานอัตโนมัติและจะอยู่ในสถานะ "Waiting"
- ก่อนการลองใหม่แต่ละครั้ง ระบบจะใช้ความล่าช้าแบบสุ่มระหว่าง 5 ถึง 60 วินาที

ไดอะล็อกนี้ช่วยให้คุณควบคุมวิธีการลองใหม่งานและลำดับการดำเนินการที่ใช้โดยอุปกรณ์ ด้านล่างเป็นตัวเลือกและวิธีใช้งาน

## การลองใหม่อัตโนมัติ

ฟังก์ชัน:

- ลองใหม่งานที่ล้มเหลวโดยอัตโนมัติจนกว่าจะสำเร็จหรือถึงขีดจำกัดการลองใหม่

วิธีใช้:

- เปิดใช้งานเมื่อความล้มเหลวมักเป็นชั่วคราว (ปัญหาเครือข่าย ปัญหาแอปชั่วคราว)
- ทดสอบกับชุดงานเล็กๆ ก่อนเปิดใช้งานทั่วทั้งระบบ

## จำนวนครั้งสูงสุดในการลองใหม่

ฟังก์ชัน:

- กำหนดจำนวนครั้งสูงสุดของการลองใหม่อัตโนมัติสำหรับงานที่ล้มเหลว

วิธีใช้:

- เลือกค่าที่อนุรักษ์นิยม (2–3) เพื่อหลีกเลี่ยงลูปการลองใหม่ที่ยาว หรือเพิ่มค่าเมื่อความล้มเหลวชั่วคราวเกิดขึ้นบ่อยครั้ง
- UI บังคับขอบเขตที่สมเหตุสมผล (เช่น 1–10)

## ลำดับแบบสุ่ม

ฟังก์ชัน:

- ดำเนินการงานในลำดับแบบสุ่มต่ออุปกรณ์เพื่อลดพฤติกรรมที่ซิงโครไนซ์

วิธีใช้:

- เปิดใช้งานสำหรับกลุ่มอุปกรณ์ขนาดใหญ่หรือเมื่อคุณต้องการรูปแบบงานที่คาดเดาได้น้อยลง

## การใช้การเปลี่ยนแปลง

- การเปลี่ยนแปลงจะมีผลทันที ไดอะล็อกจะแสดงการแจ้งเตือนความสำเร็จหรือข้อผิดพลาด

## ตำแหน่งที่จะหา

- เปิดหน้า Devices หรือ Tasks และคลิกไอคอน Task Settings (เฟือง)

## เคล็ดลับด่วน

- เริ่มต้นด้วย `Max Retry Count` = 2–3 และปรับตามความล้มเหลวที่สังเกตได้
- ใช้ `Random Order` เพื่อทำให้การรันขนาดใหญ่ดูเป็นธรรมชาติมากขึ้น
- ตรวจสอบ `Auto Retry` กับตัวอย่างเล็กๆ ก่อนใช้งานอย่างกว้างขวาง

---

หากคุณต้องการคู่มือเวอร์ชันภาษาท้องถิ่น ให้สลับภาษาของเว็บไซต์โดยใช้ตัวเลือกภาษาที่มุมบนขวา
