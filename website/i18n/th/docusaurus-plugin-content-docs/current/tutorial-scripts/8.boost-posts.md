# เพิ่มโพสต์

:::warning ประกาศยุติการใช้งาน
ฟังก์ชันนี้ได้รับการรวมเข้ากับ [Super Marketing](./14.super-marketing.md) โปรดใช้สคริปต์ Super Marketing สำหรับคุณสมบัติการเพิ่มโพสต์ที่ทรงพลังและยืดหยุ่นมากขึ้น
:::

ฟีเจอร์เพิ่มโพสต์เพิ่มการมีส่วนร่วมโดยการกดถูกใจ แชร์ บันทึก รีโพสต์ และดูโพสต์เป็นจำนวนมาก ระบบพยายามจำลองพฤติกรรมผู้ใช้จริงเพื่อลดความเสี่ยงในการตรวจจับ แต่คุณควรปฏิบัติตามกฎของแพลตฟอร์มและใช้งานอย่างพอประมาณ

## การใช้ฟีเจอร์เพิ่มโพสต์ใน Super Marketing

1. เปิดสคริปต์: เลือก "Super Marketing"
2. เลือกประเภทแหล่งข้อมูล "Post URL list"
3. วาง URL โพสต์เป็นจำนวนมากหรือนำเข้าจากไฟล์ข้อความ หลังจากนำเข้าเลือก "Append" หรือ "Overwrite" รายการที่มีอยู่
4. เลือกกลยุทธ์การใช้ข้อมูล:

    - Shared pool: บัญชีบอททั้งหมดโต้ตอบกับโพสต์เดียวกัน

    - Single-consumption: แต่ละบัญชีบอทโต้ตอบกับโพสต์ที่แตกต่างกัน ดังนั้นแต่ละโพสต์จะถูกจัดการโดยบัญชีเดียวเท่านั้น

5. ตั้งค่าจำนวนการทำซ้ำต่อโพสต์ใน "Post Actions — Repeats" หากเปิดใช้งานความคิดเห็น ตั้งค่าการทำซ้ำความคิดเห็นเป็น 1 เพื่อหลีกเลี่ยงความคิดเห็นซ้ำ
6. ใน "Post Actions" เลือกประเภทการโต้ตอบที่คุณต้องการ: Like, Share, Repost, Favorite, Comment, Watch video เป็นต้น

     หากคุณเลือก "Comment" กำหนดค่าตัวเลือกความคิดเห็นดังนี้:

     - เทมเพลตความคิดเห็นที่สร้างไว้ล่วงหน้า: วางเทมเพลตความคิดเห็นแบบหลายบรรทัดหรือหลายย่อหน้า (แต่ละบรรทัด/ย่อหน้าถูกถือว่าเป็นความคิดเห็นผู้สมัคร)

     - ลำดับการใช้เทมเพลต: เลือก **Random** หรือ **Sequential**
         - Random: เลือกเทมเพลตแบบสุ่มแต่ละครั้ง
         - Sequential: ใช้เทมเพลตตามลำดับที่วาง/นำเข้า (มีประโยชน์สำหรับการยกเลิกการทำซ้ำที่กำหนดไว้)

     - การแทรกอีโมจิ: เปิดใช้งานการแทรกอีโมจิอัตโนมัติลงในความคิดเห็นเพื่อเพิ่มความเป็นธรรมชาติ

     - การสร้างความคิดเห็นแบบเรียลไทม์ (ทางเลือก — ChatGPT API):
         - เมื่อเปิดใช้งาน สคริปต์อาจเรียก API คล้าย ChatGPT เพื่อสร้างความคิดเห็นที่ตระหนักถึงบริบทและไม่ซ้ำกันตามหัวเรื่อง/คำอธิบายโพสต์ สิ่งนี้ช่วยลดความคิดเห็นซ้ำได้อย่างมากและเหมาะสมเมื่อต้องการความคิดเห็นที่ไม่ซ้ำกันจำนวนมาก
         - หมายเหตุ: การใช้การสร้างแบบเรียลไทม์ต้องการ API key ที่ถูกต้องและให้ความสำคัญกับข้อจำกัดอัตรา ค่าใช้จ่าย และความเป็นส่วนตัว (หลีกเลี่ยงการส่งข้อมูลที่ละเอียดอ่อนไปยังบริการบุคคลที่สาม)

     - สองวิธีทั่วไปในการรับประกันความคิดเห็นที่ไม่ซ้ำกัน:
         1. โหลดและหมุนเวียนเทมเพลตที่สร้างไว้ล่วงหน้าในโหมด **Sequential** (แต่ละเทมเพลตจะถูกใช้ครั้งเดียวจนหมด)
         2. ใช้ ChatGPT API เพื่อสร้างความคิดเห็นที่ไม่ซ้ำกันต่อโพสต์ (การสร้างแบบไดนามิกตามเนื้อหาโพสต์)

     - คำแนะนำ: ทดสอบกลยุทธ์ความคิดเห็นของคุณในชุดเล็ก ตรวจสอบคุณภาพความคิดเห็นและการจัดการแพลตฟอร์ม หากใช้การสร้างแบบเรียลไทม์ ติดตามการใช้งาน API และค่าใช้จ่าย

7. กำหนดค่าพารามิเตอร์สคริปต์อื่นๆ (การทำงานพร้อมกัน ความล่าช้า กลุ่มบัญชี) ตรวจสอบการตั้งค่า จากนั้นเริ่มสคริปต์

## คำแนะนำและหมายเหตุ

- ทดสอบกับโพสต์จำนวนน้อยก่อนเพื่อให้แน่ใจว่าการตั้งค่าไม่กระตุ้นการป้องกันแพลตฟอร์ม
- รักษาข้อความความคิดเห็นให้หลากหลายและเป็นธรรมชาติ หลีกเลี่ยงการคัดลอกข้อความเดียวกัน
- ตั้งค่าความล่าช้าและการทำงานพร้อมกันที่เหมาะสมเพื่อเลียนแบบพฤติกรรมมนุษย์และลดความเสี่ยง
- สำหรับการรันที่ยาวนาน ติดตามสุขภาพบัญชีและบันทึกงาน และปรับกลยุทธ์ตามความจำเป็น
