# แสดงความคิดเห็นจำนวนมาก

:::warning
ฟีเจอร์นี้ได้รับการรวมเข้ากับ [สคริปต์ Super Marketing](./14.super-marketing.md) โปรดใช้ Super Marketing สำหรับการแสดงความคิดเห็นเป็นจำนวนมากและความสามารถในการมีส่วนร่วมที่ทรงพลังและยืดหยุ่นมากขึ้น
:::

แสดงความคิดเห็นจำนวนมากใช้สำหรับโพสต์การตอบกลับการมีส่วนร่วมในโพสต์ TikTok จำนวนมาก—มีประโยชน์สำหรับการเลี้ยงดูบัญชี การส่งเสริมการขาย หรือการเพิ่มกิจกรรม ระบบรองรับทั้งเทมเพลตที่สร้างไว้ล่วงหน้าและการสร้างแบบเรียลไทม์ และพยายามเลียนแบบเวลาแบบมนุษย์เพื่อลดความเสี่ยงในการตรวจจับ

## การใช้แสดงความคิดเห็นจำนวนมากใน Super Marketing

1. เปิดสคริปต์ Super Marketing
2. เลือกประเภทแหล่งข้อมูล: "Post URL list"
3. วาง URL โพสต์เป็นจำนวนมากหรือนำเข้าไฟล์ `.txt` (หนึ่ง URL ต่อบรรทัด) หลังจากนำเข้าคุณสามารถเลือก "Append" หรือ "Replace" รายการปัจจุบัน
4. เลือกกลยุทธ์การใช้ข้อมูล:

   - Shared pool: บัญชีบอททั้งหมดจะแสดงความคิดเห็นจากกลุ่มเป้าหมายเดียวกัน

   - Single-consumption: แต่ละบัญชีบอทประมวลผลเป้าหมายที่แตกต่างกัน ดังนั้นแต่ละโพสต์จะได้รับความคิดเห็นจากบัญชีเดียวเท่านั้น

5. ตั้งค่าจำนวนความคิดเห็นที่จะโพสต์ต่อเป้าหมาย (โดยทั่วไป 1 เพื่อหลีกเลี่ยงความคิดเห็นซ้ำ)
6. กำหนดค่าการตั้งค่าความคิดเห็น:

   - เทมเพลตที่สร้างไว้ล่วงหน้า: วางหลายบรรทัดหรือย่อหน้าของข้อความความคิดเห็นที่เตรียมไว้ แต่ละบรรทัด/ย่อหน้าถูกถือว่าเป็นความคิดเห็นผู้สมัคร

   - ลำดับความคิดเห็น: เลือก "Random" หรือ "Sequential"

   - Random: แต่ละครั้งที่ส่งเลือกเทมเพลตแบบสุ่ม

   - Sequential: หมุนเวียนเทมเพลตตามลำดับที่วาง/นำเข้า (มีประโยชน์เมื่อคุณต้องการกลยุทธ์การยกเลิกการทำซ้ำที่ทำซ้ำได้)

   - แทรกอีโมจิ: เปิดใช้งานการแทรกอีโมจิอัตโนมัติเพื่อเพิ่มความเป็นธรรมชาติ

   - แทรกหมายเลขอุปกรณ์ (ทางเลือก): ผนวกป้ายตัวเลขอุปกรณ์ลงในความคิดเห็นสำหรับการติดตาม (ใช้อย่างระมัดระวัง มีไว้สำหรับการดีบัก/ตรวจสอบ)

   - การสร้างความคิดเห็นแบบเรียลไทม์ (ทางเลือก, ChatGPT API):

      - เมื่อเปิดใช้งาน สคริปต์สามารถเรียก ChatGPT หรือ API ที่คล้ายกันเพื่อสร้างความคิดเห็นที่ตระหนักถึงบริบทและไม่ซ้ำกันตามหัวเรื่อง/คำอธิบายโพสต์หรือบริบทที่มี—เหมาะสมเมื่อคุณต้องการความคิดเห็นที่ไม่ซ้ำกันจำนวนมาก

      - หมายเหตุ: การเปิดใช้งานการสร้างแบบเรียลไทม์ต้องการ API key ที่ถูกต้องและคุณต้องพิจารณาข้อจำกัดอัตรา ค่าใช้จ่าย และความเป็นส่วนตัว (หลีกเลี่ยงการส่งข้อมูลที่ละเอียดอ่อนไปยังบริการบุคคลที่สาม)

   - สองวิธีทั่วไปในการรับประกันความคิดเห็นที่ไม่ซ้ำกัน:

      1. ใช้การหมุนเวียนตามลำดับของเทมเพลตที่สร้างไว้ล่วงหน้า (แต่ละเทมเพลตจะถูกใช้ครั้งเดียวจนหมด)

      2. ใช้การสร้างแบบเรียลไทม์ของ ChatGPT เพื่อสร้างความคิดเห็นที่ไม่ซ้ำกันต่อเป้าหมาย

7. กำหนดค่าพารามิเตอร์สคริปต์อื่นๆ (การทำงานพร้อมกัน ช่วงเวลางาน กลุ่มบัญชี ฯลฯ) และเริ่มสคริปต์เมื่อพร้อม

:::tip เกี่ยวกับช่วงเวลางาน
ตั้งค่าช่วงเวลาต่ำสุดและสูงสุดและเปิดใช้งานความล่าช้าแบบสุ่ม การกำหนดเวลาที่แตกต่างกันลดโอกาสของกิจกรรมที่ซิงโครไนซ์และลดความเสี่ยงในการตรวจจับ
:::

## คำแนะนำและหมายเหตุ

- ทดสอบกลยุทธ์ความคิดเห็นในชุดเล็กก่อนเพื่อตรวจสอบคุณภาพและการตอบสนองของแพลตฟอร์ม
- รักษาข้อความความคิดเห็นให้สอดคล้องและหลากหลาย หลีกเลี่ยงการส่งข้อความที่เหมือนกันจำนวนมากในช่วงเวลาสั้น
- หากใช้การสร้างแบบเรียลไทม์ ติดตามการใช้งาน API และค่าใช้จ่ายอย่างเข้มงวดและหลีกเลี่ยงการส่งข้อมูลที่ละเอียดอ่อน
- งานอาจล้มเหลว ตรวจสอบบันทึกและลองแถวที่ล้มเหลวอีกครั้ง เป้าหมายที่ประมวลผลแล้วจะถูกข้ามในการรันครั้งต่อไป
