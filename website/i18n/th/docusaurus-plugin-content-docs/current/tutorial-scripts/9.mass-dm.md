# ส่งข้อความโดยตรงจำนวนมาก

:::warning
ฟีเจอร์นี้ได้รับการรวมเข้ากับ [สคริปต์ Super Marketing](./14.super-marketing.md) โปรดใช้ Super Marketing สำหรับการส่งข้อความจำนวนมากและการจัดการข้อความที่ทรงพลังและยืดหยุ่นมากขึ้น
:::

ส่งข้อความโดยตรงจำนวนมากใช้สำหรับสร้างการเชื่อมต่อโดยการส่งข้อความโดยตรง (DMs) เป็นจำนวนมาก—มีประโยชน์สำหรับการส่งเสริมการขาย การติดตามลูกค้า และการมีส่วนร่วมอีกครั้ง ระบบรองรับทั้งเทมเพลตที่สร้างไว้ล่วงหน้าและการสร้างแบบเรียลไทม์ และมุ่งเป้าเลียนแบบเวลาแบบมนุษย์เพื่อลดความเสี่ยงในการตรวจจับ

## การใช้ส่งข้อความโดยตรงจำนวนมากใน Super Marketing

1. เปิดสคริปต์ Super Marketing
2. เลือกประเภทแหล่งข้อมูล: "User account list (target list)"
3. วางชื่อผู้ใช้เป้าหมายเป็นจำนวนมากหรือนำเข้าไฟล์ `.txt` (หนึ่งชื่อผู้ใช้ต่อบรรทัด) หลังจากนำเข้าคุณสามารถเลือก "Append" หรือ "Replace" รายการปัจจุบัน
4. เลือกกลยุทธ์การใช้ข้อมูล:

   - Shared pool: บัญชีบอททั้งหมดจะส่งข้อความจากกลุ่มเป้าหมายเดียวกัน

   - Single-consumption: แต่ละบัญชีบอทประมวลผลเป้าหมายที่แตกต่างกัน ดังนั้นแต่ละเป้าหมายจะได้รับข้อความจากบัญชีเดียวเท่านั้น

5. ตั้งค่าจำนวนข้อความที่จะส่งต่อเป้าหมาย (โดยทั่วไป 1 เพื่อหลีกเลี่ยงการติดต่อซ้ำ)
6. กำหนดค่าการตั้งค่าข้อความ:

   - เทมเพลตที่สร้างไว้ล่วงหน้า: วางหลายบรรทัดหรือย่อหน้าของข้อความ DM ที่เตรียมไว้ แต่ละบรรทัด/ย่อหน้าถูกถือว่าเป็นข้อความผู้สมัคร

   - ลำดับข้อความ: เลือก "Random" หรือ "Sequential"

   - Random: แต่ละครั้งที่ส่งเลือกเทมเพลตแบบสุ่ม

   - Sequential: หมุนเวียนเทมเพลตตามลำดับที่วาง/นำเข้า (มีประโยชน์เมื่อคุณต้องการกลยุทธ์การยกเลิกการทำซ้ำที่ทำซ้ำได้)

   - แทรกอีโมจิ: เปิดใช้งานการแทรกอีโมจิอัตโนมัติเพื่อเพิ่มความเป็นธรรมชาติ

   - การสร้างข้อความแบบเรียลไทม์ (ทางเลือก, ChatGPT API):

   - เมื่อเปิดใช้งาน สคริปต์สามารถเรียก ChatGPT หรือ API ที่คล้ายกันเพื่อสร้างข้อความส่วนบุคคลตามชื่อผู้ใช้เป้าหมายหรือบริบทที่มี—เหมาะสมเมื่อคุณต้องการข้อความที่ไม่ซ้ำกันจำนวนมาก

   - หมายเหตุ: การเปิดใช้งานการสร้างแบบเรียลไทม์ต้องการ API key ที่ถูกต้องและคุณต้องพิจารณาข้อจำกัดอัตรา ค่าใช้จ่าย และความเป็นส่วนตัว (หลีกเลี่ยงการส่งข้อมูลที่ละเอียดอ่อนไปยัง APIs บุคคลที่สาม)

   - สองวิธีทั่วไปในการรับประกันข้อความที่ไม่ซ้ำกัน:

   1. ใช้การหมุนเวียนตามลำดับของเทมเพลตที่สร้างไว้ล่วงหน้า (แต่ละเทมเพลตจะถูกใช้ครั้งเดียวจนหมด)

   2. ใช้การสร้างแบบเรียลไทม์ของ ChatGPT เพื่อสร้างข้อความที่ไม่ซ้ำกันต่อเป้าหมาย

7. กำหนดค่าวิธีการเปิดโปรไฟล์ผู้ใช้ (หากงานต้องการดูโปรไฟล์): เลือก "Search" หรือ "Direct Open Profile"
8. กำหนดค่าช่วงเวลางาน (ต่ำสุด/สูงสุดในหน่วยนาที) การทำงานพร้อมกัน และการตั้งค่ากลุ่มบัญชี จากนั้นเริ่มสคริปต์

:::tip เกี่ยวกับช่วงเวลางาน
ตั้งค่าช่วงเวลาต่ำสุดและสูงสุดและเปิดใช้งานความล่าช้าแบบสุ่ม การกำหนดเวลาที่แตกต่างกันลดโอกาสของกิจกรรมที่ซิงโครไนซ์และลดความเสี่ยงในการตรวจจับ
:::

## คำแนะนำและหมายเหตุ

- รักษาเนื้อหา DM ให้สอดคล้องและหลากหลาย หลีกเลี่ยงการส่งข้อความที่เหมือนกันจำนวนมากในเวลาสั้น
- หากใช้การสร้างแบบเรียลไทม์ ติดตามการใช้งาน API และค่าใช้จ่ายอย่างใกล้ชิดและหลีกเลี่ยงการส่งข้อมูลที่ละเอียดอ่อน
- งานอาจล้มเหลว ตรวจสอบบันทึกและลองแถวที่ล้มเหลวอีกครั้ง เป้าหมายที่ประมวลผลแล้วจะถูกข้ามในการรันครั้งต่อไป
