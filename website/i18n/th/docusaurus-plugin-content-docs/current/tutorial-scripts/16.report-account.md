# รายงานบัญชี

:::warning การใช้งานอย่างรับผิดชอบ
การรายงานบัญชีเป็นการกระทำที่ทรงพลัง ใช้ฟีเจอร์รายงานเฉพาะสำหรับการละเมิดที่ถูกต้อง (การฉ้อโกง การหลอกลวง สแปม ฯลฯ) เท่านั้น การใช้ในทางที่ผิดอาจส่งผลให้มีข้อจำกัดบัญชีหรือผลกระทบที่ไม่พึงประสงค์อื่นๆ
:::

คู่มือรายงานบัญชีอธิบายวิธีการใช้ความสามารถในการรายงานแบบบิวท์อินที่เปิดเผยโดยสคริปต์ Super Marketing ฟีเจอร์นี้ทำให้กระบวนการรายงานในแอพที่กำหนดไว้เป็นอัตโนมัติเพื่อช่วยประมวลผลรายการบัญชีผู้ใช้ แต่ไม่รองรับการปรับแต่งเส้นทางรายงานหรือเหตุผลการรายงานนอกเหนือจากการเลือกที่กำหนดไว้ด้านล่าง

## ภาพรวมด่วน

- กระบวนการรายงานที่ใช้โดยสคริปต์เป็นแบบกำหนดไว้และไม่สามารถเปลี่ยนแปลงใน UI หรือสคริปต์:

```txt
Report → Report account → Something else → Frauds and scams
```

- สคริปต์พยายามนำทางโปรไฟล์ผู้ใช้ เปิดเมนูรายงาน เลือกเส้นทางที่กำหนดไว้ข้างต้น และส่งรายงานเมื่อเป็นไปได้
- เนื่องจาก UI บนมือถือแตกต่างกันระหว่างเวอร์ชันแอพและอุปกรณ์ สคริปต์ใช้ตัวเลือก UI แบบพยายามให้ดีที่สุด ทดสอบกับชุดเล็กก่อนเสมอ

## ขั้นตอนการรัน (ผ่าน Super Marketing)

1. เปิด TikMatrix/IgMatrix และเลือกอุปกรณ์ที่จะรัน (รองรับการเลือกหลายรายการ)
2. เปิด `Scripts` → `Super Marketing`
3. ตั้งค่า **แหล่งข้อมูล** เป็น `Usernames` และนำเข้าไฟล์ข้อความหรือวางชื่อผู้ใช้ (หนึ่งรายการต่อบรรทัด) หลังจากนำเข้าเลือก `Append` หรือ `Overwrite`
4. ใน **User Actions** เปิดใช้งาน **Report account** (สวิตช์ใหม่) และตรวจสอบเส้นทางรายงานที่กำหนดไว้ที่แสดง
5. กำหนดค่าการทำงานพร้อมกันของงาน ความล่าช้า และการตั้งค่างานอื่นๆ ตามต้องการ
6. คลิก `Start Script` เพื่อเริ่มประมวลผลรายการ ติดตามบันทึกและหน้าจออุปกรณ์สำหรับผลสำเร็จ/ล้มเหลว

## เคล็ดลับและคำแนะนำ

- ทดสอบกับ 1–5 บัญชีก่อนเพื่อยืนยันว่า UI รายงานถูกตรวจจับและลำดับทำงานบนเวอร์ชันแอพและอุปกรณ์เป้าหมายของคุณ
- รักษาการกำหนดจังหวะและความล่าช้าที่อนุรักษ์นิยม การรายงานอัตโนมัติที่หนักสามารถกระตุ้นการป้องกันแพลตฟอร์มหรือข้อจำกัดอัตรา
- ฟีเจอร์นี้ใช้เส้นทางในแอพที่กำหนดไว้ หาก UI แอพเปลี่ยนแปลง (ข้อความหรือตำแหน่งองค์ประกอบ) การทำงานอัตโนมัติอาจล้มเหลวจนกว่าจะอัปเดตตัวเลือก
- อย่าใช้การรายงานเพื่อรังแกหรือกำหนดเป้าหมายผู้ใช้ที่ไม่มีความผิด รายงานเฉพาะการละเมิดที่ชัดเจนที่สอดคล้องกับข้อกำหนดของแพลตฟอร์ม
- ตรวจสอบบันทึกหลังจากรัน สคริปต์จะบันทึกความล้มเหลวเช่น ปุ่มเมนูที่หายไปหรือตัวเลือกรายงานที่หายไป เพื่อให้คุณสามารถปรับเวลาหรือตัวเลือก

## เมื่อไหร่ไม่ควรใช้

- หากคุณต้องการเลือกเหตุผลการรายงานที่กำหนดเองหรือส่งหลักฐานโดยละเอียดต่อกรณี — สคริปต์ไม่รองรับระดับการปรับแต่งนั้น
- สำหรับเรื่องทางกฎหมายหรือที่ละเอียดอ่อนที่ต้องการการตรวจสอบด้วยตนเอง

## การแก้ไขปัญหา

- "ไม่พบตัวเลือกรายงาน" / "ไม่พบตัวเลือก 'รายงานบัญชี'": ข้อความหรือเค้าโครง UI มีแนวโน้มว่าเปลี่ยนแปลง รันอีกครั้งด้วยชุดทดสอบเล็กและจับภาพหน้าจออุปกรณ์สำหรับการดีบัก
- หากรายงานถูกส่งแต่ไม่มีผลใดๆ แพลตฟอร์มอาจเพิกเฉยต่อรายงานอัตโนมัติหรือใช้กฎการดูแล พิจารณาการตรวจสอบด้วยตนเองสำหรับกรณีที่สำคัญ
