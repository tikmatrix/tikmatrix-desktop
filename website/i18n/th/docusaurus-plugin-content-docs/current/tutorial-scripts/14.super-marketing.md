# Super Marketing

:::info
สคริปต์ Super Marketing รวมการติดตาม/เลิกติดตาม DM การโต้ตอบโพสต์ และความคิดเห็นลงในคิวงานเดียว เหมาะสำหรับการเข้าถึงลูกค้าที่มีศักยภาพหรือการมีส่วนร่วมกับผู้ติดตามอีกครั้งในการดำเนินการเดียว
:::

## ขั้นตอนการรัน

1. ใน TikMatrix/IgMatrix เลือกอุปกรณ์ที่คุณต้องการเรียกใช้สคริปต์ (รองรับการเลือกหลายรายการ)
2. เปิด `Scripts` → `Super Marketing`
3. เลือก **แหล่งข้อมูล** (ชื่อผู้ใช้หรือลิงก์โพสต์) จากนั้นคลิก `Import txt file` เพื่อโหลดไฟล์ `.txt` หรือวางหลายบรรทัดโดยตรง หลังจากนำเข้า เลือก `Append` หรือ `Overwrite`
4. เปิดใช้งาน User Actions, Post Actions และโมดูล Comment ตามต้องการ และกำหนดค่าตามคำแนะนำด้านล่าง
5. ตั้งค่าแถบเลื่อน **ช่วงเวลางาน** เพื่อกำหนดความล่าช้าแบบสุ่มระหว่างเป้าหมาย จากนั้นคลิก `Start Script` เพื่อดำเนินการงาน

## การกำหนดค่า

### แหล่งข้อมูล

- **ชื่อผู้ใช้ / ลิงก์โพสต์**: กำหนดคุณสมบัติที่มีอยู่ การติดตาม เลิกติดตาม และ DM สามารถใช้ได้เฉพาะกับชื่อผู้ใช้เท่านั้น หากคุณต้องการโต้ตอบกับโพสต์เท่านั้น ให้เลือกโหมดลิงก์
- **วิธีการนำเข้า**: คลิก `Import txt file` เพื่อโหลดไฟล์ `.txt` หรือวางหลายบรรทัดโดยตรง หลังจากนำเข้า เลือก `Append` หรือ `Overwrite`

### กลยุทธ์การใช้ข้อมูล

- **Shared pool**: บัญชีบอททั้งหมดดึงงานจากกลุ่มเป้าหมายเดียวกัน เหมาะสำหรับสถานการณ์ที่หลายบัญชีต้องโต้ตอบกับเป้าหมายเดียวกันหลายครั้ง (เช่น การผลักดันแบบเข้มข้นในโพสต์ยอดนิยม)

- **Single-consumption**: แต่ละบัญชีบอทประมวลผลเพียงเป้าหมายที่แตกต่างกันเท่านั้น เพื่อให้แน่ใจว่าแต่ละเป้าหมายจะถูกจัดการโดยบัญชีเดียวครั้งเดียว เหมาะสำหรับสถานการณ์ที่ต้องหลีกเลี่ยงการเข้าถึงซ้ำ (เช่น DM จำนวนมากหรือความคิดเห็นที่ไม่ซ้ำกันเดียว)

  - **ขนาดจำกัดการใช้ต่อการทำงาน**: เมื่อใช้โหมด single-consumption คุณสามารถตั้งค่าจำนวนเป้าหมายสูงสุดที่แต่ละการรันงานจะใช้ สิ่งนี้ป้องกันไม่ให้ชุดงานแรกใช้ข้อมูลทั้งหมด เพื่อให้แน่ใจว่าการหมุนเวียนบัญชีในภายหลังมีเป้าหมายที่จะประมวลผล ตั้งค่าเป็น 0 สำหรับการใช้ไม่จำกัด (พฤติกรรมเริ่มต้น)

เลือกกลยุทธ์ตามเป้าหมายและการพิจารณาการจัดการความเสี่ยงของคุณ มันส่งผลต่อการกระจายคิวและพฤติกรรมการทำงานพร้อมกัน

### การกระทำผู้ใช้

- **ติดตามผู้ใช้**: ดำเนินการติดตามก่อนเมื่อเข้าหน้าโปรไฟล์
- **เลิกติดตามผู้ใช้**: เลิกติดตามผู้ใช้ สามารถรันแบบสแตนด์อโลนหรือล้างผู้ติดตามหลังจาก DM ไม่แนะนำให้เปิดใช้งานพร้อมกับ "ติดตาม" พร้อมกัน
- **รายงานบัญชี**: รายงานบัญชีผู้ใช้สำหรับพฤติกรรมที่ฉ้อโกง ฟีเจอร์นี้ใช้เส้นทางการรายงานที่กำหนดไว้และไม่รองรับเส้นทางที่กำหนดเอง

  :::warning เส้นทางรายงาน (กำหนดไว้)
  Report → Report account → Something else → Frauds and scams
  :::

  - หมายเหตุ: ฟีเจอร์นี้ใช้เส้นทางรายงานที่กำหนดไว้ล่วงหน้า ไม่รองรับเหตุผลการรายงานหรือเส้นทางที่กำหนดเอง
  - ใช้อย่างรับผิดชอบและเฉพาะวัตถุประสงค์ที่ถูกต้องเท่านั้น การใช้ในทางที่ผิดอาจส่งผลให้มีข้อจำกัดบัญชี

- **ส่ง DM**: เลือกข้อความแบบสุ่มจากเทมเพลตหลายบรรทัด เปิดใช้งาน "แทรกอีโมจิ" เพื่อผนวกอีโมจิธรรมชาติโดยอัตโนมัติ สคริปต์จะตรวจจับว่าผู้รับอนุญาต DMs หรือไม่และข้ามหากไม่มี

  - เทมเพลต DM (ที่สร้างไว้ล่วงหน้า): ใน "การตั้งค่าข้อความ" วางเทมเพลต DM หลายบรรทัดหรือหลายย่อหน้า (แต่ละบรรทัด/ย่อหน้าเป็นข้อความผู้สมัคร)

  - ลำดับการใช้เทมเพลต: เลือก "Random" หรือ "Sequential"

    - Random: เลือกเทมเพลตแบบสุ่มแต่ละครั้ง

    - Sequential: หมุนเวียนเทมเพลตตามลำดับที่วาง/นำเข้า เหมาะสำหรับกลยุทธ์การยกเลิกการทำซ้ำที่ทำซ้ำได้

  - แทรกอีโมจิ: เปิดใช้งานเพื่อแทรกอีโมจิลงในข้อความหรือความคิดเห็นโดยอัตโนมัติเพื่อเพิ่มความเป็นธรรมชาติ

  - การสร้างแบบเรียลไทม์ (ทางเลือก, ChatGPT หรือ API ที่เข้ากันได้):

    - เมื่อเปิดใช้งาน สคริปต์สามารถเรียกจุดสิ้นสุดการสร้างที่กำหนดค่าของคุณ (กรอก API Key / URL / model / System Prompt) เพื่อสร้างข้อความส่วนบุคคลตามบริบทเป้าหมาย

    - หมายเหตุ: ให้ API Key ที่ถูกต้องและตระหนักถึงข้อจำกัดอัตรา ค่าใช้จ่าย และการปฏิบัติตามความเป็นส่วนตัว (หลีกเลี่ยงการส่งข้อมูลที่ละเอียดอ่อน)

  - กลยุทธ์ความเป็นเอกลักษณ์: วิธีทั่วไปในการรับประกันว่าแต่ละข้อความ/ความคิดเห็นมีเอกลักษณ์:

      1. ใช้โหมด "Sequential" เพื่อหมุนเวียนเทมเพลตที่สร้างไว้ล่วงหน้า (เทมเพลตที่ใช้เพียงครั้งเดียวจนหมด)
      2. ใช้การสร้างแบบเรียลไทม์ (ChatGPT) เพื่อสร้างข้อความที่ไม่ซ้ำกันสำหรับแต่ละเป้าหมาย

  - ก่อนใช้ DM จำนวนมาก ตรวจสอบให้แน่ใจว่าทดสอบด้วยชุดเล็กและให้ความสำคัญกับความเป็นส่วนตัวและข้อกำหนดการปฏิบัติตาม

### การกระทำโพสต์

- **โพสต์สูงสุดที่จะประมวลผล**: กำหนดจำนวนโพสต์ที่แต่ละบัญชีจะประมวลผล (1–50) จำเป็นเฉพาะเมื่อแหล่งข้อมูลคือ "ชื่อผู้ใช้"
- **จำนวนครั้งที่ทำซ้ำ**: ทำซ้ำลูปการโต้ตอบโพสต์หลายครั้งโดยไม่ต้องรีสตาร์ทสคริปต์
- **ระยะเวลาการดู**: ตั้งค่าเวลาดูต่ำสุดและสูงสุด (0–300 วินาที) สคริปต์จะเลือกค่าแบบสุ่มภายในช่วงก่อนทำการโต้ตอบ
- **การโต้ตอบโพสต์**: สามารถเปิดใช้งาน Like, Favorite, Repost และ Share แยกกัน เมื่อเปิดใช้งานหลายการกระทำ ระบบจะกระจายในจุดเวลาที่แตกต่างกันระหว่างช่วงการดูเพื่อพฤติกรรมที่เป็นธรรมชาติมากขึ้น

- **ความคิดเห็น**: เมื่อเปิดใช้งาน คุณสามารถโพสต์ความคิดเห็นในโพสต์

     หาก "ความคิดเห็น" เปิดใช้งาน กำหนดค่าต่อไปนี้ในการตั้งค่าความคิดเห็น:

  - เทมเพลตความคิดเห็นที่สร้างไว้ล่วงหน้า: วางเทมเพลตความคิดเห็นที่สร้างไว้ล่วงหน้าหลายบรรทัดหรือหลายย่อหน้า (แต่ละบรรทัดหรือย่อหน้าเป็นความคิดเห็นผู้สมัคร)

  - ลำดับการใช้เทมเพลต: เลือก "Random" หรือ "Sequential"
    - Random: เลือกแบบสุ่มจากเทมเพลตผู้สมัครแต่ละครั้ง
    - Sequential: ใช้เทมเพลตตามลำดับที่วาง/นำเข้า (เหมาะสำหรับกลยุทธ์การยกเลิกการทำซ้ำที่ทำซ้ำได้)

  - แทรกอีโมจิ: เปิดใช้งานเพื่อแทรกอีโมจิลงในความคิดเห็นโดยอัตโนมัติเพื่อเพิ่มความเป็นธรรมชาติ

  - การสร้างความคิดเห็นแบบเรียลไทม์ (ทางเลือก, ChatGPT API):
    - เมื่อเปิดใช้งาน สคริปต์สามารถเรียก APIs คล้าย ChatGPT เพื่อสร้างข้อความความคิดเห็นที่เกี่ยวข้องกับบริบทและไม่ซ้ำกันตามหัวเรื่อง/คำอธิบายโพสต์แบบเรียลไทม์ วิธีนี้สามารถลดความเสี่ยงของความคิดเห็นซ้ำได้อย่างมาก เหมาะสำหรับสถานการณ์ที่ต้องการความคิดเห็นที่ไม่ซ้ำกันจำนวนมาก
    - หมายเหตุ: การใช้การสร้างแบบเรียลไทม์ต้องการ API Key ที่ถูกต้อง และตระหนักถึงข้อจำกัดอัตรา ค่าใช้จ่าย และการปฏิบัติตามความเป็นส่วนตัว (อย่าส่งข้อมูลที่ละเอียดอ่อนไปยังบริการบุคคลที่สาม)

  - สองวิธีทั่วไปในการรับประกันความคิดเห็นที่ไม่ซ้ำกัน:
         1. ใช้ "Sequential" เพื่อโหลดและหมุนเวียนเทมเพลตความคิดเห็นที่สร้างไว้ล่วงหน้า (รับประกันว่าแต่ละเทมเพลตจะถูกใช้เพียงครั้งเดียวจนหมด)
         2. ใช้ ChatGPT API เพื่อสร้างแต่ละความคิดเห็นแบบเรียลไทม์ (สร้างแบบไดนามิกตามหัวเรื่อง/เนื้อหาโพสต์ เป็นธรรมชาติง่ายกว่าในการรับประกันความเป็นเอกลักษณ์)

  - คำแนะนำ: ทดสอบกลยุทธ์ความคิดเห็นด้วยชุดเล็กก่อน ตรวจสอบคุณภาพความคิดเห็นและการจัดการแพลตฟอร์ม หากใช้การสร้างแบบเรียลไทม์ ติดตามปริมาณการเรียก API และค่าใช้จ่าย

  - หากใช้กลยุทธ์ "Sequential" ระบบจะรวมหมายเลขงานเพื่อหมุนเวียนเทมเพลตในหลายอุปกรณ์ ลดความน่าจะเป็นของการทำซ้ำ หากใช้การสร้าง AI แนะนำให้ใช้กฎตรวจสอบคุณภาพง่ายๆ (เช่น ช่วงความยาว ห้าม URLs / คำที่ละเอียดอ่อน)

### รวมงาน (รวมงานสำหรับชื่อผู้ใช้เดียวกัน)

- คำอธิบาย: เมื่อ "รวมงานสำหรับชื่อผู้ใช้เดียวกัน" เปิดใช้งาน เป้าหมายหลายรายการสำหรับชื่อผู้ใช้เดียวกันจะถูกดำเนินการภายในหน่วยงานเดียว (ตามลำดับหรือแบบขนานขึ้นอยู่กับการใช้งาน) แม้ว่าเป้าหมายหนึ่งล้มเหลว งานโดยรวมจะไม่ถูกยกเลิกและเป้าหมายที่เหลือจะดำเนินการต่อไป ลดภาระการสลับและการกำหนดเวลางาน

- พฤติกรรมเมื่อเปิดใช้งาน: เป้าหมายหลายรายการที่มุ่งไปที่บัญชีเดียวกันจะถูกจัดกลุ่มเป็นหน่วยงานเดียว ความล้มเหลวส่งผลกระทบเฉพาะเป้าหมายแต่ละรายการและถูกบันทึก แต่จะไม่บล็อกเป้าหมายอื่นจากการดำเนินการต่อ โหมดนี้เหมาะสำหรับการรันการตลาดขนาดใหญ่และสามารถลดภาระการเริ่มต้น/สลับงานได้อย่างมาก

- พฤติกรรมเมื่อปิดใช้งาน: แต่ละเป้าหมายถูกสร้างเป็นงานแต่ละรายการ ความล้มเหลวใดๆ จะถูกรายงานทันทีและหยุดงานย่อยนั้น (โดยไม่ส่งผลกระทบต่องานอื่นที่เป็นอิสระ) โหมดนี้เหมาะสมเมื่อคุณต้องการการติดตามสถานะการดำเนินการที่แม่นยำต่อเป้าหมายและต้องการการกระทำทันทีต่อความล้มเหลว (เช่น การแทรกแซงของมนุษย์หรือการย้อนกลับทันที)

- คำแนะนำ:
  - เปิดใช้งานโหมดนี้สำหรับการตลาดปริมาณสูงและชุดใหญ่เพื่อลดต้นทุนการกำหนดเวลา
  - ปิดใช้งานเมื่อคุณต้องการสถานะการดำเนินการที่แม่นยำต่อเป้าหมายและต้องการการกระทำทันทีต่อความล้มเหลว (เช่น การแทรกแซงของมนุษย์หรือการย้อนกลับทันที)

### ช่วงเวลางาน

- ตั้งค่าค่าต่ำสุดและสูงสุด (ในหน่วยนาที) สคริปต์จะรอแบบสุ่มภายในช่วงระหว่างแต่ละเป้าหมาย บวกกับการหยุดสั้นๆ ของตัวเอง

## ขั้นตอนการดำเนินการ

- หากเปิดใช้งานโหมดหลายบัญชี TikMatrix/IgMatrix จะเปิด TikTok/Instagram ก่อนและสลับไปยังบัญชีที่สอดคล้องกัน
- ในโหมดชื่อผู้ใช้ ระบบเปิดแต่ละหน้าโปรไฟล์โดยใช้วิธีการเข้าถึงที่เลือก รอให้โหลดเสร็จสมบูรณ์ จากนั้นดำเนินการติดตาม DM การกระทำเลิกติดตามตามลำดับ หากผู้รับบล็อก DMs จะข้ามโดยอัตโนมัติ
- การโต้ตอบโพสต์จะเลื่อนผ่านหน้าโปรไฟล์หรือเปิดลิงก์โดยตรง ดูตามจำนวนและระยะเวลาที่ตั้งไว้ และทำการกระทำ like, favorite, repost, share ข้ามโฆษณา การถ่ายทอดสด และเนื้อหาที่ถูกใจไปแล้ว
- เมื่อใช้ลิงก์โพสต์ สคริปต์จะเปิดลิงก์ทีละรายการ ดำเนินการกระทำที่กำหนดค่าไว้ และวนซ้ำตาม "จำนวนครั้งที่ทำซ้ำ"
- โมดูลความคิดเห็นจะดึงจากเทมเพลตหรือเอาต์พุต AI โหมด Sequential จะรวมหมายเลขงานเพื่อกระจายความคิดเห็นที่แตกต่างกันในหลายอุปกรณ์
- ระหว่างแต่ละเป้าหมาย สคริปต์จะเพิ่มการหยุดสุ่มเล็กน้อย จากนั้นเพิ่มการรอแบบสุ่มจากช่วงเวลางานเพื่อจำลองจังหวะของมนุษย์

## แนวทางปฏิบัติที่ดีที่สุด

- ตรวจสอบข้อความและการกำหนดจังหวะด้วยชุดเล็กก่อน จากนั้นขยายขนาดหลังจากยืนยันว่าไม่มีปัญหา
- การรวมหลายการกระทำ (เช่น ติดตาม + DM + ถูกใจ) สามารถปรับปรุงสัญญาณการมีส่วนร่วมของบัญชี
- รักษา DMs และความคิดเห็นให้เป็นการสนทนาและจริงใจเพื่อหลีกเลี่ยงการกระตุ้นการดูแลแพลตฟอร์ม
- อัปเดตไลบรารีความคิดเห็นเป็นประจำหรือปรับ AI System Prompt เพื่อให้เนื้อหามีความเกี่ยวข้องกับธีมแคมเปญมากขึ้น
- ติดตามข้อจำกัดอัตราของแพลตฟอร์ม สำหรับปริมาณใหญ่ ให้หมุนเวียนบัญชีหรือโหนด proxy
