# Pesan Massal

:::warning Pemberitahuan Penghentian
Fitur ini telah diintegrasikan ke [skrip Super Marketing](./14.super-marketing.md). Silakan prioritaskan penggunaan skrip "Super Marketing" untuk kemampuan pesan massal dan manajemen pesan yang lebih kuat dan fleksibel.
:::

Fitur Pesan Massal digunakan untuk menjalin kontak dengan user target melalui pengiriman pesan pribadi (DM) secara massal, cocok untuk promosi, customer service follow-up, dan reaktivasi user. Sistem mendukung template yang telah digenerate dan generate real-time sebagai dua sumber pesan, dan akan berusaha mensimulasikan pengiriman manual untuk mengurangi risiko deteksi.

## Menggunakan Fitur Pesan Massal dalam Skrip "Super Marketing"

1. Buka skrip: Pilih "Super Marketing".
2. Pilih tipe sumber data sebagai "Daftar Akun User (Daftar Target)".
3. Paste username target secara massal atau impor dari file teks (satu per baris); setelah impor dapat memilih "Tambahkan" atau "Timpa".
4. Pilih strategi penggunaan data:

    - Strategi Shared Pool: Semua akun bot mengirim pesan pribadi ke daftar target yang sama.

    - Strategi Single-consumption: Setiap akun bot hanya memproses target yang berbeda, memastikan setiap target hanya menerima pesan dari satu akun sekali.

5. Atur jumlah pengiriman pesan untuk setiap target (biasanya 1 kali, untuk menghindari gangguan berulang).
6. Dalam "Pengaturan Pesan" konfigurasi konten pesan pribadi dan strategi pengiriman:

    - Template pesan yang telah digenerate: Dapat paste teks pesan pribadi yang telah digenerate dengan beberapa baris atau beberapa bagian (setiap baris atau setiap bagian dianggap sebagai satu kandidat pesan).

    - Urutan penggunaan teks: Pilih "Acak" atau "Berurutan".

        - Acak: Setiap kali acak mengambil satu template untuk dikirim.

        - Berurutan: Rotasi penggunaan template sesuai urutan paste/impor (cocok untuk strategi deduplikasi yang dapat direproduksi).

    - Sisipkan emoji: Dapat mengaktifkan untuk secara otomatis menyisipkan emoji dalam pesan untuk meningkatkan kealamian.

    - Generate pesan secara real-time (opsional, ChatGPT API):

        - Setelah diaktifkan, skrip dapat memanggil ChatGPT atau API serupa, generate pesan yang dipersonalisasi berdasarkan username target atau konteks yang tersedia secara real-time, cocok untuk skenario yang memerlukan banyak pesan unik.

        - Catatan: Mengaktifkan generate real-time memerlukan API Key yang valid, dan perhatikan pembatasan rate, biaya, dan kepatuhan privasi (hindari mengirim data sensitif).

    - Dua cara umum untuk menjamin pesan unik:

        1. Gunakan "Berurutan" untuk merotasi template yang telah digenerate (setiap template hanya digunakan sekali hingga habis).

        2. Gunakan ChatGPT untuk generate setiap pesan secara real-time (generate dinamis berdasarkan informasi target, keunikan lebih tinggi).

7. Konfigurasi cara membuka user (jika perlu melihat profil user): Pilih "Cari user" atau "Buka profil langsung (Direct Open Profile)".
8. Konfigurasi interval tugas (minimum/maksimum, satuan: menit) dan konkurensi, pool akun, dll., setelah konfirmasi tidak ada masalah jalankan skrip.

:::tip Tentang Interval Tugas
Disarankan untuk mengatur interval minimum/maksimum dan mengaktifkan interval acak, untuk menghindari semua perangkat mengeksekusi secara bersamaan yang menyebabkan puncak abnormal, sehingga mengurangi risiko deteksi atau pembatasan oleh platform.
:::

## Saran Penggunaan dan Catatan

- Konten pesan pribadi harus patuh dan beragam, hindari mengirim banyak konten yang sama dalam waktu singkat.
- Jika menggunakan generate real-time, harap monitor jumlah pemanggilan API dan biaya dengan ketat, dan pastikan tidak membocorkan informasi sensitif ke luar.
- Tugas mungkin gagal; jika gagal silakan coba lagi atau lihat log, sistem akan secara otomatis melewati target yang telah selesai.
