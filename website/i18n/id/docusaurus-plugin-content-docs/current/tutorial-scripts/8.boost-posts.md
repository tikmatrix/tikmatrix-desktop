# Boost Postingan

:::warning Pemberitahuan Penghentian
Fitur skrip ini telah diintegrasikan ke [skrip Super Marketing](./14.super-marketing.md). Silakan prioritaskan penggunaan skrip "Super Marketing" untuk kemampuan boost postingan yang lebih kuat dan fleksibel.
:::

Fitur Boost Postingan dapat meningkatkan jumlah interaksi postingan melalui operasi like, share, favorit, repost, dan menonton secara massal. Sistem akan berusaha mensimulasikan perilaku user nyata untuk mengurangi risiko deteksi, tetapi saat menggunakan tetap harus mengikuti aturan platform dan mengontrol frekuensi dengan tepat.

## Menggunakan Fitur Boost Postingan dalam Skrip "Super Marketing"

1. Buka skrip: Pilih "Super Marketing".
2. Pilih tipe sumber data sebagai "Daftar Link Postingan".
3. Paste link postingan secara massal atau impor dari file teks; setelah impor dapat memilih "Tambahkan" atau "Timpa" daftar saat ini.
4. Pilih strategi penggunaan data:

    - Strategi Shared Pool: Semua akun bot melakukan interaksi pada postingan yang sama.

    - Strategi Single-consumption: Setiap akun bot hanya melakukan satu kali interaksi pada postingan yang berbeda, memastikan setiap postingan hanya diproses sekali oleh satu akun.

5. Dalam "Operasi Postingan â€” Jumlah Pengulangan" atur jumlah interaksi untuk setiap postingan. Jika termasuk komentar, disarankan mengatur jumlah komentar menjadi 1, untuk menghindari komentar berulang.
6. Dalam "Operasi Postingan" pilih jenis interaksi yang diperlukan: like, share, repost, favorit, komentar, menonton video, dll.

     Jika memilih "Komentar", silakan lakukan konfigurasi berikut dalam pengaturan komentar:

     - Template komentar yang telah digenerate: Dapat paste teks komentar yang telah digenerate dengan beberapa baris atau beberapa bagian (setiap baris atau setiap bagian dianggap sebagai satu kandidat komentar).

     - Urutan penggunaan teks: Pilih "Acak" atau "Berurutan".
         - Acak: Setiap kali acak mengambil satu teks dari kandidat.
         - Berurutan: Gunakan sesuai urutan paste/impor (cocok untuk strategi deduplikasi yang dapat direproduksi).

     - Sisipkan emoji: Dapat mengaktifkan untuk secara otomatis menyisipkan emoji dalam komentar untuk meningkatkan kealamian.

     - Generate komentar secara real-time (opsional, ChatGPT API):
         - Setelah diaktifkan, skrip dapat memanggil ChatGPT atau API serupa, generate teks komentar yang relevan dengan konteks dan unik berdasarkan judul/deskripsi postingan secara real-time. Cara ini dapat secara signifikan mengurangi risiko komentar berulang, cocok untuk skenario yang memerlukan banyak komentar unik.
         - Catatan: Menggunakan generate real-time memerlukan API Key yang tersedia, dan perhatikan pembatasan rate, biaya, dan kepatuhan privasi (jangan kirim data sensitif ke layanan pihak ketiga).

     - Dua cara umum untuk menjamin komentar unik:
         1. Gunakan "Berurutan" untuk memuat dan merotasi template komentar yang telah digenerate (menjamin setiap template hanya digunakan sekali hingga habis).
         2. Gunakan ChatGPT API untuk generate setiap komentar secara real-time (generate dinamis berdasarkan judul/konten postingan, secara alami lebih mudah menjamin keunikan).

     - Saran: Pertama uji strategi komentar dengan sejumlah kecil postingan, periksa kualitas komentar dan situasi pemrosesan platform; jika menggunakan generate real-time, silakan monitor jumlah pemanggilan API dan biaya.
7. Konfigurasi parameter skrip lain (seperti jumlah konkurensi, delay, pool akun, dll.), setelah konfirmasi tidak ada masalah jalankan skrip.

## Saran Penggunaan dan Catatan

- Disarankan untuk menguji terlebih dahulu pada sejumlah kecil postingan, konfirmasi strategi dan parameter tidak akan memicu kontrol risiko platform.
- Konten komentar harus beragam dan tetap alami, hindari copy paste teks yang sama.
- Atur delay dan konkurensi dengan wajar, simulasikan perilaku manual untuk mengurangi risiko.
- Saat menjalankan jangka panjang silakan cek kesehatan akun dan log tugas secara berkala, dan sesuaikan strategi berdasarkan situasi.
