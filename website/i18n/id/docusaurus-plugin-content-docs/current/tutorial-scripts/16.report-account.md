# Laporkan Akun

:::warning Gunakan dengan Bertanggung Jawab
Fitur pelaporan adalah operasi yang kuat. Hanya gunakan ketika dipastikan ada pelanggaran hukum atau aturan (penipuan, scam, spam, dll.). Penyalahgunaan pelaporan dapat menyebabkan pembatasan akun atau konsekuensi buruk lainnya.
:::

Panduan ini menjelaskan cara menggunakan fitur "Laporkan Akun" bawaan dalam skrip Super Marketing. Fitur ini akan secara otomatis mengeksekusi proses pelaporan in-app tetap untuk memproses batch akun user, tetapi tidak mendukung kustomisasi path atau alasan pelaporan dalam UI atau skrip.

## Ikhtisar Cepat

- Proses pelaporan tetap yang digunakan skrip adalah sebagai berikut (tidak dapat dimodifikasi):

```text
Laporkan → Laporkan akun → Something else → Frauds and scams
```

- Skrip akan mencoba masuk ke halaman profil user target, membuka item lebih banyak/menu, memilih pelaporan sesuai path tetap dan submit (jika tersedia).
- Karena aplikasi mobile pada versi dan perangkat yang berbeda UI mungkin berbeda, skrip menggunakan selector best-effort, jadi harap validasi terlebih dahulu pada batch kecil.

## Langkah-langkah Eksekusi Melalui Super Marketing

1. Buka TikMatrix/IgMatrix dan pilih perangkat yang akan dijalankan (mendukung multi-select).
2. Buka `Scripts` → `Super Marketing`.
3. Atur **Data source** ke `Usernames`, impor file teks atau paste username (satu per baris); setelah impor pilih "Tambahkan" atau "Timpa".
4. Dalam **User Actions** aktifkan toggle **Report account**, dan konfirmasi path pelaporan tetap yang ditampilkan.
5. Konfigurasi konkurensi tugas, delay, dan parameter lainnya.
6. Klik `Start Script` untuk mulai memproses daftar, monitor log dan antarmuka perangkat untuk mengkonfirmasi operasi berhasil atau gagal.

## Saran dan Catatan

- Pertama uji pada 1–5 akun, konfirmasi skrip dapat mengenali dan mengeksekusi proses pelaporan dengan benar (perubahan teks atau posisi UI akan menyebabkan kegagalan).
- Pertahankan ritme dan delay yang konservatif, pelaporan otomatis batch mungkin memicu pembatasan atau kontrol risiko platform.
- Jika UI aplikasi berubah (teks tombol, perubahan layout), skrip mungkin tidak dapat menemukan elemen yang sesuai, perlu memperbarui selector atau menyesuaikan strategi tunggu.
- Hanya gunakan pelaporan untuk konten yang jelas melanggar, hindari pelecehan atau penyalahgunaan fitur ini terhadap user yang tidak bersalah.
- Setelah berjalan periksa log, skrip akan mencatat error seperti "Tidak dapat menemukan tombol menu" atau "Tidak dapat menemukan opsi pelaporan", memudahkan Anda menyesuaikan skrip atau parameter run.

## Skenario yang Tidak Disarankan untuk Penggunaan

- Skenario yang perlu memilih alasan pelaporan khusus untuk setiap kasus atau mengunggah bukti: Skrip tidak mendukung kustomisasi granular.
- Situasi hukum/sensitif yang memerlukan review manual atau preservasi bukti.

## Troubleshooting

- Error "Could not find Report option" / "Could not find 'Report account' option": Menunjukkan teks atau layout UI aplikasi telah berubah. Silakan jalankan lagi dalam batch kecil dan kumpulkan screenshot perangkat untuk debugging.
- Jika skrip menunjukkan telah submit pelaporan tetapi platform tidak benar-benar memproses, platform mungkin menurunkan prioritas pelaporan otomatis atau memerlukan review manual; untuk kasus individual penting, silakan gunakan proses manual.
