# Super Marketing

:::info
Skrip Super Marketing mengintegrasikan follow/unfollow, pesan pribadi, interaksi postingan, dan komentar ke dalam satu antrian tugas, cocok untuk menjangkau prospek atau reaktivasi pengikut lama dengan cepat dalam satu putaran operasi.
:::

## Langkah-langkah Operasi

1. Di TikMatrix/IgMatrix pilih perangkat yang perlu mengeksekusi skrip (dapat memilih beberapa).
2. Buka `Skrip` → `Super Marketing`.
3. Pilih **Sumber Data** (username atau link postingan), kemudian dapat klik tombol `Impor file txt` untuk memuat file `.txt` yang sesuai atau langsung paste data multi-baris; setelah impor dapat memilih "Tambahkan" atau "Timpa".
4. Berdasarkan kebutuhan aktifkan modul aksi user, aksi postingan, dan komentar, dan lengkapi konfigurasi sesuai penjelasan di bawah.
5. Atur slider **Interval Tugas** untuk mendefinisikan delay acak antara target, terakhir klik `Mulai Skrip` untuk mengeksekusi tugas.

## Penjelasan Konfigurasi

### Sumber Data

- **Username / Link Postingan**: Menentukan fitur mana yang tersedia. Follow, unfollow, pesan pribadi hanya dapat digunakan dengan username; jika hanya ingin mengoperasikan postingan, dapat langsung memilih mode link.
- **Cara Impor**: Dapat klik tombol `Impor file txt` untuk memuat file `.txt` yang sesuai atau langsung paste data multi-baris; setelah impor dapat memilih "Tambahkan" atau "Timpa".

### Strategi Penggunaan Data

- **Shared Pool**: Semua akun bot mengambil tugas dari pool target yang sama, cocok untuk skenario di mana beberapa akun menghasilkan beberapa interaksi pada target yang sama (misalnya mendorong postingan populer secara terpusat).

- **Single-consumption**: Setiap akun bot hanya memproses target yang berbeda, memastikan setiap target hanya diproses sekali oleh satu akun, cocok untuk skenario yang perlu menghindari jangkauan berulang (misalnya pesan massal atau satu putaran komentar unik).

  - **Jumlah Konsumsi Maksimum Per Tugas**: Dalam mode single-consumption, dapat mengatur jumlah maksimum target yang dikonsumsi setiap kali tugas berjalan. Ini dapat mencegah batch tugas pertama mengonsumsi semua data, memastikan masih ada target yang tersedia saat rotasi akun berikutnya. Atur ke 0 berarti tidak ada batasan (perilaku default).

Pilihan strategi akan mempengaruhi alokasi antrian dan perilaku konkurensi, silakan pilih berdasarkan target dan pertimbangan kontrol risiko.

### Aksi User

- **Follow User**: Prioritaskan eksekusi operasi follow setelah masuk ke halaman profil.
- **Unfollow User**: Berhenti follow, dapat berjalan sendiri atau membersihkan pengikut setelah pesan pribadi; tidak disarankan untuk mengaktifkan bersamaan dengan "Follow".
- **Laporkan Akun**: Laporkan akun user karena penipuan. Fitur ini menggunakan path pelaporan tetap, tidak mendukung path khusus.

  :::warning Path Pelaporan (Tetap)
  Laporkan → Laporkan akun → Something else → Frauds and scams
  :::

  - Penjelasan: Fitur ini menggunakan path pelaporan yang telah ditentukan, tidak mendukung alasan atau path pelaporan khusus.
  - Harap gunakan fitur ini dengan bertanggung jawab, hanya untuk tujuan yang sah. Penyalahgunaan dapat menyebabkan pembatasan akun.
- **Kirim Pesan Pribadi**: Acak mengambil satu pesan dari beberapa baris untuk dikirim; centang "Tambah emoji otomatis" untuk secara otomatis melampirkan emoji alami. Skrip akan mendeteksi apakah pihak lain mengizinkan pesan pribadi, jika tidak dapat dikirim akan dilewati.

  - Template pesan pribadi (pre-generated): Dalam "Pengaturan Pesan" paste teks pesan pribadi multi-baris atau multi-bagian (setiap baris/bagian dianggap sebagai satu kandidat pesan).

  - Urutan penggunaan teks: Pilih "Random" atau "Sequential".

    - Random: Setiap kali acak mengambil satu template untuk dikirim.

    - Sequential: Rotasi penggunaan sesuai urutan paste/impor, cocok untuk strategi deduplikasi yang dapat direproduksi.

  - Sisipkan emoji: Dapat mengaktifkan untuk secara otomatis menyisipkan emoji dalam pesan atau komentar untuk meningkatkan kealamian.

  - Generate real-time (opsional, ChatGPT atau API kompatibel):

    - Setelah diaktifkan, skrip dapat memanggil antarmuka generate yang Anda konfigurasi (isi API Key / URL / Model / System Prompt) untuk generate teks yang dipersonalisasi berdasarkan konteks target.

    - Catatan: Perlu menyediakan API Key yang valid, dan perhatikan pembatasan rate, biaya, dan kepatuhan privasi (hindari mengirim data sensitif).

  - Strategi keunikan: Cara umum untuk menjamin setiap pesan/komentar unik:

      1. Gunakan "Sequential" untuk merotasi template pre-generated (template hanya digunakan sekali hingga habis).
      2. Gunakan generate real-time (ChatGPT) untuk generate teks unik untuk setiap target.
  - Sebelum menggunakan pesan massal, harap uji batch kecil dan perhatikan privasi serta persyaratan kepatuhan.

### Aksi Postingan

- **Jumlah postingan yang dilewati**: Jumlah postingan yang dilewati sebelum pemrosesan (0–8). Ini memungkinkan Anda melewati beberapa postingan pertama di profil pengguna sebelum mulai berinteraksi dengan konten mereka. Hanya diperlukan saat sumber data adalah "Username".
- **Jumlah Postingan yang Diproses**: Menentukan jumlah postingan yang akan dioperasikan oleh setiap akun (1–50), hanya diperlukan saat sumber data adalah "Username".
- **Jumlah Pengulangan**: Ulangi eksekusi loop interaksi postingan beberapa kali, dapat mengulang operasi pada postingan tanpa perlu memulai ulang skrip.
- **Durasi Menonton**: Atur waktu menonton minimum dan maksimum (0–300 detik). Skrip akan mengambil nilai acak dalam interval sebelum mengeksekusi interaksi.
- **Interaksi Postingan**: Dapat masing-masing centang like, favorit, repost, dan share. Ketika beberapa aksi diaktifkan, sistem akan mendistribusikannya pada titik waktu yang berbeda selama periode menonton, terlihat lebih alami.

- **Komentar**: Setelah diaktifkan dapat berkomentar pada postingan.

     Jika "Komentar" diaktifkan, silakan lakukan konfigurasi berikut dalam pengaturan komentar:

  - Template komentar pre-generated: Dapat paste teks komentar pre-generated multi-baris atau multi-bagian (setiap baris atau setiap bagian dianggap sebagai satu kandidat komentar).

  - Urutan penggunaan teks: Pilih "Random" atau "Sequential".
    - Random: Setiap kali acak mengambil satu teks dari kandidat.
    - Sequential: Gunakan sesuai urutan paste/impor (cocok untuk strategi deduplikasi yang dapat direproduksi).

  - Sisipkan emoji: Dapat mengaktifkan untuk secara otomatis menyisipkan emoji dalam komentar untuk meningkatkan kealamian.

  - Generate komentar real-time (opsional, ChatGPT API):
    - Setelah diaktifkan, skrip dapat memanggil ChatGPT atau API serupa, generate teks komentar yang relevan dengan konteks dan unik berdasarkan judul/deskripsi postingan secara real-time. Cara ini dapat secara signifikan mengurangi risiko komentar berulang, cocok untuk skenario yang memerlukan banyak komentar unik.
    - Catatan: Menggunakan generate real-time memerlukan API Key yang tersedia, dan perhatikan pembatasan rate, biaya, dan kepatuhan privasi (jangan kirim data sensitif ke layanan pihak ketiga).

  - Dua cara umum untuk menjamin komentar unik:
         1. Gunakan "Sequential" untuk memuat dan merotasi template komentar pre-generated (menjamin setiap template hanya digunakan sekali hingga habis).
         2. Gunakan ChatGPT API untuk generate setiap komentar secara real-time (generate dinamis berdasarkan judul/konten postingan, secara alami lebih mudah menjamin keunikan).

  - Saran: Pertama uji strategi komentar dengan sejumlah kecil postingan, periksa kualitas komentar dan situasi pemrosesan platform; jika menggunakan generate real-time, silakan monitor jumlah pemanggilan API dan biaya.

  - Jika menggunakan strategi "Sequential", sistem akan menggabungkan nomor tugas untuk merotasi template antara beberapa perangkat, mengurangi probabilitas pengulangan; jika menggunakan generate AI, disarankan untuk melakukan pemeriksaan kualitas sederhana pada output (seperti rentang panjang, larang menyertakan URL / kata sensitif).

### Penggabungan Tugas (Gabungkan username yang sama)

- **Penjelasan**: Ketika "Gabungkan tugas username yang sama" diaktifkan, beberapa target untuk username yang sama akan dieksekusi secara berurutan atau paralel (tergantung implementasi skrip) dalam satu tugas; bahkan jika salah satu target gagal, tugas tidak akan terganggu secara keseluruhan, target yang tersisa akan terus dieksekusi dan mencoba menyelesaikan, sehingga mengurangi overhead switching dan penjadwalan tugas.

- **Perilaku saat diaktifkan**: Beberapa target yang ditujukan untuk akun yang sama akan dikelompokkan ke dalam satu unit tugas, kegagalan hanya mempengaruhi target itu sendiri dan dicatat, tetapi tidak akan memblokir eksekusi target lain yang berkelanjutan. Cocok untuk digunakan saat marketing skala besar, dapat secara signifikan mengurangi frekuensi startup/switching tugas.

- **Perilaku saat dinonaktifkan**: Setiap target akan menghasilkan satu tugas secara independen, kegagalan target tunggal mana pun akan segera dilaporkan dan menghentikan sub-tugas tersebut (tidak akan mempengaruhi tugas independen lainnya). Cocok untuk skenario yang perlu monitoring dan alerting akurat dan per-item untuk target tunggal.

- **Saran Penggunaan**:
  - Disarankan untuk mengaktifkan saat marketing batch besar atau perlu mempertahankan throughput tinggi untuk mengurangi biaya penjadwalan.
  - Ketika Anda perlu mengkonfirmasi status eksekusi setiap target satu per satu dan segera mengambil tindakan saat gagal (misalnya intervensi manual atau rollback segera), silakan nonaktifkan opsi ini.

### Interval Tugas

- Atur nilai minimum dan maksimum (satuan: menit), skrip akan menunggu secara acak dalam interval antara setiap target, dan menumpuk jeda singkat itu sendiri.

## Alur Eksekusi

- Jika mengaktifkan mode multi-akun, TikMatrix/IgMatrix akan terlebih dahulu memulai TikTok/IgMatrix dan beralih ke akun yang sesuai.
- Dalam mode username, sistem membuka setiap halaman profil sesuai metode akses yang dipilih, menunggu pemuatan selesai kemudian secara berurutan mengeksekusi aksi follow, pesan pribadi, unfollow, dll.; jika pihak lain melarang pesan pribadi akan secara otomatis dilewati.
- Interaksi postingan akan scroll browsing halaman profil atau langsung link, menonton sesuai jumlah dan durasi yang ditetapkan dan mengeksekusi like, favorit, repost, share, dan akan melewati iklan, live, dan konten yang sudah dilike.
- Saat menggunakan link postingan, skrip akan membuka link satu per satu, mengeksekusi aksi yang dikonfigurasi, dan loop sesuai "Jumlah Pengulangan".
- Modul komentar akan mengambil kata dari template atau output AI. Mode sequential akan menggabungkan nomor tugas untuk mendistribusikan komentar yang berbeda antara beberapa perangkat.
- Antara setiap target, skrip akan menambahkan sedikit jeda acak, kemudian menumpuk tunggu acak interval tugas, mensimulasikan ritme orang nyata.

## Praktik Terbaik

- Pertama validasi teks dan ritme dengan data batch kecil, setelah konfirmasi tidak ada masalah baru perbesar skala.
- Kombinasi beberapa aksi (seperti follow + pesan pribadi + like) dapat meningkatkan sinyal interaksi akun.
- Pesan pribadi dan komentar tetap colloquial dan tulus, hindari memicu kontrol risiko platform.
- Perbarui library komentar secara berkala atau sesuaikan System Prompt AI, membuat konten lebih sesuai dengan tema kampanye.
- Perhatikan pembatasan platform, jika skala besar dapat rotasi akun atau node proxy.
