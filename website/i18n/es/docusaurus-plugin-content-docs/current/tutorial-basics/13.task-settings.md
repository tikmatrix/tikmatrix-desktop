# Configuración de tareas

## Reglas de programación de tareas

- Las tareas se ejecutarán dentro de los 10 segundos posteriores a alcanzar la hora de inicio.
- Cada dispositivo solo ejecutará 1 tarea a la vez, otras tareas se pondrán en cola automáticamente en ese dispositivo para procesamiento.
- El número máximo de tareas concurrentes está limitado por el nivel de licencia; los usuarios gratuitos no ejecutarán ninguna tarea automatizada y siempre se mostrarán como "esperando".
- El sistema agregará un retraso aleatorio de 5 a 60 segundos antes de cada reintento.

Este cuadro de diálogo se usa para controlar la estrategia de reintento y el orden de ejecución de las tareas. A continuación se explica brevemente cada opción y método de uso.

## Reintento automático

Función:

- Reintentar automáticamente tareas fallidas hasta que tengan éxito o alcancen el límite de reintentos.

Cómo usar:

- Habilítalo cuando los fallos sean principalmente fallas transitorias (fluctuaciones de red, anomalías de aplicación).
- Primero verifica el comportamiento en tareas de pequeña escala, luego decide si habilitarlo a gran escala.

## Número máximo de reintentos

Función:

- Establece el número máximo de intentos de reintento automático de tareas.

Cómo usar:

- Se recomienda establecer inicialmente en 2-3, si encuentras muchos fallos transitorios puedes aumentarlo apropiadamente.
- La interfaz lo limitará a un rango razonable (por ejemplo, 1-10).

## Orden aleatorio

Función:

- Ejecutar tareas en orden aleatorio en cada dispositivo para evitar comportamiento sincronizado.

Cómo usar:

- Habilítalo en grupos de dispositivos a gran escala o cuando desees reducir la detectabilidad.

## Efectividad y consejos

- Las modificaciones generalmente entrarán en vigor inmediatamente; la interfaz mostrará una notificación de éxito o fallo al guardar.

## Dónde abrir

- Abre la página de dispositivos o la página de tareas, haz clic en el icono de configuración de tareas (engranaje).

## Sugerencias rápidas

- Comienza ajustando `Número máximo de reintentos` = 2-3.
- Habilita `Orden aleatorio` en ejecución a gran escala para reducir patrones sincronizados.
- Prueba primero el comportamiento de `Reintento automático` en pequeña escala.
