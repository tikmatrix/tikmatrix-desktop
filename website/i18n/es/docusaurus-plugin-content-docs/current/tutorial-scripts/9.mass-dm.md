# Mensajes masivos

:::warning Aviso de descontinuación
Esta funcionalidad ha sido integrada en el [script de Super Marketing](./14.super-marketing.md). Por favor, use preferentemente el script "Super Marketing" para obtener capacidades de mensajería masiva y gestión de mensajes más potentes y flexibles.
:::

La funcionalidad de mensajes masivos se utiliza para establecer contacto con usuarios objetivo mediante el envío masivo de mensajes privados (DM), adecuado para escenarios de promoción, seguimiento de servicio al cliente y reactivación de usuarios. El sistema admite dos fuentes de mensajes: plantillas pregeneradas y generación en tiempo real, e intentará simular el envío manual para reducir el riesgo de detección.

## Usar la funcionalidad de mensajes masivos en el script "Super Marketing"

1. Abra el script: Seleccione "Super Marketing".
2. Seleccione el tipo de fuente de datos como "Lista de cuentas de usuario (lista objetivo)".
3. Pegue nombres de usuario objetivo por lotes o impórtelos desde un archivo de texto (uno por línea); después de importar, puede elegir "Agregar" o "Sobrescribir".
4. Seleccione la estrategia de uso de datos:

    - Estrategia de pool compartido: Todas las cuentas bot enviarán mensajes privados a la misma lista objetivo.

    - Estrategia de consumo único: Cada cuenta bot solo procesará objetivos diferentes, asegurando que cada objetivo solo reciba un mensaje de una cuenta una vez.

5. Establezca el número de envíos de mensajes para cada objetivo (generalmente 1 vez, para evitar acoso repetido).
6. En "Configuración de mensajes" configure el contenido de mensajes privados y la estrategia de envío:

    - Plantilla de mensajes pregenerados: Puede pegar mensajes privados pregenerados de múltiples líneas o párrafos (cada línea o párrafo se considera un mensaje candidato).

    - Orden de uso de mensajes: Seleccione "Aleatorio" o "En orden".

        - Aleatorio: Cada vez extrae aleatoriamente una plantilla para enviar.

        - En orden: Rota el uso de plantillas en el orden de pegado/importación (adecuado para estrategias de eliminación de duplicados reproducibles).

    - Insertar emojis: Puede habilitar la inserción automática de emojis en los mensajes para aumentar la naturalidad.

    - Generar mensajes en tiempo real (opcional, ChatGPT API):

        - Una vez habilitado, el script puede llamar a ChatGPT o API similar, generar mensajes personalizados en tiempo real basados en el nombre de usuario objetivo o contexto disponible, adecuado para escenarios que requieren una gran cantidad de mensajes únicos.

        - Nota: Habilitar la generación en tiempo real requiere proporcionar una clave API válida, y prestar atención a los límites de velocidad, costos y cumplimiento de privacidad (evitar enviar datos sensibles).

    - Dos formas comunes de garantizar mensajes únicos:

        1. Usar "En orden" para rotar plantillas pregeneradas (cada plantilla solo se usa una vez hasta que se agote).

        2. Usar ChatGPT para generar cada mensaje en tiempo real (generación dinámica basada en información del objetivo, mayor unicidad).

7. Configure la forma de abrir usuario (si necesita ver el perfil del usuario): Seleccione "Buscar usuario" o "Abrir perfil directamente (Direct Open Profile)".
8. Configure el intervalo de tareas (mínimo/máximo, unidad: minutos) y concurrencia, pool de cuentas y otros parámetros, confirme que todo esté correcto y ejecute el script.

:::tip Sobre el intervalo de tareas
Se recomienda establecer intervalos mínimos/máximos y habilitar intervalos aleatorios para evitar que todos los dispositivos ejecuten al mismo tiempo causando picos anormales, reduciendo así el riesgo de ser detectado o limitado por la plataforma.
:::

## Sugerencias de uso y notas

- El contenido de mensajes privados debe ser conforme y diversificado, evitar enviar grandes cantidades del mismo contenido en poco tiempo.
- Si usa generación en tiempo real, monitoree estrictamente la cantidad de llamadas API y los costos, y asegúrese de no filtrar información sensible al exterior.
- Las tareas pueden fallar; si encuentra fallas, reintente o revise los registros, el sistema saltará automáticamente los objetivos ya completados.
