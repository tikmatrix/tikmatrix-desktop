# Impulsar publicaciones

:::warning Aviso de descontinuación
Esta funcionalidad del script ha sido integrada en el [script de Super Marketing](./14.super-marketing.md). Por favor, use preferentemente el script "Super Marketing" para obtener capacidades de impulso de publicaciones más potentes y flexibles.
:::

La funcionalidad de impulsar publicaciones puede aumentar la cantidad de interacción de publicaciones mediante operaciones masivas como dar me gusta, compartir, favoritos, repostear y ver. El sistema intentará simular el comportamiento de usuarios reales para reducir el riesgo de detección, pero al usar aún debe seguir las reglas de la plataforma y controlar moderadamente la frecuencia.

## Usar la funcionalidad de impulsar publicaciones en el script "Super Marketing"

1. Abra el script: Seleccione "Super Marketing".
2. Seleccione el tipo de fuente de datos como "Lista de enlaces de publicaciones".
3. Pegue enlaces de publicaciones por lotes o impórtelos desde un archivo de texto; después de importar, puede elegir "Agregar" o "Sobrescribir" la lista actual.
4. Seleccione la estrategia de uso de datos:

    - Estrategia de pool compartido: Todas las cuentas bot interactúan con las mismas publicaciones.

    - Estrategia de consumo único: Cada cuenta bot solo interactúa con publicaciones diferentes una vez, asegurando que cada publicación solo sea procesada una vez por una cuenta.

5. En "Operaciones de publicación — Número de repeticiones" establezca el número de interacciones para cada publicación. Si incluye comentarios, se recomienda establecer el número de comentarios en 1 para evitar comentarios duplicados.
6. En "Operaciones de publicación" seleccione los tipos de interacción necesarios: dar me gusta, compartir, repostear, favoritos, comentar, ver video, etc.

     Si selecciona "Comentar", configure lo siguiente en la configuración de comentarios:

     - Plantilla de comentarios pregenerados: Puede pegar mensajes de comentarios pregenerados de múltiples líneas o párrafos (cada línea o párrafo se considera un comentario candidato).

     - Orden de uso de mensajes: Seleccione "Aleatorio" o "En orden".
         - Aleatorio: Cada vez extrae aleatoriamente un mensaje de los mensajes candidatos.
         - En orden: Usa en el orden de pegado/importación (adecuado para estrategias de eliminación de duplicados reproducibles).

     - Insertar emojis: Puede habilitar la inserción automática de emojis en los comentarios para aumentar la naturalidad.

     - Generar comentarios en tiempo real (opcional, ChatGPT API):
         - Una vez habilitado, el script puede llamar a ChatGPT o API similar, generar mensajes de comentarios únicos y relevantes al contexto basados en el título/descripción de la publicación en tiempo real. Este método puede reducir significativamente el riesgo de comentarios duplicados, adecuado para escenarios que requieren una gran cantidad de comentarios únicos.
         - Notas: Usar generación en tiempo real requiere proporcionar una clave API válida, y prestar atención a los límites de velocidad, costos y cumplimiento de privacidad (no enviar datos sensibles a servicios de terceros).

     - Dos formas comunes de garantizar comentarios únicos:
         1. Usar "En orden" para cargar y rotar plantillas de comentarios pregenerados (garantizar que cada plantilla solo se use una vez hasta que se agote).
         2. Usar ChatGPT API para generar cada comentario en tiempo real (generación dinámica basada en el título/contenido de la publicación, naturalmente más fácil de garantizar la unicidad).

     - Sugerencia: Primero pruebe la estrategia de comentarios con un pequeño lote de publicaciones, verifique la calidad de los comentarios y la situación de procesamiento de la plataforma; si usa generación en tiempo real, monitoree la cantidad de llamadas API y el costo.
7. Configure otros parámetros del script (como número de concurrencia, retrasos, pool de cuentas, etc.), confirme que todo esté correcto y ejecute el script.

## Sugerencias de uso y notas

- Se recomienda probar primero en una pequeña cantidad de publicaciones, confirmar que la estrategia y los parámetros no activen el control de riesgos de la plataforma.
- El contenido de los comentarios debe ser diversificado y mantener la naturalidad, evitar copiar y pegar el mismo texto.
- Configure razonablemente los retrasos y la concurrencia, simule el comportamiento humano para reducir el riesgo.
- Durante la ejecución a largo plazo, verifique regularmente la salud de las cuentas y los registros de tareas, y ajuste la estrategia según la situación.
