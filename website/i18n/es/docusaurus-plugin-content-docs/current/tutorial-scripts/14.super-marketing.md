# Super Marketing

:::info
El script de Super Marketing concentra seguir/dejar de seguir, mensajes privados, interacción de publicaciones y comentarios en la misma cola de tareas, adecuado para alcanzar rápidamente clientes potenciales o reactivar seguidores antiguos en una ronda de operaciones.
:::

## Pasos de operación

1. En TikMatrix/IgMatrix seleccione los dispositivos que necesitan ejecutar el script (se puede seleccionar múltiples).
2. Abra `Scripts` → `Super Marketing`.
3. Seleccione **Fuente de datos** (nombres de usuario o enlaces de publicaciones), luego puede hacer clic en el botón `Importar archivo txt` para cargar el archivo `.txt` correspondiente o pegar directamente datos de múltiples líneas; después de importar puede elegir "Agregar" o "Sobrescribir".
4. Según sus necesidades, habilite los módulos de acciones de usuario, acciones de publicación y comentarios, y complete la configuración de acuerdo con las instrucciones a continuación.
5. Configure el control deslizante de **Intervalo de tareas** para definir el retraso aleatorio entre objetivos, finalmente haga clic en `Iniciar Script` para ejecutar la tarea.

## Descripción de configuración

### Fuente de datos

- **Nombres de usuario / Enlaces de publicaciones**: Determina qué funcionalidades están disponibles. Seguir, dejar de seguir, mensajes privados solo se pueden usar con nombres de usuario; si solo desea operar publicaciones, puede seleccionar directamente el modo de enlaces.
- **Forma de importación**: Puede hacer clic en el botón `Importar archivo txt` para cargar el archivo `.txt` correspondiente o pegar directamente datos de múltiples líneas; después de importar puede elegir "Agregar" o "Sobrescribir".

### Estrategia de uso de datos

- **Pool compartido (Shared pool)**: Todas las cuentas bot toman tareas del mismo pool objetivo, adecuado para escenarios donde múltiples cuentas generan múltiples interacciones con el mismo objetivo (por ejemplo, impulsar concentradamente publicaciones populares).

- **Consumo único (Single-consumption)**: Cada cuenta bot solo procesa objetivos diferentes, asegurando que cada objetivo solo sea procesado una vez por una cuenta, adecuado para escenarios que necesitan evitar alcance repetido (por ejemplo, mensajes masivos o una ronda de comentarios únicos).

  - **Cantidad máxima de consumo por tarea**: En modo de consumo único, puede establecer la cantidad máxima de objetivos consumidos por cada ejecución de tarea. Esto puede evitar que el primer lote de tareas consuma todos los datos, asegurando que las cuentas posteriores aún tengan objetivos disponibles al rotar. Establecer en 0 significa sin límite (comportamiento predeterminado).

La elección de estrategia afectará la asignación de cola y el comportamiento de concurrencia, elija según los objetivos y consideraciones de control de riesgos.

### Acciones de usuario

- **Seguir usuario**: Ejecuta la operación de seguir prioritariamente después de entrar al perfil.
- **Dejar de seguir usuario**: Dejar de seguir, puede ejecutarse solo o limpiar seguidores después de mensajes privados; no se recomienda habilitar junto con "Seguir".
- **Reportar cuenta**: Reportar cuenta de usuario por comportamiento fraudulento. Esta funcionalidad usa una ruta de reporte fija, no admite rutas personalizadas.

  :::warning Ruta de reporte (fija)
  Reportar → Reportar cuenta → Something else → Frauds and scams
  :::

  - Descripción: Esta funcionalidad usa una ruta de reporte predefinida, no admite razones o rutas de reporte personalizadas.
  - Use esta funcionalidad responsablemente, solo para propósitos legítimos. El abuso puede resultar en restricción de cuenta.
- **Enviar mensaje privado**: Extrae aleatoriamente un mensaje de múltiples líneas de mensajes para enviar; marcar "Añadir emojis automáticamente" puede adjuntar emojis naturales automáticamente. El script detectará si la otra parte permite mensajes privados, saltará si no se puede enviar.

  - Plantilla de mensajes privados (pregenerados): En "Configuración de mensajes" pegue mensajes privados pregenerados de múltiples líneas o párrafos (cada línea/párrafo se considera un mensaje candidato).

  - Orden de uso de mensajes: Seleccione "Aleatorio (Random)" o "En orden (Sequential)".

    - Aleatorio: Cada vez extrae aleatoriamente una plantilla para enviar.

    - En orden: Rota el uso en el orden de pegado/importación, adecuado para estrategias de eliminación de duplicados reproducibles.

  - Insertar emojis: Puede habilitar la inserción automática de emojis en mensajes o comentarios para aumentar la naturalidad.

  - Generación en tiempo real (opcional, ChatGPT o API compatible):

    - Una vez habilitado, el script puede llamar a la interfaz de generación que configuró (completar API Key / URL / Modelo / System Prompt) para generar mensajes personalizados basados en el contexto objetivo.

    - Nota: Necesita proporcionar una clave API válida, y prestar atención a los límites de velocidad, costos y cumplimiento de privacidad (evitar enviar datos sensibles).

  - Estrategia de unicidad: Formas comunes de garantizar que cada mensaje/comentario sea único:

      1. Usar "En orden" para rotar plantillas pregeneradas (plantillas solo se usan una vez hasta que se agoten).
      2. Usar generación en tiempo real (ChatGPT) para generar texto único para cada objetivo.
  - Antes de usar mensajes masivos, asegúrese de probar en pequeños lotes y preste atención a los requisitos de privacidad y cumplimiento.

### Acciones de publicación

- **Número de publicaciones a procesar**: Determina el número de publicaciones que cada cuenta operará (1–50), solo necesario cuando la fuente de datos es "Nombres de usuario".
- **Número de repeticiones**: Repite el ciclo de interacción de publicaciones múltiples veces, puede operar repetidamente en publicaciones sin reiniciar el script.
- **Duración de visualización**: Establece el tiempo de visualización mínimo y máximo (0–300 segundos). El script extraerá un valor aleatorio dentro del intervalo antes de ejecutar la interacción.
- **Interacción de publicaciones**: Puede marcar individualmente me gusta, favoritos, repostear y compartir. Cuando se habilitan múltiples acciones, el sistema las distribuirá en diferentes momentos del período de visualización, pareciendo más natural.

- **Comentarios**: Una vez habilitado, puede publicar comentarios en publicaciones.

     Si habilitó "Comentarios", configure lo siguiente en la configuración de comentarios:

  - Plantilla de comentarios pregenerados: Puede pegar mensajes de comentarios pregenerados de múltiples líneas o párrafos (cada línea o párrafo se considera un comentario candidato).

  - Orden de uso de mensajes: Seleccione "Aleatorio" o "En orden".
    - Aleatorio: Cada vez extrae aleatoriamente un mensaje de los mensajes candidatos.
    - En orden: Usa en el orden de pegado/importación (adecuado para estrategias de eliminación de duplicados reproducibles).

  - Insertar emojis: Puede habilitar la inserción automática de emojis en los comentarios para aumentar la naturalidad.

  - Generar comentarios en tiempo real (opcional, ChatGPT API):
    - Una vez habilitado, el script puede llamar a ChatGPT o API similar, generar mensajes de comentarios únicos y relevantes al contexto basados en el título/descripción de la publicación en tiempo real. Este método puede reducir significativamente el riesgo de comentarios duplicados, adecuado para escenarios que requieren una gran cantidad de comentarios únicos.
    - Notas: Usar generación en tiempo real requiere proporcionar una clave API válida, y prestar atención a los límites de velocidad, costos y cumplimiento de privacidad (no enviar datos sensibles a servicios de terceros).

  - Dos formas comunes de garantizar comentarios únicos:
         1. Usar "En orden" para cargar y rotar plantillas de comentarios pregenerados (garantizar que cada plantilla solo se use una vez hasta que se agote).
         2. Usar ChatGPT API para generar cada comentario en tiempo real (generación dinámica basada en título/contenido de la publicación, naturalmente más fácil de garantizar la unicidad).

  - Sugerencia: Primero pruebe la estrategia de comentarios con un pequeño lote de publicaciones, verifique la calidad de los comentarios y la situación de procesamiento de la plataforma; si usa generación en tiempo real, monitoree la cantidad de llamadas API y el costo.

  - Si usa la estrategia "Secuencial", el sistema combinará el número de tarea para rotar plantillas entre múltiples dispositivos, reduciendo la probabilidad de duplicados; si usa generación de IA, se recomienda hacer reglas simples de verificación de calidad de salida (como rango de longitud, prohibir incluir URL / palabras sensibles).

### Fusión de tareas (Fusionar tareas del mismo nombre de usuario)

- **Descripción**: Cuando se habilita "Fusionar tareas del mismo nombre de usuario", múltiples objetivos para el mismo nombre de usuario se ejecutarán en secuencia o en paralelo (según la implementación del script) dentro de una tarea; incluso si falla algún objetivo, la tarea no se interrumpirá por completo, los objetivos restantes continuarán ejecutándose e intentarán completarse, reduciendo así el costo de cambio y programación de tareas.

- **Comportamiento al habilitar**: Múltiples objetivos para la misma cuenta se agruparán en la misma unidad de tarea, el fallo solo afecta al objetivo en sí y se registra, pero no bloqueará la ejecución continua de otros objetivos. Adecuado para uso en marketing a gran escala, puede reducir significativamente la frecuencia de inicio/cambio de tareas.

- **Comportamiento al deshabilitar**: Cada objetivo generará una tarea individual, cualquier fallo de un solo objetivo se reportará inmediatamente y detendrá esa subtarea (no afectará otras tareas independientes). Adecuado para escenarios que requieren monitoreo y alerta precisos y línea por línea de cada objetivo individual.

- **Sugerencia de uso**:
  - Se recomienda habilitar para reducir el costo de programación en marketing por lotes o cuando se necesita mantener un alto rendimiento.
  - Cuando necesite confirmar el estado de ejecución de cada objetivo línea por línea y tomar medidas inmediatas cuando falle (por ejemplo, intervención manual o reversión inmediata), desactive esta opción.

### Intervalo de tareas

- Establezca el valor mínimo y máximo (unidad: minutos), el script esperará aleatoriamente dentro del intervalo entre cada objetivo, y superpondrá sus propias pausas breves.

## Flujo de ejecución

- Si se especificó y habilitó el modo de múltiples cuentas, TikMatrix/IgMatrix primero iniciará TikTok/IgMatrix y cambiará a la cuenta correspondiente.
- En modo de nombres de usuario, el sistema abre cada perfil según la forma de acceso seleccionada, espera a que se complete la carga y luego ejecuta acciones como seguir, mensajes privados, dejar de seguir en secuencia; si la otra parte prohíbe mensajes privados, se saltará automáticamente.
- La interacción de publicaciones deslizará para navegar por el perfil o enlace directo, según la cantidad y duración establecidas, verá y ejecutará me gusta, favoritos, repostear, compartir, y saltará anuncios, transmisiones en vivo y contenido que ya tiene me gusta.
- Al usar enlaces de publicaciones, el script abrirá enlaces uno por uno, ejecutará las acciones configuradas y repetirá según el "número de repeticiones".
- El módulo de comentarios tomará palabras de plantillas o salida de IA. El modo secuencial combinará el número de tarea para asignar diferentes comentarios entre múltiples dispositivos.
- Entre cada objetivo, el script añadirá pequeñas pausas aleatorias, luego superpondrá la espera aleatoria del intervalo de tareas, simulando el ritmo humano.

## Mejores prácticas

- Primero verifique mensajes y ritmo con pequeños lotes de datos, confirme que todo esté correcto antes de escalar.
- Combinar múltiples acciones (como seguir + mensaje privado + me gusta) puede mejorar la señal de interacción de la cuenta.
- Mensajes privados y comentarios deben ser coloquiales y sinceros, evitar activar el control de riesgos de la plataforma.
- Actualice regularmente la biblioteca de comentarios o ajuste el System Prompt de IA para que el contenido se adapte mejor al tema de la actividad.
- Preste atención a la limitación de velocidad de la plataforma, si la escala es grande, puede rotar cuentas o nodos proxy.
