# Reportar cuenta

:::warning Usar responsablemente
La funcionalidad de reporte es una operación poderosa. Solo úsela cuando confirme que existe comportamiento ilegal o infractor (fraude, estafa, spam, etc.). El abuso de reportes puede resultar en restricción de cuenta u otras consecuencias adversas.
:::

Esta guía explica cómo usar la funcionalidad integrada de "Reportar cuenta" del script Super Marketing. Esta funcionalidad ejecutará automáticamente el proceso de reporte fijo dentro de la aplicación para procesar un lote de cuentas de usuario, pero no admite personalización de rutas o razones de reporte en la UI o script.

## Vista rápida

- El proceso de reporte fijo que usa el script es el siguiente (no modificable):

```text
Reportar → Reportar cuenta → Something else → Frauds and scams
```

- El script intentará entrar al perfil del usuario objetivo, abrir el menú/más elementos, seleccionar reporte según la ruta fija y enviar (si está disponible).
- Debido a que la UI de la aplicación móvil puede variar en diferentes versiones y dispositivos, el script usa selectores de mejor esfuerzo, por lo tanto, verifique primero en pequeños lotes.

## Pasos para ejecutar a través de Super Marketing

1. Abra TikMatrix/IgMatrix y seleccione los dispositivos a ejecutar (admite selección múltiple).
2. Abra `Scripts` → `Super Marketing`.
3. Establezca **Fuente de datos (Data source)** en `Usernames`, importe archivo de texto o pegue nombres de usuario (uno por línea); después de importar seleccione "Agregar" o "Sobrescribir".
4. En **Acciones de usuario (User Actions)** habilite el interruptor **Reportar cuenta (Report account)**, y confirme la ruta de reporte fija mostrada.
5. Configure concurrencia de tareas, retrasos y otros parámetros.
6. Haga clic en `Start Script` para comenzar a procesar la lista, monitoree los registros e interfaz del dispositivo para confirmar éxito o fallo de operaciones.

## Sugerencias y notas

- Primero pruebe en 1–5 cuentas, confirme que el script puede identificar y ejecutar correctamente el proceso de reporte (cambios en texto o posición de UI causarán fallo).
- Mantenga un ritmo conservador y retrasos, el reporte automático por lotes puede activar limitación de velocidad o control de riesgos de la plataforma.
- Si la UI de la aplicación cambia (texto de botón, cambio de diseño), el script puede no encontrar los elementos correspondientes, necesitará actualizar selectores o ajustar estrategia de espera.
- Solo use reporte para contenido infractor confirmado, evite acoso o abuso de esta funcionalidad contra usuarios inocentes.
- Después de ejecutar, verifique los registros, el script registrará errores como "no se puede encontrar botón de menú" o "no se puede encontrar opción de reporte", conveniente para que ajuste el script o parámetros de ejecución.

## Escenarios no recomendados para uso

- Escenarios que requieren seleccionar razón de reporte personalizada o subir evidencia para cada caso: El script no admite personalización granular.
- Eventos legales/sensibles que requieren revisión manual o preservación de evidencia.

## Solución de problemas

- Error "Could not find Report option" / "Could not find 'Report account' option": Indica que el texto o diseño de la UI de la aplicación ha cambiado. Ejecute nuevamente en pequeños lotes y recopile capturas de pantalla del dispositivo para depuración.
- Si el script muestra que el reporte se envió pero la plataforma no lo procesó realmente, la plataforma puede haber degradado reportes automáticos o requerir revisión manual; para casos importantes individuales, use el proceso manual.
