# הגדרות שיקוף מסך

שיקוף מסך מאפשר לך לצפות ולשלוט במכשירים Android מהמחשב.

## יתרונות

- 👀 **צפייה בזמן אמת** - ראה מה קורה במכשיר
- 🎮 **שליטה מרחוק** - שלוט במכשיר מהמחשב
- 🐛 **ניפוי שגיאות** - אבחן בעיות בקלות
- 📹 **הקלטה** - הקלט את פעילות המכשיר
- 🖼️ **צילומי מסך** - צלם מסכים בקלות

## הפעלת שיקוף מסך

### שלב 1: חבר מכשיר

```
TikMatrix → מכשירים → בדוק שהמכשיר מחובר
```

### שלב 2: הפעל שיקוף

```
לחץ על אייקון מכשיר → שיקוף מסך
או
לחיצה כפולה על המכשיר
```

### שלב 3: הגדר תצורה

```
שיקוף מסך → הגדרות → התאם פרמטרים
```

## הגדרות תצורה

### רזולוציה

בחר רזולוציה מתאימה:

| רזולוציה | שימוש | איכות | ביצועים |
|-----------|-------|-------|----------|
| 360p | ניטור בסיסי | נמוכה | מהיר מאוד |
| 480p | עבודה יומיומית | בינונית | מהיר |
| 720p | איכות טובה | גבוהה | בינוני |
| 1080p | איכות מקסימלית | מאוד גבוהה | איטי |

**מומלץ:** 480p-720p לרוב המשתמשים

### קצב רענון (FPS)

| FPS | שימוש | ביצועים |
|-----|-------|----------|
| 15 | ניטור סטטי | חסכוני |
| 30 | עבודה רגילה | אופטימלי |
| 60 | פעולות מהירות | משאבים גבוהים |

**מומלץ:** 30 FPS לרוב המשימות

### איכות

| רמה | תיאור | שימוש |
|-----|--------|-------|
| נמוכה | דחיסה גבוהה | חיבור איטי |
| בינונית | איזון | שימוש יומיומי |
| גבוהה | דחיסה מינימלית | חיבור מהיר |

**מומלץ:** איכות בינונית

### השהיה (Latency)

- **נמוכה:** < 100ms - שליטה בזמן אמת
- **בינונית:** 100-300ms - עבודה נוחה
- **גבוהה:** > 300ms - ניטור בלבד

## פתרון בעיות

### מסך שחור

**סיבות:**
- המכשיר נעול
- הרשאות חסרות
- דרייבר מיושן

**פתרון:**

1. בטל נעילת המכשיר
2. הענק הרשאות:
   ```bash
   adb shell settings put global development_settings_enabled 1
   adb shell settings put global stay_on_while_plugged_in 7
   ```
3. עדכן דרייבר

### פיגורים (Lag)

**סיבות:**
- רזולוציה גבוהה מדי
- FPS גבוה מדי
- חיבור USB איטי

**פתרון:**

```
הגדרות → שיקוף:
- הפחת רזולוציה ל-480p
- הגדר FPS ל-15
- הפחת איכות
- השתמש ביציאת USB 3.0
```

### ניתוק מתמיד

**פתרון:**

```bash
# הפעל מחדש שרת ADB
adb kill-server
adb start-server

# הפעל מחדש שיקוף
TikMatrix → מכשיר → הפעל מחדש שיקוף
```

### צבעים שגויים

**פתרון:**

```
הגדרות → שיקוף → קידוד צבע:
- נסה RGB
- נסה YUV
```

### לא מגיב לקלט

**פתרון:**

1. בדוק הרשאות ADB
2. נסה חיבור מחדש
3. הפעל מחדש מכשיר

## שאלות נפוצות

**ש: האם שיקוף משפיע על ביצועי מכשיר?**  
ת: מעט. הפחת FPS ורזולוציה למינימום השפעה.

**ש: האם אפשר לשקף דרך WiFi?**  
ת: כן, השתמש ב-ADB over WiFi. פחות יציב מ-USB.

**ש: כמה מכשירים אפשר לשקף בו-זמנית?**  
ת: תלוי במחשב. מומלץ 4-9 למחשב ממוצע.

**ש: האם יש עיכוב?**  
ת: בדרך כלל 50-150ms, מספיק לשימוש נוח.

## טיפים

1. **השתמש ברזולוציה נמוכה לניטור ריבוי מכשירים**
2. **הפעל האצת חומרה אם זמינה**
3. **כבה שיקוף כשלא בשימוש לחיסכון במשאבים**
4. **השתמש ב-USB 3.0 לשיקוף איכותי**
5. **סגור יישומים מיותרים להפחתת עומס**
