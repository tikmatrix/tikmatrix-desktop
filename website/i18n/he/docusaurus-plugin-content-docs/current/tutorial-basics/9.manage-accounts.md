# ניהול חשבונות

ניהול חשבונות שומר על עקביות בין כל זהות TikTok / Instagram למכשיר שלה, פרטי כניסה והנתונים שעליהם האוטומציה מסתמכת. רשימת חשבונות נקייה מאפשרת לך להקצות מכשירים, לנטר סטטוס חשבונות ולחמם אותם לפעילות ללא צורך בטיפול בכל טלפון בנפרד.

## יצירת חשבון ידנית

1. פתח את **חשבונות** בניווט הצד השמאלי.
2. לחץ על **חדש** כדי לפתוח את חלון העריכה.
3. מלא את השדות הבסיסיים ברצף:
   - **אימייל**: האימייל המשמש להתחברות לאפליקציה. TikMatrix/IgMatrix ינסו תחילה להתחבר באמצעות אימייל + סיסמה; אם האימייל ריק, המערכת תעבור אוטומטית להתחברות באמצעות שם משתמש + סיסמה.
   - **סיסמה**: הסיסמה המשמשת להתחברות לאפליקציה. IgMatrix תומך בהוספת מפתח 2FA לסיסמה, ראה פורמט למטה.
   - **שם משתמש**: שם החשבון המוצג ב-TikTok/Instagram, זהו השדה היחיד הנדרש.
   - **סטטוס כניסה**: סמן האם כבר התחברת במכשיר הנוכחי (`1` = מחובר, `0` = לא מחובר).
   - **סטטוס**: שולט האם החשבון משתתף במשימות (`0` = מופעל, `1` = מושבת).
4. בחר תג מכשיר כדי לקשור את החשבון לטלפון או לטלפון ענן. אם לא נבחר, החשבון יישאר לא מוקצה.
5. לחץ על **שמור**, והרשימה תתרענן אוטומטית.

## כניסה

- אם **אימייל** ו-**סיסמה** של החשבון כבר מולאו, TikMatrix/IgMatrix ינסו להשתמש במידע זה כדי להתחבר לאפליקציית TikTok/Instagram.
- אם **אימייל** ריק, המערכת תנסה להתחבר באמצעות **שם משתמש** ו-**סיסמה**.
- יש לבחור מספר סידורי של מכשיר כדי שהסקריפט יבצע פעולת כניסה במכשיר המתאים.
- סקריפט הכניסה יזהה אוטומטית את סטטוס הכניסה של החשבון וידלג על חשבונות שכבר מחוברים.

## תמיכת 2FA (IgMatrix בלבד)

IgMatrix תומך באימות דו-שלבי (2FA) אוטומטי:

### פורמט סיסמה עם 2FA

לשימוש ב-2FA, הגדר את הסיסמה בפורמט הבא:

```
הסיסמה_שלך:מפתח_2FA_שלך
```

**דוגמה:**

```
mypassword123:JBSWY3DPEHPK3PXP
```

## ייבוא ​​וייצוא חשבונות

- **ייצוא** יכתוב את כל החשבונות לקובץ `accounts.xlsx` לספריית ההורדות של TikMatrix/IgMatrix, יכול לשמש כגיבוי או תבנית לעריכה בכמות גדולה.
- **ייבוא** תומך ביצירה או עדכון של חשבונות בכמות גדולה מקובץ Excel זה. אם אינך בטוח בכותרות הטבלה, לחץ תחילה על **ייצוא**, ערוך את הקובץ שהורד וייבא אותו.
- בעת ייבוא, שורות עם `id` יעדכנו חשבונות קיימים, שורות ללא `id` ייצרו חשבונות חדשים.

> כניסת IgMatrix עם 2FA דורשת רק למלא את `username` ו-`password`, בפורמט `הסיסמה_שלך:מפתח_2FA_שלך`, ו-`device_index` מספר סידורי של המכשיר.

### מפת שדות Excel

| שדה | נדרש | הסבר |
| --- | --- | --- |
| `id` | לא | מזהה ייחודי פנימי, שמור רק כשצריך לעדכן חשבון קיים. |
| `email` | לא | אימייל כניסה לאפליקציה, נוח ליצירת קשר ושחזור. |
| `pwd` | לא | סיסמת כניסה, יכולה לשמש להפניה מהירה או כניסה אוטומטית. |
| `username` | כן | שם המשתמש של TikTok/Instagram, צריך להיות ייחודי בתוך סביבת העבודה. |
| `device` | לא | מספר סידורי של המכשיר הקשור או מזהה טלפון ענן. |
| `device_index` | לא | שם המכשיר המוצג בממשק, משמש בעיקר להפניה. |
| `logined` | לא | סמן סשן (`1` = מחובר, `0` = לא מחובר). |
| `status` | לא | סמן זמינות (`0` = מופעל, `1` = מושבת). חשבונות מושבתים לא ישתתפו במשימות סקריפט. |
| `nickname` | לא | הכינוי האחרון שהתקבל באמצעות סנכרון נתוני TikTok. |
| `country` | לא | המדינה/אזור שהוסק מפרופיל TikTok. |
| `followers` | לא | מספר העוקבים שסונכרן. |
| `following` | לא | מספר האנשים שהחשבון עוקב אחריהם. |
| `hearts` | לא | מספר הלייקים הכולל שהחשבון קיבל. |
| `videos` | לא | מספר הסרטונים שפורסמו. |
| `friends` | לא | מספר החברים המשותפים (אם זמין). |

### הבדלי שדות בין TikMatrix ל-IgMatrix

- **TikMatrix** ימלא את שדות הסטטיסטיקה (`nickname`, `country`, `followers`, `following`, `hearts`, `videos`, `friends`) לאחר ביצוע **סנכרון נתוני TikTok** או לחיצה על כפתור שאילתת הנתונים.
- **IgMatrix** משתמש רק בשדות הבסיסיים (`email`, `pwd`, `username`, `device`, `logined`, `status`) ויתעלם משדות הסטטיסטיקה.

## התנהגויות אחרות

- חשבונות המסומנים כ-**מושבתים** לא ישתתפו בשום סקריפטים אוטומטיים, משימות מתוזמנות או פעולות בכמות גדולה. יש להפעיל מחדש כדי להמשיך במשימות.
- הסקריפט **התאמת חשבונות** יסרוק סשנים של אפליקציות שכבר מחוברות במכשיר, יצור אוטומטית ערכים מתאימים ברשימת החשבונות ויקשור את המכשירים שזוהו, זוהי דרך מהירה לקליטת טלפונים שהוכנו ידנית.
