# סיבוב פרוקסי

השתמש בסיבוב פרוקסי להגנה טובה יותר על חשבונות ולהימנע מחסימות.

## מדוע פרוקסי?

פרוקסי מספקים:

- 🔒 **הסתרת IP** - החבא את כתובת ה-IP האמיתית שלך
- 🌍 **גיאו-לוקיישן** - הופיע ממדינות שונות
- 🛡️ **הגנה מחסימות** - הפחת סיכון לחסימת חשבון
- 📈 **ניהול מותאם** - פרוקסי שונה לכל חשבון

## סוגי פרוקסי

### פרוקסי מגורים (Residential)

**יתרונות:**
- ✅ IP אמיתי ממשתמשים רגילים
- ✅ שיעור זיהוי נמוך
- ✅ מומלץ ביותר

**חסרונות:**
- �� יקר יותר
- 🐌 לפעמים איטי

**מומלץ עבור:** חשבונות ערך גבוה, פעילות אינטנסיבית

### פרוקסי datacenter

**יתרונות:**
- 💸 זול יותר
- ⚡ מהיר מאוד
- 📊 יציב

**חסרונות:**
- ⚠️ קל יותר לזיהוי
- 🚫 סיכון גבוה יותר לחסימה

**מומלץ עבור:** מבחנים, פעילות בינונית

### פרוקסי ניידים (Mobile)

**יתרונות:**
- 🌟 איכות הטובה ביותר
- 📱 IP אמיתי ממכשירים ניידים
- 🔐 קשה מאוד לזיהוי

**חסרונות:**
- 💰💰 יקר מאוד
- 🌐 זמינות מוגבלת

**מומלץ עבור:** חשבונות פרימיום, מצבים מיוחדים

## הגדרה ב-TikMatrix

### שיטה 1: פרוקסי גלובלי

הגדר פרוקסי אחד לכל המכשירים:

```
TikMatrix → הגדרות → רשת → פרוקסי
```

1. הפעל **שימוש בפרוקסי**
2. הזן פרטי פרוקסי:
   - Host: `proxy.example.com`
   - Port: `8080`
   - Username: `user`
   - Password: `pass`
3. לחץ **שמור**

### שיטה 2: פרוקסי לכל מכשיר

הקצה פרוקסי ספציפי לכל מכשיר:

```
TikMatrix → מכשירים → [בחר מכשיר] → הגדרות
```

1. הפעל **השתמש בפרוקסי מותאם**
2. הזן פרטי פרוקסי
3. לחץ **החל**

### שיטה 3: סיבוב אוטומטי

הפעל סיבוב פרוקסי אוטומטי:

```
TikMatrix → הגדרות → רשת → סיבוב פרוקסי
```

**אפשרויות:**
- **לפי זמן:** שנה כל X דקות
- **לפי משימה:** שנה לכל משימה חדשה
- **לפי מגבלה:** שנה אם מגיע למגבלת קצב

### שיטה 4: רשימת פרוקסי

טען רשימת פרוקסי מקובץ:

```
TikMatrix → הגדרות → רשת → ייבא רשימה
```

**פורמט קובץ (CSV):**

```csv
host,port,username,password,type
proxy1.com,8080,user1,pass1,residential
proxy2.com,8081,user2,pass2,datacenter
proxy3.com,8082,user3,pass3,mobile
```

## אסטרטגיות סיבוב

### אסטרטגיה 1: פרוקסי ייעודי לחשבון

```
חשבון 1 → Proxy 1 (קבוע)
חשבון 2 → Proxy 2 (קבוע)
חשבון 3 → Proxy 3 (קבוע)
```

**יתרונות:**
- עקביות
- קל לעקוב
- מומלץ למרבית המשתמשים

### אסטרטגיה 2: סיבוב מתוזמן

```
09:00-12:00 → Proxy 1
12:00-15:00 → Proxy 2
15:00-18:00 → Proxy 3
```

**יתרונות:**
- חיסכון בעלויות
- מדמה שינוי מיקום

### אסטרטגיה 3: סיבוב אקראי

```
משימה 1 → Proxy רנדומלי
משימה 2 → Proxy רנדומלי
משימה 3 → Proxy רנדומלי
```

**יתרונות:**
- קשה לזיהוי תבנית
- מקסימום אבטחה

## בדיקת פרוקסי

### בדיקה ידנית

ודא שהפרוקסי עובד:

```bash
# בדוק IP
curl --proxy http://user:pass@proxy.com:8080 https://ipinfo.io

# בדוק מהירות
curl --proxy http://user:pass@proxy.com:8080 -o /dev/null -s -w "%{time_total}\n" https://google.com
```

### בדיקה ב-TikMatrix

```
TikMatrix → הגדרות → רשת → בדוק פרוקסי
```

TikMatrix יבדוק:
- ✅ חיבור
- ✅ IP נוכחי
- ✅ מיקום גיאוגרפי
- ✅ מהירות

## שיטות עבודה מומלצות

### 1. התאם פרוקסי למיקום חשבון

```
חשבון בארה"ב → Proxy ארה"ב
חשבון בגרמניה → Proxy גרמניה
```

### 2. השתמש בפרוקסי מגורים לפעולות רגישות

- יצירת חשבון
- פרסום ראשון
- שינוי הגדרות

### 3. סבב IP באופן קבוע

```json
{
  "rotation_interval": 3600,
  "rotation_on_error": true,
  "max_requests_per_ip": 100
}
```

### 4. נטר מצב פרוקסי

עקוב אחר:
- זמן תגובה
- שיעור הצלחה
- שימוש ברוחב פס

### 5. גבה פרוקסי עצירה

תמיד יש פרוקסי חלופי:

```
Primary: proxy1.com
Backup: proxy2.com
Emergency: proxy3.com
```

## פתרון בעיות

### פרוקסי לא מגיב

**בדוק:**

1. חיבור אינטרנט
2. פרטי התחברות
3. מצב ספק פרוקסי

**פתרון:**

```bash
# בדוק חיבור
ping proxy.com

# בדוק יציאה
telnet proxy.com 8080

# נסה פרוקסי חלופי
```

### מהירות איטית

**בדוק:**

1. סוג פרוקסי (מגורים איטי יותר)
2. מיקום שרת
3. עומס רשת

**שפר:**

```
→ החלף לפרוקסי קרוב יותר
→ השתמש ב-datacenter לפעולות לא רגישות
→ הפחת מספר משימות במקביל
```

### חשבונות נחסמים למרות פרוקסי

**סיבות אפשריות:**

1. ✗ פרוקסי datacenter משותף
2. ✗ שימוש רב מדי באותו IP
3. ✗ פרוקסי בבלאקליסט

**פתרון:**

```
→ עבור לפרוקסי מגורים
→ סבב IP בתדירות גבוהה יותר
→ השתמש בספק אחר
```

### שגיאות אימות

```
407 Proxy Authentication Required
```

**פתרון:**

1. ווידא שם משתמש וסיסמה
2. בדוק אם הפרוקסי פג
3. רענן אישורים

## תצורות מתקדמות

### שרשרת פרוקסי (Proxy Chain)

מעבר דרך מספר פרוקסי:

```
מחשב → Proxy 1 → Proxy 2 → יעד
```

:::warning
מאט את החיבור משמעותית.
:::

### SOCKS5 vs HTTP

**SOCKS5:**
- תמיכה מלאה בכל הפרוטוקולים
- מהיר יותר
- מומלץ

**HTTP:**
- פשוט יותר
- תמיכה רחבה
- טוב למרבית המקרים

### אימות

**בסיסי (Basic):**

```
http://username:password@proxy.com:8080
```

**IP Whitelist:**

```
http://proxy.com:8080
# הוסף IP שלך ברשימה הלבנה של הספק
```

## מחירים והערכה

### עלות ממוצעת

| סוג | מחיר לGB | מחיר לIP | מומלץ עבור |
|-----|----------|---------|------------|
| Residential | $5-15 | $0.50-2 | חשבונות חשובים |
| Datacenter | $0.10-1 | $0.05-0.20 | מבחנים |
| Mobile | $10-30 | $1-5 | פרימיום |

### תכנון תקציב

```
10 חשבונות × 1GB/חשבון × $10/GB = $100/חודש
```

**טיפים לחיסכון:**
- קנה חבילות גדולות
- השתמש ב-datacenter לפעולות בסיסיות
- שתף פרוקסי בין חשבונות (בזהירות)

## שאלות נפוצות

**ש: האם פרוקסי חובה?**  
ת: לא חובה, אך מומלץ מאוד לניהול ריבוי חשבונות.

**ש: כמה פרוקסי אני צריך?**  
ת: בערך פרוקסי אחד ל-3-5 חשבונות, תלוי בפעילות.

**ש: האם פרוקסי חינמי טובים?**  
ת: לא מומלץ. הם איטיים, לא אמינים וסיכון אבטחה.

**ש: האם VPN מספיק?**  
ת: VPN טוב אך פרוקסי עדיף - יותר גמישות ושליטה.
