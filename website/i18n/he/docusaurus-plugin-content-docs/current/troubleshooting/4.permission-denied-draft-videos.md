# הרשאה נדחתה לסרטוני טיוטה

פתרון בעיות גישה לסרטוני טיוטה במכשיר Android.

## הסיבה

TikMatrix זקוק להרשאות מערכת קבצים כדי לגשת לסרטוני טיוטה עבור פרסום אוטומטי.

## תסמינים

- ❌ "Permission Denied"
- ❌ "Cannot access /storage/..."
- ❌ משימות פרסום נכשלות
- ❌ לא מצליח לקרוא קבצי וידאו

## פתרון מהיר

### Android 11+

1. **פתח הגדרות** במכשיר
2. נווט ל**אפליקציות**
3. מצא **TikTok/Instagram**
4. הקש על **הרשאות**
5. הפעל **אחסון** → **אפשר ניהול כל הקבצים**

### Android 10 ומטה

1. **הגדרות** → **אפליקציות**
2. בחר **TikTok/Instagram**
3. **הרשאות** → **אחסון**
4. אפשר **קריאה וכתיבה**

## פתרונות מפורטים

### פתרון 1: הגדרות אפליקציה

**TikTok:**

```
הגדרות Android
→ אפליקציות
→ TikTok
→ הרשאות
→ קבצים ומדיה: אפשר
→ אחסון: אפשר
```

**Instagram:**

```
הגדרות Android
→ אפליקציות
→ Instagram
→ הרשאות
→ אחסון: אפשר
→ מצלמה: אפשר (לגלריה)
```

### פתרון 2: ADB Commands

אם אי אפשר לשנות ידנית, השתמש ב-ADB:

```bash
# TikTok
adb shell pm grant com.zhiliaoapp.musically android.permission.READ_EXTERNAL_STORAGE
adb shell pm grant com.zhiliaoapp.musically android.permission.WRITE_EXTERNAL_STORAGE

# Instagram
adb shell pm grant com.instagram.android android.permission.READ_EXTERNAL_STORAGE
adb shell pm grant com.instagram.android android.permission.WRITE_EXTERNAL_STORAGE
```

### פתרון 3: אתחל הרשאות

1. הסר התקנה של האפליקציה
2. נקה מטמון
3. התקן מחדש
4. הענק הרשאות בפעם הראשונה

### פתרון 4: בדוק הרשאות תיקייה

במכשיר, בדוק הרשאות:

```bash
# דרך ADB
adb shell ls -la /storage/emulated/0/DCIM/
adb shell ls -la /storage/emulated/0/Pictures/
```

ודא שהתיקיות קיימות וקריאות.

## בעיות ספציפיות

### Android 13+

Android 13 שינה מערכת הרשאות מדיה:

**דרושות הרשאות חדשות:**
- קריאת תמונות
- קריאת סרטונים
- קריאת אודיו

**פתרון:**

```bash
adb shell pm grant [package] android.permission.READ_MEDIA_IMAGES
adb shell pm grant [package] android.permission.READ_MEDIA_VIDEO
adb shell pm grant [package] android.permission.READ_MEDIA_AUDIO
```

### SELinux Enforcing

חלק מהמכשירים עם SELinux strict:

**בדוק מצב:**

```bash
adb shell getenforce
```

**אם Enforcing גורם לבעיות:**

```bash
# זמני (נמחק בהפעלה מחדש)
adb shell setenforce 0

# קבוע (דורש root)
adb shell su -c "setenforce 0"
```

:::warning
שינוי SELinux עלול להשפיע על אבטחה. השתמש בזהירות.
:::

### הרשאות Scoped Storage

Android 10+ השתמש ב-Scoped Storage:

**פתרון:**

הוסף ל-AndroidManifest.xml:

```xml
<application
    android:requestLegacyExternalStorage="true">
```

## אימות הרשאות

### שיטה 1: בדוק דרך הגדרות

```
הגדרות Android
→ אפליקציות
→ [אפליקציה]
→ הרשאות
```

ודא שהכל מאושר.

### שיטה 2: בדוק דרך ADB

```bash
# הצג כל ההרשאות
adb shell dumpsys package [package_name] | grep permission

# דוגמה TikTok
adb shell dumpsys package com.zhiliaoapp.musically | grep permission
```

### שיטה 3: נסה קריאת קובץ

```bash
# נסה לקרוא קובץ דרך TikMatrix
# אם נכשל, יש בעיית הרשאות
```

## שאלות נפוצות

**ש: האם צריך root?**  
ת: לא, כל ההרשאות הנדרשות זמינות ללא root.

**ש: האם בטוח להעניק כל ההרשאות?**  
ת: כן, TikMatrix זקוק להן רק לפונקציונליות.

**ש: מה קורה אם אני מסרב?**  
ת: פרסום אוטומטי לא יעבוד, תצטרך פרסום ידני.

**ש: האם ההרשאות מתאפסות?**  
ת: לפעמים לאחר עדכוני אפליקציה או מערכת.
