# לא ניתן לזהות טלפון

מדריך לפתרון בעיות כאשר TikMatrix לא מזהה את מכשיר ה-Android שלך.

## בדיקות מהירות

לפני שתתחיל בפתרון בעיות, בדוק:

- ✅ כבל USB מחובר כראוי
- ✅ המכשיר פועל
- ✅ המכשיר לא נעול (נמצא במסך הבית)
- ✅ TikMatrix פועל

## סיבות נפוצות

### 1. מצב ניפוי שגיאות USB לא מופעל

**תסמינים:**
- המכשיר לא מופיע ברשימה
- אין התראת חיבור USB

**פתרון:**

1. פתח **הגדרות** במכשיר Android
2. נווט ל**אודות הטלפון**
3. הקש על **מספר גרסה** 7 פעמים להפעלת מצב מפתח
4. חזור להגדרות → **אפשרויות מפתח**
5. הפעל **ניפוי שגיאות USB**
6. התחבר מחדש למחשב

### 2. דרייברים חסרים או מיושנים

**Windows:**

1. הורד דרייברים של יצרן המכשיר:
   - Samsung: [Samsung USB Driver](https://developer.samsung.com/mobile/android-usb-driver.html)
   - Xiaomi: [Mi USB Driver](https://developer.android.com/studio/run/oem-usb)
   - אחר: [Universal ADB Driver](https://adb.clockworkmod.com/)

2. התקן והפעל מחדש את המחשב

**macOS:**

בדרך כלל לא נדרשים דרייברים נוספים.

**Linux:**

הוסף כללי udev:

```bash
sudo usermod -aG plugdev $USER
sudo apt-get install android-tools-adb
```

### 3. כבל USB פגום

**תסמינים:**
- חיבור לא יציב
- ניתוק תכוף
- טעינה עובדת אך העברת נתונים לא

**פתרון:**
- נסה כבל USB אחר
- השתמש בכבל מקורי של היצרן
- ווידא שהכבל תומך בהעברת נתונים (לא רק טעינה)

### 4. יציאת USB לא עובדת

**פתרון:**
- נסה יציאת USB אחרת במחשב
- השתמש ביציאות USB 2.0 במקום 3.0
- נסה מחשב אחר לבדיקה

### 5. הגבלות אבטחה

**Samsung Knox / Mi Cloud:**

חלק מהמכשירים דורשים:

```
הגדרות → אבטחה → מקורות לא ידועים → אפשר
```

## אבחון שלב אחר שלב

### שלב 1: בדוק חיבור ADB

פתח מסוף/CMD והרץ:

```bash
adb devices
```

**תוצאה מצופה:**
```
List of devices attached
ABC123456789    device
```

**אם רשימה ריקה:**
- ניפוי שגיאות USB לא מופעל
- דרייברים חסרים
- כבל לא עובד

**אם מציג "unauthorized":**
- אשר את הפרומפט על המכשיר
- הפעל מחדש את המכשיר

### שלב 2: אתחל שרת ADB

```bash
adb kill-server
adb start-server
```

### שלב 3: בדוק ב-TikMatrix

1. פתח TikMatrix
2. לך ל**מכשירים**
3. לחץ על **רענן מכשירים**
4. המכשיר אמור להופיע

## פתרונות מתקדמים

### פתרון 1: אתחל הגדרות מפתח

```bash
# במכשיר Android
הגדרות → אפשרויות מפתח → בטל הרשאות ניפוי שגיאות USB
הפעל מחדש את אפשרויות מפתח
הפעל מחדש ניפוי שגיאות USB
```

### פתרון 2: שנה מצב USB

במכשיר, כשמחובר:

```
גלול מטה הודעות → התראת USB
שנה מ"טעינה בלבד" ל"העברת קבצים" או "PTP"
```

### פתרון 3: השתמש בחיבור TCP/IP

אם USB לא עובד, השתמש ב-WiFi:

```bash
# חבר דרך USB תחילה
adb tcpip 5555
# נתק USB
adb connect <IP_DEVICE>:5555
```

### פתרון 4: בדוק סכסוכי תוכנות

השבת זמנית:

- תוכנות אנטי-וירוס
- חומת אש
- תוכנות וירטואליזציה (VirtualBox, VMware)

## טיפים ספציפיים לפלטפורמה

### Windows

**נהל התקנים:**

1. Win + X → **נהל התקנים**
2. חפש **Android Device** או **ADB Interface**
3. אם סימן קריאה צהוב → עדכן דרייבר
4. לחצן ימני → **עדכן דרייבר** → **חפש אוטומטית**

### macOS

**בעיות הרשאות:**

```bash
sudo killall -STOP -c usbd
sudo launchctl stop com.apple.usbd
```

### Linux

**הוסף כללי udev:**

```bash
echo 'SUBSYSTEM=="usb", ATTR{idVendor}=="****", MODE="0666", GROUP="plugdev"' | sudo tee /etc/udev/rules.d/51-android.rules
sudo udevadm control --reload-rules
```

החלף `****` ב-vendor ID של המכשיר שלך.

## שאלות נפוצות

**ש: האם אני צריך root?**  
ת: לא, TikMatrix לא דורש גישת root.

**ש: האם אפשר לחבר דרך WiFi?**  
ת: כן, השתמש ב-ADB over TCP/IP לאחר חיבור USB ראשוני.

**ש: כמה מכשירים אפשר לחבר?**  
ת: תלוי בתוכנית שלך - עד 100 מכשירים בו-זמנית.

**ש: האם כל מכשירי Android נתמכים?**  
ת: רוב המכשירים עם Android 7.0+ נתמכים. כמה דגמים עשויים לדרוש הגדרה מיוחדת.

## קבל עזרה

אם הבעיה נמשכת:

1. שלח יומנים: `TikMatrix → הגדרות → שלח יומני ניפוי שגיאות`
2. בקר ב[פורום](https://forum.tikmatrix.com)
3. צור קשר עם [תמיכה](https://tikmatrix.com/support)

כלול:
- דגם מכשיר ו Android
- מערכת הפעלה של מחשב
- תוצאת `adb devices`
- צילומי מסך של שגיאות
