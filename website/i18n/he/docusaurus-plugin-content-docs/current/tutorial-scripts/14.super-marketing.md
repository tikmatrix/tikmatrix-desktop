# שיווק-על

:::info
סקריפט שיווק-על מרכז מעקב/ביטול מעקב, הודעות פרטיות, אינטראקציה עם פוסטים ותגובות לאותה תור משימות, מתאים להגעה מהירה ללקוחות פוטנציאליים או להערת מחדש של מעריצים ותיקים בסיבוב אחד.
:::

## שלבי פעולה

1. ב-TikMatrix/IgMatrix בחר את המכשירים שצריכים להריץ את הסקריפט (ניתן לבחור מרובים).
2. פתח `סקריפטים` → `שיווק-על`.
3. בחר **מקור נתונים** (שמות משתמש או קישורי פוסטים), ואז ניתן ללחוץ על כפתור `ייבוא קובץ txt` כדי לטעון את קובץ ה-`.txt` המתאים או פשוט להדביק מספר שורות של נתונים; לאחר הייבוא ניתן לבחור "הוספה" או "החלפה".
4. הפעל מודולי פעולות משתמש, פעולות פוסטים ותגובות לפי הצורך, והשלם את התצורה בהתאם להסבר למטה.
5. הגדר את המחוון **מרווח משימה** כדי להגדיר עיכוב אקראי בין יעדים, ולבסוף לחץ על `התחלת סקריפט` כדי לבצע את המשימה.

## הסבר תצורה

### מקור נתונים

- **שם משתמש / קישור פוסט**: קובע אילו פונקציות זמינות. מעקב, ביטול מעקב, הודעות פרטיות יכולים לשמש רק עם שמות משתמש; אם רוצים לפעול רק על פוסטים, ניתן לבחור ישירות במצב קישור.
- **שיטת ייבוא**: ניתן ללחוץ על כפתור `ייבוא קובץ txt` כדי לטעון את קובץ ה-`.txt` המתאים או פשוט להדביק מספר שורות של נתונים; לאחר הייבוא ניתן לבחור "הוספה" או "החלפה".

### אסטרטגיית שימוש בנתונים

- **מאגר משותף (Shared pool)**: כל חשבונות הבוט לוקחים משימות מאותו מאגר יעדים, מתאים לתרחישים שבהם מספר חשבונות יוצרים אינטראקציות מרובות על אותו יעד (לדוגמה דחיפה ריכוזית של פוסטים פופולריים).

- **צריכה חד-פעמית (Single-consumption)**: כל חשבון בוט מטפל רק ביעדים שונים, מבטיח שכל יעד מטופל רק על ידי חשבון אחד פעם אחת, מתאים לתרחישים שצריכים למנוע הגעה כפולה (לדוגמה הודעות פרטיות בכמות גדולה או תגובה ייחודית אחת).

  - **כמות צריכה מקסימלית למשימה**: במצב צריכה חד-פעמית, ניתן להגדיר את המספר המקסימלי של יעדים שנצרכים בכל הרצת משימה. זה יכול למנוע מהמשימה הראשונה לצרוך את כל הנתונים, ולהבטיח שיעדים עדיין זמינים כאשר חשבונות מתחלפים בהמשך. הגדרה ל-0 משמעותה ללא הגבלה (התנהגות ברירת מחדל).

בחירת האסטרטגיה תשפיע על הקצאת התור והתנהגות המקבילות, אנא בחר בהתאם ליעד ולשיקולי בקרת סיכונים.

### פעולות משתמש

- **מעקב אחר משתמש**: בצע פעולת מעקב תחילה לאחר כניסה לדף הבית.
- **ביטול מעקב אחר משתמש**: בטל מעקב, ניתן להריץ בנפרד או לנקות מעריצים לאחר הודעה פרטית; לא מומלץ להפעיל עם "מעקב" בו-זמנית.
- **דיווח על חשבון**: דווח על חשבון משתמש על קיום התנהגות הונאה. פונקציה זו משתמשת בנתיב דיווח קבוע, ואינה תומכת בנתיבים מותאמים אישית.

  :::warning נתיב דיווח (קבוע)
  דיווח → דיווח על חשבון → Something else → Frauds and scams
  :::

  - הסבר: פונקציה זו משתמשת בנתיב דיווח מוגדר מראש, ואינה תומכת בסיבות או נתיבי דיווח מותאמים אישית.
  - אנא השתמש בפונקציה זו באחריות, רק למטרות חוקיות. שימוש לרעה עלול להוביל להגבלת חשבון.
- **שליחת הודעה פרטית**: בחר באקראי הודעה אחת משורות מרובות לשליחה; סימון "הוספת אמוג'י אוטומטית" יכול לצרף אוטומטית אמוג'י טבעיים. הסקריפט יבדוק האם הצד השני מאפשר הודעות פרטיות, וידלג אם לא ניתן לשלוח.

  - תבניות הודעות פרטיות (מוכנות מראש): הדבק מספר שורות או פסקאות של טקסט הודעות פרטיות ב-"הגדרות הודעה" (כל שורה/פסקה נחשבת כהודעה מועמדת אחת).

  - סדר שימוש בטקסט: בחר "אקראי (Random)" או "לפי סדר (Sequential)".

    - אקראי: כל פעם בוחרים באקראי תבנית אחת לשליחה.

    - לפי סדר: משתמשים לפי סדר ההדבקה/ייבוא, מתאים לאסטרטגיית דה-דופליקציה שניתנת לשחזור.

  - הוספת אמוג'י (emoji): ניתן להפעיל הוספה אוטומטית של אמוג'י בהודעות או תגובות כדי להגביר את הטבעיות.

  - יצירה בזמן אמת (אופציונלי, ChatGPT או API תואם):

    - לאחר הפעלה, הסקריפט יכול לקרוא לממשק היצירה שהגדרת (מלא API Key / URL / Model / System Prompt) כדי ליצור טקסט מותאם אישית המבוסס על הקשר היעד.

    - הערה: דורש מתן API Key תקף, ושימו לב למגבלות קצב, עלויות ועמידה בפרטיות (הימנע משליחת נתונים רגישים).

  - אסטרטגיית ייחודיות: דרכים נפוצות להבטיח שכל הודעה/תגובה היא ייחודית:

      1. שימוש ב-"לפי סדר" כדי לעבור על תבניות מוכנות מראש (תבנית משמשת רק פעם אחת עד לסיום).
      2. שימוש ביצירה בזמן אמת (ChatGPT) כדי ליצור טקסט ייחודי לכל יעד.
  - לפני שימוש בהודעות פרטיות בכמות גדולה, אנא בדוק כמות קטנה ושימו לב לפרטיות ודרישות עמידה.

### פעולות פוסטים

- **מספר פוסטים לעיבוד**: קובע את מספר הפוסטים שכל חשבון צריך לפעול עליהם (1–50), נדרש רק כאשר מקור הנתונים הוא "שם משתמש".
- **מספר חזרות**: חזור על לולאת אינטראקציה עם פוסטים מספר פעמים, ללא צורך להפעיל מחדש את הסקריפט כדי לבצע פעולות שוב על פוסטים.
- **משך צפייה**: הגדר זמן צפייה מינימלי ומקסימלי (0–300 שניות). הסקריפט יבחר ערך אקראי בטווח ולאחר מכן יבצע אינטראקציה.
- **אינטראקציה עם פוסט**: ניתן לסמן בנפרד לייק, שמירה, שיתוף ושיתוף מחדש. כאשר מופעלות מספר פעולות, המערכת תפזר אותן בנקודות זמן שונות במהלך תקופת הצפייה, מה שנראה יותר טבעי.

- **תגובה**: לאחר הפעלה ניתן לפרסם תגובות על פוסטים.

     אם הפעלת "תגובה", אנא בצע את התצורה הבאה בהגדרות תגובה:

  - תבניות תגובה מוכנות מראש: ניתן להדביק מספר שורות או פסקאות של טקסט תגובה מוכן מראש (כל שורה או פסקה נחשבת כתגובה מועמדת אחת).

  - סדר שימוש בטקסט: בחר "אקראי" או "לפי סדר".
    - אקראי: כל פעם בוחרים באקראי טקסט אחד מהטקסטים המועמדים.
    - לפי סדר: משתמשים לפי סדר ההדבקה/ייבוא (מתאים לאסטרטגיית דה-דופליקציה שניתנת לשחזור).

  - הוספת אמוג'י (emoji): ניתן להפעיל הוספה אוטומטית של אמוג'י בתגובות כדי להגביר את הטבעיות.

  - יצירת תגובות בזמן אמת (אופציונלי, ChatGPT API):
    - לאחר הפעלה, הסקריפט יכול לקרוא ל-ChatGPT API, ליצור טקסט תגובה רלוונטי להקשר וייחודי בהתבסס על כותרת/תיאור הפוסט. שיטה זו יכולה להפחית משמעותית את הסיכון לתגובות כפולות, מתאים לתרחישים שדורשים כמות גדולה של תגובות ייחודיות.
    - הערות: שימוש ביצירה בזמן אמת דורש מתן API Key זמין, ושימו לב למגבלות קצב, עלויות ועמידה בפרטיות (אל תשלחו נתונים רגישים לשירותי צד שלישי).

  - שתי דרכים נפוצות להבטיח תגובות ייחודיות:
         1. שימוש ב-"לפי סדר" לטעינה ועבירה על תבניות תגובה מוכנות מראש (מבטיח שכל תבנית משמשת רק פעם אחת עד לסיום).
         2. שימוש ב-ChatGPT API כדי ליצור כל תגובה בזמן אמת (יצירה דינמית המבוססת על כותרת/תוכן הפוסט, טבעית יותר להבטחת ייחודיות).

  - המלצה: בדוק תחילה את אסטרטגיית התגובה על כמות קטנה של פוסטים, בדוק את איכות התגובה ואת הטיפול של הפלטפורמה; אם משתמשים ביצירה בזמן אמת, נטר את כמות קריאות ה-API ואת העלות.

  - אם משתמשים באסטרטגיית "סדר", המערכת תשלב את מספר המשימה כדי לעבור על תבניות במכשירים מרובים, להפחית את ההסתברות לכפילות; אם משתמשים ביצירת AI, מומלץ לבצע בדיקות איכות פשוטות על הפלט (כגון טווח אורך, איסור על הכללת URL / מילים רגישות).

### מיזוג משימות (מיזוג משימות של אותו שם משתמש)

- **הסבר**: כאשר מופעל "מיזוג משימות של אותו שם משתמש", מספר יעדים עבור אותו שם משתמש יבוצעו בסדר או במקביל (תלוי ביישום הסקריפט) באותה משימה; גם אם יעד מסוים נכשל, המשימה לא תיפסק כולה, היעדים הנותרים ימשיכו לבצע ולנסות להשלים, ובכך להפחית את עלות החלפה ותזמון המשימות.

- **התנהגות בעת הפעלה**: מספר יעדים עבור אותו חשבון יאוחדו ליחידת משימה אחת, כשל ישפיע רק על היעד עצמו וירשם, אך לא יחסום את ההמשך של יעדים אחרים. מתאים לשימוש בעת הרצת שיווק בקנה מידה גדול, יכול להפחית משמעותית את תדירות ההפעלה/החלפה של משימות.

- **התנהגות בעת כיבוי**: כל יעד יייצר משימה נפרדת, כשל של כל יעד בודד ידווח מיידית ויעצור את אותה תת-משימה (לא ישפיע על משימות עצמאיות אחרות). מתאים לתרחישים שדורשים ניטור מדויק ואזעקה לכל יעד בנפרד.

- **המלצות שימוש**:
  - בעת שיווק בכמות גדולה או צורך לשמור על תפוקה גבוהה, מומלץ להפעיל כדי להפחית את עלות התזמון.
  - כאשר צריך לאשר את מצב הביצוע של כל יעד בנפרד ולפעול מיידית בעת כשל (לדוגמה התערבות ידנית או ביטול מיידי), אנא כבה אפשרות זו.

### מרווח משימה

- הגדר ערך מינימלי ומקסימלי (יחידה: דקות), הסקריפט יחכה באקראי בין כל יעד לפי הטווח, ויוסיף עיכוב קצר משלו.

## תהליך ביצוע

- אם צוין שהופעל מצב חשבונות מרובים, TikMatrix/IgMatrix יפעיל תחילה את TikTok/IgMatrix ויעבור לחשבון המתאים.
- במצב שם משתמש, המערכת פותחת כל דף בית לפי שיטת הגישה שנבחרה, ממתינה להשלמת הטעינה ולאחר מכן מבצעת ברצף פעולות כגון מעקב, הודעה פרטית, ביטול מעקב וכו'; אם הצד השני אוסר על הודעות פרטיות תדלג אוטומטית.
- אינטראקציה עם פוסט תגלול לעיון בדף הבית או תעבור ישירות לקישור, תצפה ותבצע לייק, שמירה, שיתוף, שיתוף מחדש לפי המספר והזמן שהוגדרו, ותדלג על מודעות, שידורים חיים ותוכן שכבר קיבל לייק.
- בעת שימוש בקישורי פוסטים, הסקריפט יפתח כל קישור ברצף, יבצע את הפעולות שהוגדרו, ויחזור על כך לפי "מספר חזרות".
- מודול התגובה ייקח מילים מתבניות או פלט AI. במצב סדר, הוא ישלב את מספר המשימה כדי להקצות תגובות שונות בין מכשירים מרובים.
- בין כל יעד, הסקריפט יוסיף עיכוב אקראי קטן, ואז יוסיף את ההמתנה האקראית של מרווח המשימה, כדי לחקות קצב אנושי.

## שיטות עבודה מומלצות

- בדוק תחילה עם כמות קטנה של נתונים את הטקסט והקצב, אשר שאין טעויות ולאחר מכן הגדל את הקנה מידה.
- שילוב של מספר פעולות (כגון מעקב + הודעה פרטית + לייק) יכול לשפר את אותות האינטראקציה של החשבון.
- הודעות פרטיות ותגובות צריכות להיות דיבוריות וכנות, הימנע מהפעלת בקרת סיכונים של הפלטפורמה.
- עדכן באופן קבוע את ספריית התגובות או התאם את ה-System Prompt של ה-AI כדי להפוך את התוכן לרלוונטי יותר לנושא הפעילות.
- שימו לב למגבלת קצב הפלטפורמה, אם הכמות גדולה ניתן להחליף חשבונות או צמתי פרוקסי.
