# 翻译项目说明

## 项目背景

这是一个基于 **Docusaurus** 的文档网站。目录是./website
`i18n` 目录中已创建 **21 种语言**，所有目标语言文件均已存在，目前内容为**英文占位文本**。

其中 **zh-Hans（简体中文）** 已完整且高质量翻译，作为**唯一参考标准**，用于：

* 术语一致性
* 语气与文风
* 句式结构
* 技术准确性
* 文档风格

### 你的任务

将各语言目录中的**英文占位内容**翻译为对应语言。

### 翻译优先级（Source of Truth）

翻译时优先参考顺序：

1. **zh-Hans 的表达方式**
2. 原始英文含义（兜底）

> 如果中文使用了更自然或更本地化的表达，应**优先采用**，而不是直译英文。
>

### 文件类型

* Markdown（`.md` / `.mdx`）
* JSON 翻译文件
* Docusaurus theme i18n 文件

### Markdown & Docusaurus 规则（⚠️关键）

**不要翻译：**

* frontmatter 的 key（value 如 `title` 需要翻译）
* Markdown 语法
* 代码块、行内代码
* 文件路径、URL、CLI 命令
* 产品名 **TikMatrix**

**必须保留：**

* 标题层级
* 链接
* 图片
* 表格
* Callout / Admonition

### 写作风格

* 专业技术文档语气
* 清晰、简洁、符合母语习惯
* 无机翻痕迹
* 无俚语、无 emoji
* 不偏离原意

### 语言专项要求

* **阿拉伯语 / 希伯来语**：RTL 语序与标点正确
* **日语 / 韩语**：简洁、技术文档风格
* **德语**：正式、严谨
* **罗曼语系**：中性 SaaS 文档语气
* **印地语 / 泰语 / 越南语**：现代技术词汇

### 约束说明

* 不评估项目规模
* 不讨论可行性
* 不提前中断
* 不泛化、不发挥

### 执行方式说明（重要）

* **不需要**一次性将所有语言目录加载到上下文中
* **按语言逐个处理即可**：

  * 每次只读取 **一个语言目录**
  * 参考 `zh-Hans`
  * 完成该语言的全部文件翻译后，再继续下一个语言
* 各语言之间**互不依赖**，禁止混用上下文内容
* 不要因为语言数量多而省略、合并或跳过任何文件
